
## Git.Rebase
- `git rebase -i 최초커밋`
  - `git rebase -i 2c2f89f`
  - `git rebase -i 살리고싶은커밋`
- `d`

  - `pick ...`부분
  - `d ...`   변경

##
커밋 히스토리를 깔끔하게 정리하고 싶으시군요! 즉, **최초 커밋(First Commit)**과 **가장 최신 커밋(Last Commit)**만 남기고 그 사이의 자잘한 수정 기록들을 하나로 합치거나 삭제하고 싶으신 상황이네요.

가장 안전하고 확실한 방법은 **`git rebase -i` (대화형 리베이스)**를 사용하는 것입니다.

---

### 1. 커밋 히스토리 확인
먼저 터미널에서 아래 명령어를 입력해 커밋의 해시값(ID)을 확인합니다.
```bash
git log --oneline
```
그러면 아래와 비슷하게 뜰 거예요.
* `ee0c6ce` (최신) - **살릴 것**
* `c6a45fe` (중간) - 지울 것
* `164b202` (중간) - 지울 것
* `a1b2c3d` (최초) - **살릴 것**

---

### 2. 리베이스(Rebase) 시작
최초 커밋(`a1b2c3d`) 바로 다음부터 최신까지를 범위로 잡아야 합니다.
```bash
# 최초 커밋의 해시값을 입력하세요
git rebase -i a1b2c3d
```

### 3. 커밋 편집 (중요!)
명령어를 입력하면 텍스트 편집기(보통 Vim)가 열립니다. 목록이 위에서 아래로 **오래된 순서**로 나열됩니다.

1.  가장 위에 있는(가장 오래된 중간 커밋)은 건드리지 않거나, 
2.  지우고 싶은 커밋 앞의 단어를 `pick`에서 **`drop`** (또는 **`d`**)으로 바꿉니다.
3.  만약 지우는 게 아니라 **내용은 남기고 기록만 합치고 싶다면** **`squash`** (또는 **`s`**)를 씁니다.

```text
pick 164b202 중간 커밋 1  --> d 164b202 중간 커밋 1 (삭제)
pick c6a45fe 중간 커밋 2  --> d c6a45fe 중간 커밋 2 (삭제)
pick ee0c6ce 최신 커밋    --> pick ee0c6ce (그대로 둠)
```
* **Vim 사용법:** `i`를 눌러 수정 모드로 진입 -> 글자 수정 -> `Esc` 누름 -> `:wq` 입력 후 엔터.

### 4. GitHub에 강제 반영
로컬의 히스토리를 강제로 바꿨기 때문에, 서버(GitHub)와 충돌이 납니다. 이때는 강제로 덮어씌워야 합니다.

```bash
git push origin main --force
```

---

### ⚠️ 주의할 점 (꼭 읽어주세요!)
* **`drop`을 하면:** 그 커밋에서 작업했던 **코드 내용 자체가 사라집니다.** * **내용은 유지하고 기록만 지우고 싶다면:** `drop` 대신 **`squash`**를 선택하세요. 그러면 이전 커밋과 합쳐지면서 코드는 살아남고 커밋 로그만 깔끔해집니다.
* **최초 커밋 자체는 리베이스 대상이 될 수 없습니다.** (뿌리이기 때문입니다.) 그래서 최초 커밋의 해시를 입력해 그 이후부터 편집하는 방식을 씁니다.



---

### 💡 만약 너무 꼬여서 다 필요 없고 "현재 상태"를 최초 커밋으로 만들고 싶다면?
이게 가장 속 편한 방법일 수도 있습니다.

1.  `.git` 폴더 삭제: `rm -rf .git`
2.  다시 시작: `git init`
3.  원격 연결: `git remote add origin [주소]`
4.  전체 추가: `git add .`
5.  새로운 최초 커밋: `git commit -m "Initial Commit (Cleaned)"`
6.  강제 푸시: `git push -u origin main --force`

이렇게 하면 기존의 지저분한 히스토리가 싹 날아가고 지금 내 컴퓨터에 있는 파일 상태 그대로가 **첫 번째 커밋**이 됩니다. (협업 중이 아닌 개인 프로젝트라면 이 방법이 가장 깔끔합니다!)

어떤 방식으로 진행해 볼까요? 히스토리를 세세하게 만지는 게 처음이라면 두 번째 방법(다시 시작)을 추천드려요!
