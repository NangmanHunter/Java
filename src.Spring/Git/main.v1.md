이미 GitHub에 원격 레포지토리를 만드셨군요! 이제 내 컴퓨터에 있는 프로젝트를 그곳으로 쏘아 올릴 차례입니다.

터미널(또는 CMD)을 열고 프로젝트의 **루트 폴더(build.gradle 파일이 있는 곳)**로 이동한 뒤, 아래 명령어를 순서대로 입력하시면 됩니다.

---

### 1. 로컬 저장소 초기화 및 연결

가장 먼저 내 폴더를 "Git이 관리하는 폴더"로 만들고, GitHub 주소를 알려줘야 합니다.

```bash
# 1. 현재 폴더를 Git 저장소로 초기화
git init

# 2. 모든 파일을 장바구니(Stage)에 담기
git add .

# 3. 첫 번째 기록 남기기
git commit -m "Initial Commit: JPA 게시판 프로젝트 시작"

# 4. GitHub 원격 주소 연결 (주소 부분은 본인 레포지토리 주소로 수정)
git remote add origin https://github.com/사용자아이디/레포지토리이름.git

# 5. 기본 브랜치 이름을 main으로 변경 (최신 표준)
git branch -M main
```

---

### 2. GitHub로 코드 밀어넣기 (Push)

이제 연결이 끝났으니 실제 파일을 서버로 보냅니다.

```bash
# 6. 원격 저장소(origin)의 main 브랜치로 업로드
git push -u origin main
```

> **잠깐!** 만약 GitHub 레포지토리를 만들 때 `README.md`나 `LICENSE` 파일을 미리 생성했다면, `push` 할 때 에러가 날 수 있습니다. 그럴 땐 당황하지 말고 아래 명령어를 먼저 입력하세요.
> `git pull origin main --rebase` (서버에 있는 걸 먼저 가져와서 합치는 과정입니다.)

---

### 3. 스프링 프로젝트에서 꼭 체크할 것 (`.gitignore`)

스프링 프로젝트를 올릴 때는 **올려서는 안 되는 파일들**이 있습니다. (예: `build/` 폴더, `.gradle` 폴더, 인텔리제이 설정 파일 등)

* 프로젝트 루트에 **`.gitignore`** 파일이 있는지 확인하세요.
* `start.spring.io`에서 만드셨다면 자동으로 생성되어 있을 겁니다. 
* 만약 이 파일이 없으면 `build` 폴더 같은 쓰레기 파일들까지 전부 GitHub에 올라가서 나중에 프로젝트가 지저분해지니 꼭 확인하고 `git add .`를 하시는 게 좋습니다.

---

### 4. 인텔리제이(IntelliJ)를 쓰고 계신다면?

명령어가 익숙하지 않다면 인텔리제이 메뉴를 통해서도 아주 쉽게 할 수 있습니다.

1.  상단 메뉴 **[Git]** -> **[Manage Remotes...]** 클릭 후 GitHub 주소 등록
2.  **`Ctrl + K`** (Commit): 왼쪽 창에서 파일 선택 후 메시지 적고 `Commit` 버튼 클릭
3.  **`Ctrl + Shift + K`** (Push): `Push` 버튼 클릭 끝!



---

### 💡 한 줄 요약
**`init -> add -> commit -> remote add -> push`** 이 5단계만 기억하시면 됩니다.
