`git push -u origin main`에서 **`-u`**는 **`--set-upstream`**의 약자입니다.

쉽게 설명하자면, **"앞으로 내가 이 프로젝트에서 `git push`나 `git pull`이라고만 쳐도, 자동으로 `origin`의 `main` 브랜치랑 통신하도록 연결해줘!"**라고 설정하는 옵션입니다.

구체적으로 어떤 역할을 하는지 정리해 드릴게요.

---

### 1. "업스트림(Upstream)" 설정이란?
내 컴퓨터에 있는 로컬 브랜치(`main`)와 GitHub에 있는 원격 브랜치(`origin/main`) 사이에 **고정적인 연결 통로**를 만드는 것입니다. 

* **`-u`를 쓰기 전:** 매번 `git push origin main`, `git pull origin main`처럼 주소(`origin`)와 브라우저 이름(`main`)을 다 적어줘야 합니다.
* **`-u`를 한 번 쓴 후:** 다음부터는 그냥 **`git push`** 또는 **`git pull`**만 입력해도 Git이 "아, 저번에 연결해둔 `origin`의 `main`으로 보내달라는 거구나!"라고 알아서 처리합니다.

---

### 2. 왜 처음 한 번만 쓰나요?
이 연결 설정은 한 번만 해두면 Git 설정 파일에 저장됩니다. 그래서 프로젝트를 처음 GitHub에 올릴 때 딱 한 번만 사용하고, 그 뒤로는 생략해도 되는 것입니다.



---

### 3. 쓰면 좋은 점 (편의성)
* **타이핑 단축:** 매번 뒤에 긴 이름을 붙일 필요가 없어 편리합니다.
* **상태 확인:** `git status`를 쳤을 때, 내 로컬 코드가 서버보다 몇 개의 커밋이 앞서 있는지 혹은 뒤처져 있는지 구체적으로 알려줍니다. 
    * 예: *"Your branch is ahead of 'origin/main' by 2 commits."*
