##
`main` 메서드가 실행(Run) 버튼이 활성화되지 않거나 실행이 안 되는 이유는 보통 **IDE(IntelliJ 등)의 설정 문제**이거나 **프로젝트 구조 인식 문제**일 확률이 높습니다.

가장 흔한 원인들과 해결 방법을 단계별로 체크해 보세요.

---

### 1. `build.gradle` 새로고침 (가장 빈번한 원인)
프로젝트를 처음 만들거나 외부에서 가져왔을 때, IDE가 Gradle 설정을 제대로 읽지 못해 자바 소스 코드를 일반 텍스트 파일로 인식할 때가 있습니다.

* **해결:** 오른쪽 사이드바에 있는 **Gradle 탭**을 열고, **새로고침(Reimport/Sync) 아이콘**을 클릭하세요.
* 하단 상태 표시줄에서 `Indexing...`이나 `Gradle Sync`가 끝날 때까지 기다린 후 다시 확인해 보세요.

### 2. 소스 루트(Source Root) 설정 확인
현재 패키지 경로(`org.bitcopark.base`)가 프로젝트의 소스 폴더 경로와 일치해야 합니다.

* **확인:** 프로젝트 폴더 구조에서 `BaseApplication.java`가 `src/main/java` 폴더 안에 정상적으로 들어있는지 확인하세요.
* 만약 폴더 아이콘이 파란색이 아니라면, `src/main/java` 우클릭 -> **Mark Directory as** -> **Sources Root**를 선택하세요.

### 3. JDK 설정 누락
프로젝트에 자바 버전(SDK)이 할당되지 않았을 때 발생합니다.

* **IntelliJ 기준:** `File` > `Project Structure` > `Project` 탭에서 **SDK**가 올바르게 선택되어 있는지 확인하세요. (없다면 설치된 JDK를 선택해주세요.)

### 4. IntelliJ 캐시 문제
설정은 다 맞는데 가끔 IDE가 꼬여서 안 보일 때가 있습니다.

* **해결:** 상단 메뉴에서 `File` > `Invalidate Caches...` 를 누르고, 모든 항목 체크 후 `Invalidate and Restart`를 클릭하세요.

### 5. 코드상의 미세한 오타 (패키지 구조)
작성하신 코드를 보면 패키지명이 `org.bitcopark.base`로 되어 있습니다. 
실제 폴더 구조가 `src/main/java/org/bitcopark/base/BaseApplication.java`와 정확히 일치하는지 다시 한번 확인해 보세요.

---

**💡 팁:** 만약 실행 버튼은 뜨는데 에러가 나면서 멈춘다면, 에러 메시지를 복사해서 알려주세요! 그 메시지에 바로 답이 있을 확률이 높습니다. 

현재 **Member 파트** 개발을 위해 환경 세팅 중이신 것 같은데, 이 단계만 넘기면 바로 `Member` 엔티티부터 만드실 수 있을 거예요. 해결이 안 되면 어떤 IDE(IntelliJ, Eclipse 등)를 쓰시는지도 알려주세요!
