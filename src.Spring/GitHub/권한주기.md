## 권한주기
- `Settings`
- `Access`
- `Collaborators and teams`
- `Add people`


##
팀 프로젝트 레포지토리라면 단순히 주소만 알려준다고 해서 팀원이 코드를 올릴(Push) 수는 없습니다. **'Collaborator(협업자)'**로 등록해서 권한을 부여해야 합니다.

가장 빠르고 정확한 방법을 단계별로 알려드릴게요.

---

### 1. 팀원 초대하기 (Collaborator 추가)

팀원이 내 레포지토리에 커밋하고 푸시할 수 있게 하려면 이 과정을 반드시 거쳐야 합니다.

1.  GitHub 레포지토리 페이지 상단의 **[Settings]** 탭을 클릭합니다.
2.  왼쪽 메뉴에서 **[Access] -> [Collaborators]**를 선택합니다.
3.  **[Add people]** 버튼을 누릅니다.
4.  팀원의 **GitHub ID** 또는 **이메일 주소**를 입력해 검색한 뒤 추가합니다.
5.  **중요:** 초대장을 보내면 팀원이 본인의 **이메일이나 GitHub 알림**에서 **[Accept Invitation]**을 눌러 수락해야 최종적으로 권한이 생깁니다. (수락 전에는 'Pending' 상태로 뜹니다.)



---

### 2. 권한 설정 (Role)

개인 계정의 레포지토리라면 기본적으로 **'Write'** 권한이 부여됩니다. 이 권한이면 팀원이 코드를 수정하고, 푸시하고, 브랜치를 만드는 모든 작업을 할 수 있습니다.

* 만약 **GitHub Organization(단체 계정)**을 사용 중이라면 `Read`, `Triage`, `Write`, `Maintain`, `Admin` 등 세부 권한을 줄 수 있는데, 일반적인 개발 협업은 **`Write`** 권한이면 충분합니다.

---

### 3. 브랜치 보호 설정 (선택 사항이지만 추천)

여러 명이 작업하다 보면 실수로 `main` 브랜치의 코드를 날려버릴 수 있습니다. 이를 방지하기 위해 **Branch Protection Rule**을 설정하는 것이 좋습니다.

1.  **[Settings] -> [Code and automation] -> [Branches]**로 이동합니다.
2.  **[Add branch protection rule]**을 누릅니다.
3.  `Branch name pattern`에 **main**을 입력합니다.
4.  **[Require a pull request before merging]**을 체크합니다.
    * 이렇게 하면 팀원이 직접 `main`에 푸시하지 못하고, 반드시 **Pull Request(PR)**를 올려서 검토를 받아야만 합칠 수 있게 됩니다.

---

### 💡 팀원이 해야 할 일

초대를 수락한 팀원은 이제 자기 컴퓨터로 코드를 가져가야 합니다. 팀원에게 아래 내용을 전달해 주세요.

```bash
# 레포지토리 복제하기
git clone https://github.com/BitcoPark/bitcopark-base.git

# 본인만의 작업 브랜치 만들기
git checkout -b feature/기능이름
```



**요약하자면:** 지금 바로 **Settings -> Collaborators**에서 팀원 ID를 넣고 초대장을 보내세요! 그리고 팀원에게 "메일함 확인해서 수락 버튼 눌러줘"라고 말하면 끝납니다. 

팀원들이 다 들어오면 이제 누가 어떤 기능을 맡아서 개발할지 정하셨나요?
