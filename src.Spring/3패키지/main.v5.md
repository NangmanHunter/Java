자바와 스프링 패키지 네이밍에는 아주 엄격하고 명확한 **대원칙**이 있습니다. 결론부터 말씀드리면, 패키지 명에는 **케밥 케이스(`kebab-case`)를 절대 사용하지 않습니다.**

패키지 네이밍의 핵심 규칙과 케이스 규정을 정리해 드릴게요.

---

### 1. 패키지명은 무조건 '소문자' (Lower Case)
자바의 공식 관습상 패키지 이름은 **모두 소문자**로 작성합니다. 단어와 단어 사이를 구분할 때도 대문자를 쓰지 않는 것이 원칙입니다.

* **올바른 예:** `com.shop.order`, `com.shop.memberlogin`
* **잘못된 예:** `com.shop.Order`, `com.shop.MemberLogin`

### 2. 케밥 케이스(`-`)를 쓰지 못하는 이유
자바에서 패키지 명은 실제 소스 코드 내에서 **식별자(Identifier)**로 사용됩니다. 

* 자바 코드에서 하이픈(`-`)은 **뺄셈 연산자**로 인식됩니다.
* 만약 패키지 이름이 `com.my-project`라면, 자바 컴파일러는 이를 "com.my에서 project를 빼라"는 식으로 오해하여 에러를 발생시킵니다. 
* 따라서 하이픈(`-`)이나 특수문자는 패키지 명에 사용할 수 없습니다.

### 3. 단어 구분은 어떻게 하나요?
여러 단어를 조합해야 할 때 자바 개발자들이 사용하는 방식은 크게 두 가지입니다.

#### ① 그냥 붙여 쓰기 (가장 권장)
자바 공식 문서에서는 가급적 단어를 그냥 소문자로 붙여 쓰는 것을 권장합니다. 조금 읽기 불편하더라도 말이죠.
* 예: `memberlogin`, `orderhistory`

#### ② 언더스코어(`_`) 사용 (스네이크 케이스)
단어가 너무 길어서 도저히 읽기 힘들 때는 언더스코어를 쓸 수는 있지만, 이마저도 자바 커뮤니티에서는 **최후의 수단**으로 여깁니다. 웬만하면 피하는 것이 좋습니다.
* 예: `member_login` (가능은 하지만 선호되지 않음)

#### ③ 점(`.`)으로 계층 나누기
단어를 구분하는 가장 '자바스러운' 방법은 패키지를 한 단계 더 파는 것입니다.
* 예: `com.shop.member.login` (추천)

### 📋 자바 주요 케이스 규정 요약

| 대상 | 권장 케이스 | 예시 |
| :--- | :--- | :--- |
| **패키지 (Package)** | **전체 소문자** | `com.shop.api` |
| **클래스 (Class)** | **파스칼 케이스** (첫 글자 대문자) | `MemberService` |
| **변수/메서드 (Variable/Method)** | **카멜 케이스** (첫 글자 소문자) | `orderCount`, `saveMember()` |
| **상수 (Constant)** | **스크림 스네이크 케이스** (대문자+`_`) | `MAX_RETRY_COUNT` |



### 💡 팁: 케밥 케이스는 어디서 쓰나요?
질문하신 **케밥 케이스(`kebab-case`)**는 자바 코드 안이 아니라, 주로 **외부 설정**에서 쓰입니다.
* `application.properties`나 `application.yml` 같은 설정 파일의 키 값
* 프로젝트의 루트 폴더 이름 (Artifact ID)
* URL 주소 (예: `localhost:8080/member-login`)

스프링 프로젝트를 하실 때는 **"코드는 무조건 자바 스타일(소문자/카멜), 설정과 URL은 케밥 스타일"**이라고 머릿속에 넣어두시면 편합니다!
