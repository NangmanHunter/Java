에러 로그를 보니 원인이 아주 명확합니다! 범인은 바로 **"디폴트 패키지(Default Package)"** 사용과 **"잘못된 프로젝트 구조"**입니다.

에러 로그의 이 부분을 주목해 주세요:
> `** WARNING ** : Your ApplicationContext is unlikely to start due to a @ComponentScan of the default package.`

---

### 1. 원인 분석
현재 `LinkViewerApplication` 클래스가 **패키지 선언 없이** `src/main/java` 바로 아래에 있는 것 같습니다. 

스프링 부트는 `@SpringBootApplication`이 붙은 클래스의 위치를 기준으로 그 하위 패키지들을 다 뒤져서 `@Service`, `@RestController`를 찾습니다. 그런데 클래스가 패키지 없이(Default Package) 있으면, 스프링은 **자바 라이브러리 내부 패키지까지 전부 뒤지려고 시도**하게 됩니다. 그러다 보니 내부에 있지도 않은 데이터베이스 관련 클래스(`r2dbc`) 등을 건드리다가 에러가 빵 터진 겁니다.

---

### 2. 해결 방법: 패키지 구조 만들기

프로젝트 구조를 아래와 같이 **반드시** 패키지 폴더 안에 넣어주어야 합니다.

**[잘못된 구조]**
```text
src/main/java/
  ├── LinkViewerApplication.java (여기에 있으면 안 됨!)
  ├── MetaDataService.java
  └── LinkController.java
```

**[올바른 구조 (추천)]**
```text
src/main/java/
  └── com/
      └── example/
          └── linkviewer/
              ├── LinkViewerApplication.java
              ├── MetaDataService.java
              └── LinkController.java
```

### 3. 단계별 수정 사항

1.  **폴더 생성**: `src/main/java` 아래에 `com/example/linkviewer` 폴더를 만드세요.
2.  **파일 이동**: 작성하신 `.java` 파일 3개를 모두 그 폴더 안으로 옮기세요.
3.  **코드 수정**: 모든 파일의 맨 윗줄에 패키지 경로를 적어주세요.

**LinkViewerApplication.java 예시:**
```java
package com.example.linkviewer; // 반드시 이 줄이 있어야 합니다!

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class LinkViewerApplication {
    public static void main(String[] args) {
        SpringApplication.run(LinkViewerApplication.class, args);
    }
}
```

**MetaDataService.java / LinkController.java** 파일 상단에도 똑같이 `package com.example.linkviewer;`를 넣어주세요.
