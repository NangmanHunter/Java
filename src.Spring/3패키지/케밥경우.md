## 붙여쓰기
```
- Domain : C.B.A
- Package: A.B.C
```
```
₋ Domain : C.B₂₋B₁.A
₋ Package: A.B₂B₁.C
```
```
- Domain : D-C.B.A
- Package: A.B.CD
```


## 케밥도메인
- [x] 붙여쓰기`A.B.CD`
- [ ] 스네이크`A.B.C_D`
- [ ] 닷케이스`A.B.C.D`


##
진짜 도메인 주소에 하이픈(`-`)이 포함된 경우(예: `my-shop.com`), 자바 패키지 규약과 충돌하기 때문에 고민이 깊어지실 수 있습니다. 앞서 말씀드린 대로 자바 패키지명에는 하이픈을 쓸 수 없으니까요.

이런 상황에서 현업 개발자들이 사용하는 **3가지 해결책**을 정리해 드립니다.

---

### 1. 하이픈 제거 후 붙여쓰기 (가장 추천)
가장 일반적이고 권장되는 방식입니다. 하이픈을 단순히 제거하고 모든 단어를 소문자로 붙여서 씁니다.

* **실제 도메인:** `my-shop.com`
* **패키지 명:** `com.myshop.api`
* **장점:** 자바의 표준 관습에 가장 부합하며, 코드가 깔끔해집니다. 대부분의 오픈소스 라이브러리들도 이 방식을 사용합니다.


### 2. 언더스코어(`_`)로 대체
하이픈을 언더스코어로 바꾸는 방식입니다. 도메인의 시각적 구조를 유지하고 싶을 때 사용합니다.

* **실제 도메인:** `my-shop.com`
* **패키지 명:** `com.my_shop.api`
* **단점:** 자바 패키지명에 언더스코어를 쓰는 것은 문법적으로는 가능하지만, 관습적으로는 "지저분하다"고 느껴져 기피하는 경향이 있습니다.


### 3. 점(`.`)으로 계층 분리
하이픈 앞뒤 단어를 별도의 패키지 단계로 나누는 방식입니다.

* **실제 도메인:** `my-shop.com`
* **패키지 명:** `com.my.shop.api`
* **장점:** 단어 구분이 명확해집니다.
* **단점:** 폴더 깊이가 한 단계 더 깊어지므로 프로젝트 구조가 복잡해 보일 수 있습니다.


---



### 💡 결정 가이드: 어떤 게 제일 나을까요?

| 우선순위 | 방식 | 예시 | 비고 |
| :--- | :--- | :--- | :--- |
| **1순위** | **그냥 붙이기** | `com.myshop` | **강력 추천.** 가장 자바다운 방식입니다. |
| **2순위** | **계층 분리** | `com.my.shop` | 단어의 의미를 꼭 분리해야 할 때 사용합니다. |
| **3순위** | **언더스코어** | `com.my_shop` | 도메인 이름을 그대로 보존해야 할 때만 씁니다. |

**결론적으로 `com.myshop.api` 형식을 가장 추천드립니다.** 실제로 `spring.io`나 `google.com` 같은 거대 기업들도 내부 프로젝트에서 하이픈이 있는 이름을 써야 할 때는 보통 하이픈을 없애고 붙여버리는 방식을 택합니다. 주소창의 도메인과 패키지명이 100% 글자 하나하나 일치하지 않아도 개발자들은 그것이 해당 도메인을 의미한다는 것을 충분히 이해하니까요.
