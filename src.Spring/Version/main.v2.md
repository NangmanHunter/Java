아하, `start.spring.io` 사이트 구성이 가끔 업데이트되면서 아주 최신 버전들만 노출될 때가 있죠. 지금 화면에 **3.4.4**나 **3.4.5** 같은 숫자는 없고 바로 **3.5.x**대부터 보인다면, 그중에서 **(SNAPSHOT)**이 붙지 않은 가장 낮은 숫자를 선택하시면 됩니다.

하지만 만약 목록에 있는 모든 3.5 버전 뒤에 `(SNAPSHOT)`이 붙어 있다면, 차선책으로 그 바로 아래에 있는 **3.4.x (GA)** 버전을 선택하는 것이 가장 안전합니다.

---

### 1. 버전 선택 가이드 (목록에 보인다면)

1.  **최우선 선택:** `3.4.x` (뒤에 아무 글자도 없는 것. 예: 3.4.4)
    * 현재 실무와 가장 가깝고 모든 라이브러리가 안정적으로 돌아가는 버전입니다.
2.  **차선책:** `3.5.0` (뒤에 SNAPSHOT이 없는 정식 버전이 있다면)
3.  **피해야 할 것:** `3.5.x (SNAPSHOT)`
    * 이건 오늘 아침에 막 수정된 코드가 포함된 버전일 수 있어서, JPA 공부하다가 발생하는 에러가 내 코드 잘못인지 스프링 버그인지 알 수 없게 됩니다.

---

### 2. 게시판 프로젝트를 위한 Dependencies 최종 체크

버전을 고르셨다면 아래 4개는 꼭 추가해 주세요. (검색창에 치면 바로 나옵니다.)

* **Spring Data JPA**: DB를 자바 객체처럼 다루게 해주는 핵심 도구.
* **H2 Database**: 별도 설치 없이 메모리에서 바로 실행되는 연습용 DB.
* **Spring Web**: 게시판 API 만들 때 필수.
* **Lombok**: 자바의 지루한 코드(Getter/Setter)를 줄여주는 마법 같은 도구.



---

### 3. Java 버전 주의사항
혹시 내 컴퓨터에 깔린 Java 버전을 확인해 보셨나요?
스프링 부트 3.x 버전부터는 **Java 17 미만(Java 8, 11 등)은 절대 실행되지 않습니다.**

* 터미널(CMD)에서 `java -version`을 쳤을 때 숫자가 **17**이나 **21**로 시작하는지 확인해 보세요.
* 만약 Java 11 이하를 쓰고 계신다면, `start.spring.io`에서 프로젝트를 만들기 전에 Java 버전을 먼저 업데이트하거나, 사이트 하단 `Java` 선택 칸에서 본인의 버전에 맞게 설정해야 하지만... 사실 부트 3.x는 17이 최소 사양입니다.

---

### 4. 만약 정말 3.5.x(SNAPSHOT) 밖에 없다면?
그럴 리는 없겠지만, 만약 사이트 오류 등으로 SNAPSHOT만 보인다면 프로젝트를 생성한 뒤에 `build.gradle` 파일에서 버전을 살짝 강제로 낮추는 방법도 있습니다.

일단 **3.4.x** 대의 안정적인 버전이 목록에 있는지 다시 한번만 확인해 보시겠어요? 보통 SNAPSHOT 버전들 바로 아래에 안정화 버전(숫자만 있는 것)들이 모여 있습니다.

프로젝트 다운로드(GENERATE) 받으셨나요? 받으셨다면 인텔리제이에서 여는 단계로 넘어가 보겠습니다!
