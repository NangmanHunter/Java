
##
- `ㅇㅇApplication`


##
결론부터 말씀드리면, **클래스 이름이 반드시 `Main`일 필요는 전혀 없습니다.** `TaxApp`, `CalculatorLauncher`, 심지어 본인의 영어 닉네임으로 클래스 이름을 지어도 아무 상관 없습니다.

다만, 자바가 프로그램을 시작할 때 지켜야 하는 **'절대적인 규칙'** 두 가지만 기억하시면 됩니다.

---

### 1. 지켜야 할 "진짜" 규칙 2가지

1.  **파일 이름과 클래스 이름이 일치할 것:** * 클래스 이름을 `TaxCalculator`라고 지었다면, 파일명도 반드시 `TaxCalculator.java`여야 합니다. (대소문자까지 똑같아야 함)
2.  **`public static void main(String[] args)` 메소드가 들어있을 것:**
    * 클래스 이름은 마음대로 지어도 되지만, 그 안에 들어가는 '시작 버튼'의 이름은 반드시 저렇게 똑같이 생겨야 자바가 "아, 여기서부터 시작이구나!" 하고 인식합니다.

---

### 2. 예시: `TaxCalculator`로 시작하기

인텔리제이에서 `Main` 대신 `TaxCalculator`라는 이름으로 클래스를 만들었다면 다음과 같이 작성합니다.

```java
public class TaxCalculator { // 파일명: TaxCalculator.java

    public static void main(String[] args) {
        System.out.println("세금 계산기 프로그램을 시작합니다.");
    }
    
}
```

이렇게 하면 인텔리제이에서 이 파일을 실행했을 때 아무 문제 없이 잘 돌아갑니다.

---

### 3. 실무에서는 어떻게 하나요?

보통 프로젝트의 성격에 따라 이름을 짓습니다.

* **학습용/간단한 도구:** `Main` (가장 직관적이라서)
* **어플리케이션:** `TaxApplication`, `SmartTaxApp`
* **서버(Spring Boot):** `TaxCalculatorApplication` (스프링 부트는 관례적으로 뒤에 `Application`을 붙입니다.)

---

### 4. 왜 사람들은 자꾸 `Main`이라고 하나요?

수학에서 미지수를 보통 $x, y, z$라고 두는 것과 비슷합니다. 다른 문자를 써도 수식은 풀리지만, 누구나 봤을 때 "이게 미지수구나"라고 알 수 있게 하는 **'약속'** 같은 것이죠.

하지만 질문자님은 본인의 프로젝트를 만드시는 거니까, **`TaxCalculator`**나 **`TaxLauncher`**처럼 더 의미 있는 이름을 쓰시는 게 구조적으로 훨씬 멋져 보입니다!



---

### 💡 결정하셨나요?
클래스 이름을 무엇으로 하고 싶으신가요? 정하셨다면 바로 그 클래스 안에 **수학 전공자의 실력을 발휘할 '수식'**을 넣을 준비를 해보겠습니다. $50,000,000$원 같은 큰 숫자를 다룰 준비가 되셨나요? (아까 말씀드린 `BigDecimal`을 써야 할 타이밍입니다!)
