재민아, 백엔드 개발자 입장에서 뷰(Vue) 빌드는 자바의 **Gradle이나 Maven으로 `.jar` 파일을 만드는 과정**이랑 똑같다고 생각하면 돼.

다만, 뷰는 빌드 결과물이 실행 가능한 파일이 아니라 **브라우저가 읽을 수 있는 정적 파일(HTML, CSS, JS)**로 나온다는 게 차이점이야. 아주 빠르게 핵심만 짚어줄게.

---

### 1. 빌드 명령어 (터미널에서 실행)
요즘 뷰 프로젝트(Vite 기반)라면 프로젝트 루트 폴더에서 아래 명령어를 치면 돼.

```bash
# npm을 쓴다면
npm run build

# yarn을 쓴다면
yarn build
```

이 명령어를 실행하면 프로젝트 폴더 안에 **`dist/`** (또는 `build/`)라는 폴더가 생기고, 그 안에 모든 프론트엔드 파일이 압축되어서 들어가.

---

### 2. 빌드된 파일을 스프링 부트랑 합치기 (꿀팁)
재민이가 "가장 빠르게" 하고 싶다면, 뷰랑 스프링 서버를 따로 띄우는 게 아니라 **스프링 부트가 뷰 파일을 직접 서빙**하게 만드는 게 제일 편해.

1.  **Vue 빌드 결과물(`dist` 폴더 안의 내용)**을 복사해.
2.  **Spring Boot 프로젝트**의 `src/main/resources/static` 폴더에 붙여넣어.
3.  이제 스프링 서버(`localhost:8080`)만 띄우고 접속하면 뷰 화면이 바로 떠!



---

### 3. 더 편하게! Gradle로 자동화하기
매번 복사-붙여넣기 하기 귀찮지? 백엔드 개발자답게 `build.gradle`에 설정을 추가해서 **자바 빌드할 때 뷰 빌드도 같이 되도록** 만들 수 있어.

```groovy
// build.gradle 예시 (간략버전)
task buildVue(type: Exec) {
    workingDir "./frontend" // 뷰 프로젝트 폴더 위치
    commandLine "npm", "run", "build"
}

// 자바 빌드 전에 뷰 빌드부터 실행하고, 결과물을 static으로 복사하는 로직 추가
processResources.dependsOn(buildVue)
```
이렇게 해두면 `gradle build` 한 번으로 프론트와 백엔드가 합쳐진 하나의 `.jar` 파일이 완성돼.

---

### 🧐 요약하자면
* **명령어:** `npm run build`
* **결과물:** `dist/` 폴더 안의 정적 파일들
* **배포:** 그 파일들을 스프링의 `static` 폴더에 넣고 스프링 서버 실행!

백엔드 서버 하나만 관리하면 되니까 배포가 훨씬 쉬워질 거야. 빌드하다가 혹시 `npm`이 없다거나 빨간 글씨 에러 뜨면 바로 말해줘! 같이 해결해보자.
