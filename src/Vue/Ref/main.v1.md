### 1. 왜 그냥 `let`이나 `const`를 안 쓰나요?
일반적인 자바스크립트 변수는 값이 바뀌어도 화면이 눈치를 채지 못합니다.

```javascript
let count = 0;
count = 1; // 값은 바뀌었지만, 화면에 그려진 '0'은 그대로입니다.
```


### 3. 가장 중요한 주의점: `.value`
스크립트 영역(`setup`)에서 `ref` 변수의 값을 읽거나 수정할 때는 반드시 **`.value`**를 붙여야 합니다.

```javascript
const posts = ref([]) // 박스를 만듦

// 데이터를 넣을 때
posts.value = [{ title: '안녕' }] 

// 데이터를 읽을 때
console.log(posts.value)
```

하지만 **`<template>` 영역(HTML)**에서는 Vue가 알아서 박스를 까주기 때문에 `.value`를 안 써도 됩니다. (이게 아주 편하죠!)

---

### 4. 비유하자면 '박스'입니다
`ref`는 값을 그냥 두는 게 아니라, **특수한 박스**에 담아두는 것과 같습니다.

* **일반 변수**: 그냥 종이에 쓴 숫자. (바꿔 써도 아무도 모름)
* **`ref`**: **전광판과 연결된 매직 박스**. (박스 안의 내용물을 바꾸면 전광판 숫자가 즉시 바뀜)



### 💡 왜 게시판 목록에 `ref([])`를 쓸까요?
처음 페이지가 떴을 때는 게시글이 없으니 빈 리스트(`[]`)로 시작했다가, 서버에서 데이터를 받아오면 `posts.value = 서버데이터` 코드를 실행해 화면에 게시글이 **'짜잔'** 하고 나타나게 하기 위해서입니다.

이제 코드에서 `ref`를 보시면 "아, 이건 화면이랑 연결된 감시 대상이구나!"라고 생각하시면 됩니다. :)
