
가장 근본적인 차이는 **"Vue가 그 변수를 지켜보고 있느냐(감시)"**와 **"화면을 새로 그리느냐(반응)"**에 있습니다.




### 1. 일반 변수 (Plain JavaScript Variable)
일반 변수는 메모리에 값을 저장할 뿐, 그 값이 바뀌었다고 해서 누구에게도 알리지 않습니다.

*   **동작:** 값이 바뀌어도 화면(DOM)은 그대로입니다.
*   **용도:** 계산용 임시 변수, 로직 내부에서만 쓰이는 플래그, 화면에 보여줄 필요가 없는 데이터.

```javascript
let count = 0;

const increment = () => {
  count++; // 값은 1로 바뀌지만, 화면의 글자는 여전히 0입니다.
  console.log(count); // 콘솔에는 1이 찍힙니다.
};
```

---

### 2. ref 변수 (Reactive Reference)
`ref`는 값을 **'반응형 객체'**라는 특별한 박스에 넣은 것입니다. Vue는 이 박스를 24시간 감시합니다.

*   **동작:** `.value`가 바뀌는 순간, Vue가 "어? 값이 바뀌었네? 이 변수를 쓰고 있는 화면 부분을 다시 그려!"라고 명령합니다. (이것을 **Reactivity**, 반응성이라고 합니다.)
*   **용도:** 화면에 표시되는 데이터, 입력 폼의 값, API로 받아온 목록 등.

```javascript
const count = ref(0);

const increment = () => {
  count.value++; // 이 순간 Vue가 감지하고 화면의 숫자도 1로 바꿉니다.
};
```

---

### 3. 결정적인 차이점 비교

| 구분 | 일반 변수 (`let`, `const`) | `ref` 변수 |
| :--- | :--- | :--- |
| **반응성** | 없음 (값이 변해도 UI 고정) | **있음 (값 변하면 UI 즉시 갱신)** |
| **접근 방식** | `count` | `count.value` (스크립트 기준) |
| **템플릿 사용** | `{{ count }}` | `{{ count }}` (자동 언래핑 되어 편리함) |
| **성능** | 아주 미세하게 빠름 | 감시 로직 때문에 미세한 오버헤드 있음 |

---

### 4. 왜 `.value`를 써야 할까?
이게 가장 귀찮고 헷갈리는 부분이죠? JavaScript 언어 자체에는 "변수 값이 바뀔 때 특정 함수를 실행해라"라는 기능이 없습니다. 

그래서 Vue는 **객체(Object)**를 하나 만들고, 그 안에 `value`라는 속성을 만든 뒤에 그 속성을 **Proxy(대리인)**라는 기술로 감싸버린 겁니다. 

*   **일반 변수:** 그냥 종이에 쓴 숫자. (숫자를 고쳐 써도 종이는 가만히 있음)
*   **ref 변수:** 숫자가 적힌 전광판. (리모컨(`.value`)으로 숫자를 바꾸면 전광판 전체에 불이 들어오며 바뀜)

### 5. 실전 팁





아까 물어보신 `UTable`의 `ref="table"`도 마찬가지입니다. `const table = ref(null)`로 선언하는 이유는, 나중에 실제 테이블 객체가 들어왔을 때 Vue가 그걸 인지하고 우리에게 알려줘야 하기 때문입니다. 일반 변수로 선언하면 테이블이 로드되어도 변수에 담기기만 할 뿐, 우리가 그 타이밍을 잡기가 매우 어렵습니다.
