네, 맞습니다. `Button-SignUp.vue`처럼 하이픈을 섞거나 너무 구체적인 기능을 파일명에 박아두면 나중에 관리하기가 꽤 까다로워집니다.

Vue와 Nuxt 커뮤니티에서 통용되는 **컴포넌트 네이밍 컨벤션**을 정리해 드릴게요.

---

### 1. 파스칼 케이스 (PascalCase) 사용
Vue 공식 스타일 가이드에서는 컴포넌트 파일명을 **PascalCase**로 작성할 것을 강력히 권장합니다.
* ✅ `SignUpButton.vue`
* ❌ `sign-up-button.vue` / `signupbutton.vue`

### 2. "기본 단어"를 앞에 두기 (Prefix)
컴포넌트를 찾기 쉽게 하려면 **가장 일반적인 범주(Base/Common)**의 단어를 앞에 두는 것이 좋습니다. 버튼이라면 `BaseButton`이나 `AppButton`을 먼저 만들고, 특정 기능을 하는 버튼은 그 뒤에 서술어를 붙입니다.

* `BaseButton.vue` (가장 기본이 되는 공통 버튼)
* `ButtonSignUp.vue` (보다는 `SignUpButton.vue`가 더 자연스럽지만, 파일 시스템에서 버튼끼리 모아보고 싶다면 `BtnSignUp.vue` 식으로 통일하기도 합니다.)

### 3. 추천하는 구조 (디렉토리 활용)
파일 이름에 모든 정보를 다 넣기보다는 **폴더 구조**를 활용하는 것이 훨씬 깔끔합니다.

* **방법 A: 기능별 분류 (추천)**
    `components/auth/SignUpButton.vue`
    `components/common/BaseButton.vue`

* **방법 B: 요소별 분류**
    `components/ui/Button.vue` (Nuxt UI가 이미 `UButton`을 쓰므로, 이걸 래핑한다면 `AppButton.vue`)

### 4. Nuxt UI를 래핑할 때 주의할 점
이미 Nuxt UI에 `UButton`이 있기 때문에, 본인이 직접 만드는 버튼 컴포넌트 이름이 `Button.vue`라면 나중에 `UButton`과 혼동될 수 있습니다. 그래서 보통 프로젝트 이름을 접두어로 붙이거나 `App`을 붙입니다.

* 예: `AppButton.vue`, `MyButton.vue`

---

### 💡 실무에서 많이 쓰는 네이밍 패턴

| 유형 | 예시 | 비고 |
| :--- | :--- | :--- |
| **기본 공통 컴포넌트** | `AppButton.vue`, `BaseInput.vue` | 어디서나 쓰이는 기초 UI |
| **특정 페이지용** | `LoginSubmitButton.vue` | 특정 맥락에서만 쓰임 |
| **아이콘 포함 버튼** | `IconButton.vue` | 형태에 따른 분류 |

### 정리하자면
질문하신 `Button-SignUp.vue` 보다는 **`SignUpButton.vue`** 혹은 폴더를 나눠서 **`auth/SignUpButton.vue`**로 관리하시는 것이 훨씬 "Vue스러운" 방식입니다. 

개발 효율을 중시하신다면, `UButton`의 속성을 그대로 상속받는 `AppButton.vue` 하나를 잘 만들어두고 재사용하는 방식을 추천드려요!
