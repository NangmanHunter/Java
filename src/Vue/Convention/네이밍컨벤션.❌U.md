아, 그건 Nuxt의 **자동 이름 생성 규칙** 때문입니다! 원인을 바로 짚어드릴게요.

### 원인: 이름이 중복되었습니다.

Nuxt는 폴더 구조를 기반으로 이름을 만들 때, **폴더명 + 파일명**을 합칩니다. 
현재 구조가 `components/auth/SignUpButton.vue`라면:

1.  폴더명: `auth`
2.  파일명: `SignUpButton`
3.  최종 컴포넌트 이름: **`AuthSignUpButton`** (U가 붙지 않음)

그런데 지금 `<UAuthSignUpButton />`이라고 쓰셨죠? 맨 앞의 **`U`**는 Nuxt UI 라이브러리에서 제공하는 컴포넌트들에만 붙는 접두어입니다. 직접 만드신 컴포넌트에는 `U`가 자동으로 붙지 않습니다.

---

### 해결 방법 1: 이름 수정 (가장 빠름)
`index.vue`에서 `U`를 빼고 호출해보세요.
```vue
<template>
  <AuthSignUpButton />
</template>
```

### 해결 방법 2: 파일명 수정
만약 Nuxt UI 버튼을 래핑했다는 의미로 꼭 `U`를 붙이고 싶다면, 파일명을 바꾸시면 됩니다.
* 경로: `components/auth/USignUpButton.vue`
* 사용: `<AuthUSignUpButton />`

### 해결 방법 3: 이름 확인 팁
가끔 이름이 헷갈릴 때는 **Nuxt DevTools**를 활용하면 한 번에 해결됩니다.
1. 브라우저 하단의 Nuxt 아이콘을 클릭해 **DevTools**를 엽니다.
2. **Components** 탭에 들어갑니다.
3. 내가 만든 파일이 어떤 이름으로 등록되어 있는지 리스트에서 바로 확인할 수 있습니다.

---

### 💡 추가 체크포인트
만약 `U`를 뺐는데도 안 나온다면 아래 두 가지만 더 확인해 보세요:

* **서버 재시작:** 간혹 폴더를 새로 만들었을 때 Nuxt가 실시간으로 감지 못할 때가 있습니다. 터미널에서 `npm run dev`를 껐다 켜보세요.
* **컴포넌트 폴더 설정:** `nuxt.config.ts`에서 `components: false` 같은 설정을 따로 하신 적이 없다면 기본적으로 `components/` 폴더는 자동 스캔됩니다.

이제 `<AuthSignUpButton />`으로 바꿔서 한번 시도해 보시겠어요? 바로 뜰 겁니다!
