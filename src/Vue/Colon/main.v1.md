##
속성 앞에 붙는 콜론(`:`)은 **Vue.js에서 제공하는 `v-bind`라는 문법의 단축 표기(Shorthand)**입니다.

Nuxt UI 역시 Vue.js 기반이기 때문에 이 문법을 그대로 사용합니다.

### 1. `:`의 정체: `v-bind`
원래 정식 문법은 `v-bind:속성명="값"`입니다. 하지만 매번 쓰기 번거롭기 때문에 앞의 `v-bind`를 생략하고 `:`만 남겨서 쓰기로 약속한 것입니다.

* **Full Syntax:** `<UCheckbox v-bind:default-value="true" />`
* **Shorthand:** `<UCheckbox :default-value="true" />`

---

### 2. 왜 이걸 써야 하나요?
HTML의 기본 속성과 자바스크립트 데이터를 구분하기 위해서입니다.

| 문법 | 데이터 해석 방식 | 결과 타입 |
| :--- | :--- | :--- |
| `value="true"` | 따옴표 안을 **단순 문자열**로 처리 | `String` ("true") |
| **`:value="true"`** | 따옴표 안을 **자바스크립트 코드**로 처리 | **`Boolean`** (true) |

---

### 3. 다른 곳에서는 어떻게 쓰이나요?
단순히 `true/false`뿐만 아니라, 스크립트 단에서 정의한 **변수나 함수, 숫자**를 넣을 때도 무조건 사용합니다.

```vue
<script setup>
const count = 10
const isDisabled = false
const themeColor = 'red'
</script>

<template>
  <UInput :maxlength="count" />

  <UButton :disabled="isDisabled">클릭</UButton>

  <div :style="{ color: themeColor }">색상이 변합니다.</div>
</template>
```

### 요약하자면
지금 `UCheckbox`의 명세에서 요구하는 `defaultValue`는 **논리값(Boolean)** 형태여야 합니다. 그런데 콜론 없이 `default-value="true"`라고 쓰면 컴퓨터는 이걸 "true"라는 이름의 **글자**로 인식해서 무시해버리는 것이죠.

**"콜론(`:`)은 따옴표 안을 자바스크립트로 실행해라!"** 라는 신호라고 생각하시면 가장 정확합니다.

혹시 지금 개발 중인 프로젝트의 `package.json`에서 `@nuxt/ui` 버전을 한 번만 더 확인해 보시겠어요? 4.7이라는 버전이 생소해서 정확한 가이드를 드리고 싶네요!




#


상식적으로 생각하면 `"true"`라는 글자가 당연히 논리값 `true`로 해석될 것 같지만, **웹 표준(HTML)과 Vue(프레임워크)의 동작 방식** 때문에 엄격하게 구분됩니다.

이걸 이해하면 나중에 다른 라이브러리를 쓸 때도 삽질을 확 줄일 수 있습니다. 핵심 이유는 크게 두 가지입니다.

---

### 1. HTML은 기본적으로 모든 속성을 '문자열'로 취급합니다
HTML 태그에 적는 모든 값은 일단 **텍스트**입니다.

* `<input type="checkbox" value="true">` 라고 적으면, 브라우저는 이걸 보고 "아, 이 체크박스의 값은 `t-r-u-e`라는 네 글자구나"라고 생각합니다.
* 만약 라이브러리 내부 코드가 `if (props.defaultValue === true)` 처럼 **엄격한 비교(`===`)**를 하고 있다면, 문자열 `"true"`는 논리값 `true`와 다르기 때문에 조건문을 통과하지 못하고 무시됩니다.

### 2. Vue의 속성 상속 (Attribute Inheritance)
Vue 컴포넌트에서 속성을 전달할 때 `:`를 붙이느냐 안 붙이느냐에 따라 처리 결과가 완전히 달라집니다.

* **콜론 없이 (`default-value="true"`)**:
  Vue는 이걸 **"정적 속성(Static Attribute)"**으로 봅니다. 자바스크립트 엔진을 거치지 않고 그냥 `"true"`라는 문자열을 컴포넌트에게 던져줍니다.
* **콜론과 함께 (`:default-value="true"`)**:
  따옴표 안쪽을 **자바스크립트 영역**으로 선언하는 것입니다. 자바스크립트에서 `true`는 예약어이므로, 엔진이 이를 실행하여 **논리값(Boolean)** 타입으로 변환한 뒤 컴포넌트에게 전달합니다.

---

### 3. 타입 선언이 되어 있는데 왜 못 잡을까?
Nuxt UI(또는 Radix UI) 내부 코드는 대략 이렇게 설계되어 있을 겁니다.
```typescript
// 내부 코드 예시 (TypeScript)
interface CheckboxProps {
  defaultValue?: boolean; // 여기서 타입을 boolean으로 딱 박아둠
}
```
컴포넌트 내부에서 `defaultValue`가 들어왔을 때, 타입이 `boolean`이 아니면(즉, 문자열이 들어오면) **에러를 내거나 무시하도록** 설계되어 있습니다. 

> **왜 자동으로 변환 안 해주나요?**
> 만약 `"false"`라는 문자열이 들어왔다고 가정해 봅시다. 자바스크립트에서 **비어있지 않은 문자열은 논리적으로 `true`**로 취급됩니다(`Boolean("false")`는 `true`입니다). 이런 혼란을 막기 위해 현대적인 라이브러리들은 타입을 아주 엄격하게 체크합니다.
