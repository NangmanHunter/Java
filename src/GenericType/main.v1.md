##
- 제네릭타입
  - 제네릭인터페이스
  - 제네릭클래스


##
엄격하게 따지면 **제네릭 타입(Generic Type)**과 **제네릭 클래스(Generic Class)**는 완전히 같은 의미는 아니지만, 맥락에 따라 혼용해서 쓰기도 합니다. 

이해를 돕기 위해 두 개념의 차이점과 관계를 정리해 드릴게요.

---

### 1. 포함 관계
결론부터 말씀드리면, **제네릭 타입**이 더 큰 범주입니다. 

* **제네릭 타입:** 클래스와 인터페이스를 모두 포함하는 용어입니다.
* **제네릭 클래스:** 제네릭 타입 중 '클래스'로 선언된 것을 구체적으로 지칭하는 용어입니다.

즉, 모든 제네릭 클래스는 제네릭 타입이지만, 모든 제네릭 타입이 제네릭 클래스인 것은 아닙니다 (인터페이스일 수도 있으니까요).

### 2. 정확한 정의

| 용어 | 정의 | 예시 (Java/Spring 기준) |
| :--- | :--- | :--- |
| **제네릭 타입** | 타입을 파라미터로 가지는 클래스와 인터페이스를 통칭 | `List<E>`, `Map<K, V>`, `ResponseEntity<T>` |
| **제네릭 클래스** | 선언부에 타입 파라미터가 들어간 클래스 | `class Box<T> { ... }` |
| **제네릭 인터페이스** | 선언부에 타입 파라미터가 들어간 인터페이스 | `interface Repository<T, ID> { ... }` |



---

### 3. 실무에서는 어떻게 말하나?

대화할 때 "제네릭 클래스"라고 말해도 대개는 소통에 문제가 없습니다. 다만, 다음과 같은 상황에서는 구분이 필요할 수 있습니다.

* **인터페이스를 다룰 때:** 만약 `List`나 `Map` 같은 인터페이스를 보며 "이 제네릭 클래스는~"이라고 하면 엄밀히 말해 틀린 표현이 됩니다. 이때는 **"제네릭 타입"** 혹은 **"제네릭 인터페이스"**라고 부르는 것이 정확합니다.
* **학습이나 면접 시:** "제네릭 타입은 제네릭 클래스와 제네릭 인터페이스를 포함한다"는 개념을 알고 있는 것이 좋습니다.

---

### 요약
일상적인 개발 대화에서는 **"제네릭 클래스"**라고 해도 무방하지만, 대상이 인터페이스일 수도 있다는 점을 고려하면 **"제네릭 타입"**이라고 부르는 것이 가장 안전하고 정확한 표현입니다.

지금 만들고 계신 게시판 모듈화 프로젝트(`bitcopark-base`)에서 공통 응답 객체나 베이스 클래스를 설계 중이시라면, 클래스 형태일 경우가 많으니 "제네릭 클래스"라고 불러도 큰 무리는 없을 것 같네요!
