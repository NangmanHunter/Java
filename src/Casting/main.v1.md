

##
자바의 **업캐스팅(Upcasting)**과 **다운캐스팅(Downcasting)**은 상속 관계에 있는 클래스들 사이에서 객체의 타입을 변환하는 것을 말해요. 쉽게 말해 "부모 노릇 하기"와 "다시 본래의 나로 돌아오기"라고 이해하면 편합니다.

---

## 1. 업캐스팅 (Upcasting)
자식 클래스의 객체가 부모 클래스 타입으로 변환되는 것을 말합니다. 

* **특징:** 별도의 형변환 연산자가 필요 없으며(자동 형변환), 항상 안전합니다.
* **이유:** 자식은 부모의 유전자를 모두 물려받았기 때문에, 부모 타입인 척해도 문제가 없기 때문이죠.
* **제한:** 부모 타입으로 변환되면, 자식 클래스에만 있는 독자적인 메서드는 사용할 수 없게 됩니다.

```java
Parent p = new Child(); // 업캐스팅 (자동)
```



---

## 2. 다운캐스팅 (Downcasting)
업캐스팅된 객체를 **다시 원래의 자식 클래스 타입**으로 되돌리는 것을 말합니다.

* **특징:** 반드시 명시적으로 형변환(`(타입)`)을 해줘야 합니다.
* **주의사항:** 원래 그 객체가 해당 자식 타입이 아니었는데 억지로 바꾸려 하면 오류(`ClassCastException`)가 발생합니다. 그래서 보통 `instanceof` 연산자로 확인 후 진행합니다.

```java
Child c = (Child) p; // 다운캐스팅 (명시적)
```

---

## 3. 한눈에 비교하기

| 구분 | 업캐스팅 (Upcasting) | 다운캐스팅 (Downcasting) |
| :--- | :--- | :--- |
| **방향** | 자식 → 부모 | 부모 → 자식 |
| **형변환** | 자동 (생략 가능) | 명시적 (생략 불가) |
| **안전성** | 항상 안전함 | 위험할 수 있음 (확인 필요) |
| **목적** | 다형성 (여러 객체를 하나로 관리) | 자식의 고유 기능을 다시 쓰기 위해 |

---

### 💡 왜 쓸까? (핵심 이유)
가장 큰 이유는 **다형성** 때문이에요. 
예를 들어 `개`, `고양이`, `사자` 클래스가 모두 `동물` 클래스를 상속받았다면, `동물[] 배열` 하나에 이들을 모두 담아서 관리할 수 있죠. (업캐스팅) 
그러다가 특정 동물의 고유 기능(예: 개의 `꼬리 흔들기`)이 필요할 때만 잠시 원래 타입으로 바꿔서 사용하는 겁니다. (다운캐스팅)
