- 점원 (API): 손님(클라이언트)이 주방(서버)에 직접 들어가서 요리할 수 없으니, 중간에서 주문을 받아 주방에 전달하고 음식을 가져다주는 존재입니다.
- 메뉴판 (API 명세서): 손님이 주문할 수 있는 목록과 방법이 적혀 있습니다. "1번 메뉴는 비빔밥이고 10,000원입니다"라고 약속된 규칙이죠.
- REST API: "메뉴판을 작성할 때 전 세계적으로 통용되는 아주 깔끔하고 직관적인 서술 방식을 사용한 메뉴판"이라고 볼 수 있습니다.
- REST API는 웹 환경에서 가장 널리 쓰이는 깔끔한 API 설계 스타일 중 하나다.
- 결국 "API를 만든다"는 것은 외부에서 우리 서버의 데이터를 안전하고 편리하게 가져갈 수 있도록 '창구를 개설한다'는 의미로 생각하시면 됩니다.
