맞아요, **JShell**은 사실 '자바 문법 공부'보다는 **'이 API가 어떻게 동작하나?'**를 빠르게 찔러보는 테스트 베드에 가까워요. 클래스도 없고 메소드도 없이 그냥 줄 단위로 실행되니까 자바 특유의 구조적인 느낌이 싹 빠져버리죠.

만약 **"자바의 구조(메소드, 클래스 등)는 유지하면서도, 빌드 설정 없이 가볍게 파일 하나로 스크립트처럼 쓰고 싶다"**는 거라면, 아래 방식이 가장 입맛에 맞으실 거예요.

---

### 1. Java 11+ 단일 파일 실행 (가장 추천)
이 방식은 **완전한 자바 문법**을 그대로 쓰면서, `javac`로 컴파일해서 `.class` 파일을 만드는 번거로운 과정만 생략하는 방식입니다.

**`CheckStatus.java`**
```java
import java.util.List;

public class CheckStatus {
    public static void main(String[] args) {
        var items = List.of("Apple", "Banana", "Cherry");
        
        for (var item : items) {
            System.out.println("Item: " + item);
        }
    }
}
```
* **실행:** 터미널에서 `java CheckStatus.java`
* **장점:** * 우리가 아는 그 자바 문법 그대로입니다.
    * 클래스, 메소드 구조가 다 살아있어서 나중에 실제 프로젝트 코드로 옮기기 매우 쉽습니다.

---

### 2. Java 21+ "Unnamed Classes" (최신 트렌드)
"클래스 이름 붙이고 `public static void main` 치는 게 너무 길다!" 하시는 분들을 위해 나온 방식입니다. 자바의 정체성은 유지하되 껍데기만 깎아냈습니다.

**`MyScript.java`**
```java
// 클래스 선언 없음! (하지만 내부적으로는 클래스로 취급됨)
void main() {
    String message = "이건 진짜 자바 문법이죠?";
    sayHello(message);
}

void sayHello(String msg) {
    System.out.println(msg);
}
```
* **실행:** `java --source 21 --enable-preview MyScript.java`
* **특징:** * **메소드 호출, 변수 타입 지정** 등 자바 고유의 규칙은 엄격하게 지킵니다.
    * 단순히 `main` 메소드를 감싸는 불필요한 보일러플레이트만 제거한 형태입니다.

### 결론: 무엇을 쓰면 좋을까요?

| 특징 | JShell | 단일 파일 실행 (추천) |
| :--- | :--- | :--- |
| **문법 엄격도** | 낮음 (세미콜론 생략 등) | **높음 (표준 자바와 동일)** |
| **코드 구조** | 파편화된 코드 조각 | **메소드/클래스 구조 유지** |
| **용도** | 한 줄 단위 빠른 확인 | **작은 유틸리티/스크립트 작성** |

JShell이 너무 가벼워서 장난감 같으셨다면, **1번(Java 11+) 방식**으로 파일 하나에 코드를 짜서 `java 파일명.java`로 실행해 보세요. 자바의 손맛은 그대로면서 실행은 스크립트처럼 빠릅니다.
