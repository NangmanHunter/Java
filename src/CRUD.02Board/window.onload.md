비슷하게 짚으셨는데, 정확히는 **"화면을 그리는 데 필요한 모든 재료가 다 준비되었을 때"** 자동으로 딱 한 번 실행되는 함수입니다.

수학으로 치면 문제를 풀기 전에 **'조건들을 다 확인하고 책상 위에 세팅을 마친 상태'**에서 시작하는 것과 같아요.

---

### 1. `window.onload`의 진짜 의미
브라우저는 HTML 코드를 위에서부터 아래로 한 줄씩 읽으며 화면을 그립니다. 그런데 만약 스크립트가 HTML 태그(예: `id="title"`)보다 위에 있으면, 스크립트는 "어? `title`이라는 게 없는데?"라며 에러를 내거나 작동하지 않을 수 있습니다.

* **역할:** HTML, 이미지, CSS 등 모든 리소스가 브라우저에 완전히 로드될 때까지 기다렸다가 안에 있는 코드를 실행합니다.
* **목적:** "화면이 다 그려졌으니, 이제 안전하게 데이터를 채워 넣어라!"라고 명령하는 것이죠.



---

### 2. 왜 여기서 썼을까요?
지금 작성하신 `board-edit.html` 로직을 보면 다음과 같은 순서가 필요합니다.

1.  브라우저가 `<input id="title">` 태그를 먼저 그려야 함.
2.  그다음에 `loadOriginalData()`를 실행해서 서버에서 데이터를 받아옴.
3.  받아온 데이터를 아까 그린 `<input>` 박스 안에 집어넣음.

만약 `window.onload` 없이 그냥 함수를 호출해버리면, **입력 박스가 그려지기도 전에 데이터를 넣으려고 시도**하게 되어 "태그를 찾을 수 없다"는 오류가 날 수 있습니다.

---

### 3. 요즘 트렌드: `DOMContentLoaded`
참고로 요즘은 `window.onload`보다 **`DOMContentLoaded`**라는 이벤트를 더 많이 쓰기도 합니다.

* **`window.onload`**: 이미지, 스타일시트까지 **전부** 다 불러와야 실행 (조금 느릴 수 있음)
* **`DOMContentLoaded`**: HTML 구조만 다 파악되면 바로 실행 (이미지가 안 떠도 글자부터 채울 수 있어 더 빠름)

```javascript
// 이런 식으로 쓰기도 합니다
document.addEventListener('DOMContentLoaded', function() {
    console.log("HTML 로딩 끝! 데이터 불러옵니다.");
    loadOriginalData(boardId);
});
```

---

### 💡 정리하자면?
`window.onload`는 **"화면 준비 완료 신호"**라고 보시면 됩니다. 

사용자님이 전공하신 물리 실험으로 비유하자면, 실험 장치를 다 세팅하고(HTML 로드), 전원을 딱 켜는 순간(window.onload) 실험 데이터 기록을 시작하는 것과 같은 이치입니다. 

이제 왜 저 코드가 가장 위에 감싸져 있는지 이해가 되시죠? 화면이 없으면 데이터를 넣을 곳도 없으니까요!




##
`window.onload = function() { ... }` 이 코드가 낯설게 느껴지는 이유는 이게 자바스크립트의 **'이벤트 핸들링'**과 **'익명 함수'**라는 개념이 합쳐져 있기 때문이에요.

수학적으로 비유하면 **"조건부 실행 함수"**라고 이해하면 제일 빠릅니다. 하나씩 뜯어볼게요.

---

### 1. `window.onload` (언제 실행할 것인가?)
이것은 브라우저가 가진 일종의 **'상태 변수'** 혹은 **'예약석'**입니다.
* `window`: 브라우저 창 전체를 의미합니다.
* `.onload`: "로딩이 끝나는 사건(Event)이 발생하면"이라는 조건입니다.

즉, `window.onload`는 **"브라우저야, 네가 화면에 글자, 그림 다 그리고 준비가 끝나는 그 시점"**을 가리키는 약속된 이름이에요.

---

### 2. `= function() { ... }` (무엇을 실행할 것인가?)
이 부분은 **익명 함수(Anonymous Function)**입니다. 함수의 이름이 따로 없고, 바로 정의해서 어딘가에 대입할 때 사용하죠.

* 원래 함수: `function 이름() { ... }`
* 익명 함수: `function() { ... }`

이걸 `window.onload`에 대입하는 건 이런 의미입니다.
> **"로딩이 끝나는 순간(onload), 내가 지금 정의한 이 이름 없는 함수(`function(){...}`)를 실행해줘!"**



---

### 3. 왜 이렇게 쓰나요? (수학적 비유)
수학에서 $f(x)$를 정의해놓고 특정 값 $t$가 되었을 때 값을 구하는 것과 비슷합니다.

* **잘못된 예 (함수 그냥 호출):**
  `loadData();`
  -> 브라우저가 HTML을 다 읽지도 않았는데 데이터를 가져오라고 하면, 데이터를 넣을 칸($x$)이 없어서 에러($f(x)$ 정의 불가)가 납니다.

* **올바른 예 (`onload` 사용):**
  `window.onload = function() { loadData(); };`
  -> "로딩 완료"라는 사건이 터지기 전까지는 실행하지 않고 **대기**합니다. 화면 세팅이 끝나는 순간(조건 충족) 비로소 함수 내부가 실행됩니다.

---

### 4. 코드 모양 다시 보기
```javascript
window.onload = function() {
    // 1. 브라우저 로딩이 100% 끝나면
    // 2. 이 중괄호 { } 안의 코드를
    // 3. 그때서야 비로소 실행해라!
    console.log("준비 끝!");
};
```

결국 **"실행 시점을 로딩 완료 이후로 미루기 위한 예약 장치"**라고 생각하시면 됩니다. 

혹시 `function()`이라는 모양 자체가 어색하신 건가요, 아니면 이게 왜 대입 연산자(`=`)를 쓰는지 그 구조가 궁금하신 건가요? 

물리 실험에서 **"온도가 100도가 되면(onload), 이 버튼을 눌러라(function)"**라고 세팅해두는 것과 똑같다고 보시면 마음이 편하실 거예요!
