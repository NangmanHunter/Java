
##

- Enum.FactroyMethod
- Enum.StaticFactroyMethod


##
이 코드는 기술적으로 **'정적 팩토리 메서드(Static Factory Method)'**라고 부릅니다. 

하지만 `Enum`에서 특정 용도로 쓰일 때는 더 구체적인 이름들로 불리기도 해요. 상황에 따라 부르는 명칭을 정리해 드릴게요.

---

### 1. 기술적 명칭: 정적 팩토리 메서드 (Static Factory Method)
가장 정확한 표현입니다. 객체(여기서는 Enum 상수)를 생성하거나 찾아주는 역할을 하는 `static` 메서드를 통칭합니다. 
* "Enum에 **정적 팩토리 메서드**를 만들어서 대소문자 처리를 했어."라고 말하면 됩니다.

### 2. 역할 기반 명칭: 역직렬화 메서드 (Deserialization Method)
`@JsonCreator`가 붙어 있기 때문에 생기는 이름입니다. 
* **직렬화(Serialization):** Java 객체 → JSON 문자열 (나갈 때)
* **역직렬화(Deserialization):** JSON 문자열 → Java 객체 (들어올 때)
* "JSON 데이터를 Enum으로 변환할 때 쓰는 **역직렬화 로직**이야."라고 표현합니다.

### 3. 관례적 명칭: From 메서드 (of / from)
Java의 새로운 관례(Effective Java 등)에 따르면, 매개변수를 받아서 해당 타입의 인스턴스를 반환하는 메서드 이름을 `from`이나 `of`로 짓습니다.
* "BoardType에 **from 메서드** 하나 추가했어."라고 편하게 부르기도 합니다.

---

### 💡 왜 '생성자'라고 부르지 않나요?

`Enum`은 태생적으로 **생성자가 private**으로 고정되어 있습니다. 외부에서 `new BoardType()`을 할 수 없죠. 

우리가 만든 `from` 메서드는 새로운 객체를 '생성'하는 게 아니라, 이미 정의된 `NOTICE`나 `COMMUNITY` 중 하나를 **'찾아서 반환'**하는 것이기 때문에 **생성자(Constructor)**라는 표현보다는 **팩토리(Factory)**라는 표현이 더 적절합니다.

---

### 🛠️ 코드의 의미 요약
이 코드를 한 문장으로 설명한다면 이렇게 말해보세요:

> "입력받은 문자열을 **대문자로 변환**한 뒤, 그에 맞는 **Enum 상수를 매핑**해주는 **정적 팩토리 메서드**야."

이제 누군가 "이 코드 뭐야?"라고 물어보면, "어, 이거 Enum **역직렬화**할 때 대소문자 구분 안 하려고 만든 **팩토리 메서드**야."라고 멋지게 대답하시면 됩니다! 

이 표현들 중에서 어떤 게 가장 입에 붙으시나요? 확실히 '생성자'보다는 '팩토리 메서드'가 개발자들 사이에서 소통할 때 더 정확한 느낌을 줍니다. :)
