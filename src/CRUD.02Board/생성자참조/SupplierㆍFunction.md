자바 8에서 람다와 스트림이 도입되면서 함께 등장한 **표준 함수형 인터페이스**들입니다. 이름이 어려워 보이지만, 실상은 **"자주 쓰는 함수 패턴을 미리 이름 붙여서 만들어둔 그릇"**일 뿐입니다.

가장 대표적인 4인방 중에서 궁금해하신 `Supplier`와 `Function`을 중심으로 설명해 드릴게요.

---

### 1. Supplier (공급자)
**"인자는 없는데, 결과물만 툭 내놓는"** 녀석입니다. 마치 자판기 버튼을 누르면(호출하면) 물건이 나오는 것과 같습니다.

* **구조:** `() -> T`
* **특징:** 매개변수가 없고, 리턴값만 있습니다.
* **핵심 메서드:** `get()`

```java
// String을 공급해주는 공급자 정의
Supplier<String> s = () -> "Hello World";

// 실제로 꺼낼 때 get() 사용
String result = s.get(); 
```

### 2. Function (함수)
가장 전형적인 함수의 형태입니다. **"무언가를 넣으면(Input), 다른 무언가로 변환해서 내놓는(Output)"** 녀석입니다.

* **구조:** `T -> R`
* **특징:** 매개변수 하나를 받아서 처리한 뒤 결과를 돌려줍니다.
* **핵심 메서드:** `apply(T t)`

```java
// String을 넣으면 Integer로 바꿔주는(길이를 재는) 함수 정의
Function<String, Integer> f = (str) -> str.length();

// 실행할 때 apply() 사용
Integer length = f.apply("Nangman"); // 7 반환
```

---

### 3. 왜 이런 이름들을 만들어 놨을까?

컴파일러가 `::new` 같은 생성자 참조나 람다식을 해석할 때, **"이 함수는 입력을 몇 개 받고 출력이 뭐지?"**를 판단할 기준이 필요하기 때문입니다.

| 이름 | 입력을 받는가? | 출력을 하는가? | 실제 비유 |
| :--- | :---: | :---: | :--- |
| **Supplier** | X | **O** | **생산자:** 재고(객체)를 새로 찍어내서 줌 |
| **Consumer** | **O** | X | **소비자:** 데이터를 받아서 출력하거나 쓰고 끝냄 |
| **Function** | **O** | **O** | **변환기:** A를 넣으면 B로 바꿔서 줌 |
| **Predicate** | **O** | **O (boolean)** | **판독기:** 맞는지 틀린지 검사함 |



---

### 4. 아까 질문하신 `::new`와 연결해보기

이제 왜 아까 `Supplier`와 `Function`에 따라 `::new`가 다르게 동작했는지 이해되실 거예요!

* `Supplier<User> s = User::new;`
    * `Supplier`는 입력이 없네? → 그럼 `new User()` (기본 생성자) 호출해야지!
* `Function<String, User> f = User::new;`
    * `Function`은 입력이 `String` 하나 있네? → 그럼 `new User(String s)` (파라미터 하나 있는 생성자) 호출해야지!

결국 이 인터페이스들은 **자바가 람다식을 어떤 메서드로 변환할지 알려주는 "타입 가이드라인"** 역할을 한다고 보시면 됩니다.

이 중에서 혹시 실제 프로젝트에서 데이터 변환할 때(예: Entity를 DTO로 바꿀 때) 어떤 걸 써야 할지 감이 좀 오시나요? 보통은 `Function`이 제일 많이 쓰이긴 합니다!
