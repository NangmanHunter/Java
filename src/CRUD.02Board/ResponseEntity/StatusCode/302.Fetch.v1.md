##
네, 정확하게 짚으셨습니다! 원인은 바로 **`fetch` (또는 `useFetch`)를 이용한 비동기 요청** 때문입니다.

일반적인 웹 브라우저 주소창에 URL을 입력하거나 `<form>` 태그로 요청을 보내는 것과, 자바스크립트의 `fetch`로 요청을 보내는 것은 **302 Found(리다이렉트)**를 처리하는 방식이 완전히 다릅니다.

---

### 1. 왜 페이지가 그대로인가요? (Fetch의 동작 원리)

1.  **브라우저의 자동 처리**: `fetch` 요청을 보냈을 때 서버가 `302 Found`와 함께 `Location` 헤더를 주면, 브라우저는 **사용자 몰래(백그라운드에서)** 그 `Location` 주소로 다시 요청을 보냅니다.
2.  **결과물만 전달**: 브라우저는 최종적으로 리다이렉트된 페이지(예: 로그인 페이지의 HTML)를 받아온 뒤, 그 결과를 자바스크립트(`fetch`의 결과값)에게 던져줍니다.
3.  **화면 갱신 안 됨**: `fetch`는 데이터를 가져오는 "도구"일 뿐입니다. 가져온 결과가 HTML이든 JSON이든, 자바스크립트 코드에서 `window.location.href = ...` 같은 명령을 내리지 않는 한 **브라우저의 주소창이나 전체 화면은 바뀌지 않습니다.**

그래서 화면은 그대로인데 네트워크 탭에서만 "302 -> 200(로그인 페이지)"의 흐름이 보이는 것입니다.

---

### 2. 지금 코드에서의 문제점

현재 작성하신 `SecurityConfig`를 보면 두 가지 설정이 충돌하고 있습니다.

* **설정**: `.anyRequest().authenticated()` (모든 요청은 로그인 필요)
* **상황**: 프론트엔드에서 로그인을 안 한 채로 `/api/boards`를 `fetch`로 호출함.
* **서버 응답**: 시큐리티가 "로그인 안 했네? `http://localhost:3000/login`으로 가!"라며 **302 리다이렉트**를 보냄.
* **프론트 결과**: `useFetch`는 게시글 목록(JSON) 대신 로그인 페이지의 **HTML 소스**를 응답으로 받게 됨. 데이터 형식이 다르니 에러가 나거나 화면에 아무것도 안 뜨게 됩니다.

---

### 3. 어떻게 해결해야 하나요?

게시글 목록 API가 302 리다이렉트를 뱉지 않게 하려면 **두 가지 방법** 중 하나를 선택해야 합니다.

#### 방법 A: 게시글 목록 API를 권한 허용 목록에 추가 (가장 추천)
게시글 목록은 로그인을 안 해도 볼 수 있는 것이 일반적이므로, `permitAll()`에 추가하세요.

```java
.authorizeHttpRequests(auth -> auth
        .requestMatchers("/", "/api/members/signup", "/member/login", "/member/join", "/css/**", "/js/**").permitAll()
        // 추가: GET 방식의 게시글 관련 API는 모두 허용
        .requestMatchers(HttpMethod.GET, "/api/boards/**").permitAll() 
        .anyRequest().authenticated()
)
```

#### 방법 B: 302 대신 401 Unauthorized 응답 보내기
API 서버라면 권한이 없을 때 "페이지 이동(302)"이 아니라 "너 권한 없어(401)"라고 명확하게 에러를 줘야 합니다. 그래야 프론트엔드에서 `if (error.statusCode === 401) { ... }` 같은 처리를 할 수 있습니다.

```java
.exceptionHandling(handler -> handler
        .authenticationEntryPoint((request, response, authException) -> {
            // 리다이렉트 대신 401 상태코드와 에러 메시지 반환
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.setCharacterEncoding("UTF-8");
            response.getWriter().write("로그인이 필요한 서비스입니다.");
        })
)
```

---

### 💡 요약
* **302인데 페이지가 안 바뀌는 이유**: `fetch` 요청은 브라우저가 백그라운드에서 리다이렉트를 처리해버리고 화면은 갱신하지 않기 때문입니다.
* **해결책**: 게시글 목록 API(`GET /api/boards`)를 **`permitAll()`**에 등록하여 시큐리티가 리다이렉트를 시키지 않도록 만드세요.

지금 바로 `SecurityConfig`에 `.requestMatchers(HttpMethod.GET, "/api/boards/**").permitAll()`를 추가해서 다시 테스트해 보세요! 화면에 데이터가 바로 나타날 겁니다.
