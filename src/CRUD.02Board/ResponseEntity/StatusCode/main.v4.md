

##
실무에서 REST API를 설계할 때 `200`번대(Success) 코드를 세분화하는 것은 프론트엔드 개발자나 API 소비자와의 **"무언의 약속"**과 같습니다. 단순히 "성공했어"라고 하는 것보다 "성공해서 새로 만들었어"라고 말해주는 것이 훨씬 친절하니까요.

실무에서 가장 빈번하게 사용하는 200번대 상태 코드 4가지를 정리해 드릴게요.

---

## 실무 핵심 200대 상태 코드 TOP 4

### 1. `200 OK` (가장 범용적)
* **의미:** 요청이 성공적으로 처리됨.
* **언제 쓰는가:** 데이터 조회(GET), 단순 수정(PUT/PATCH) 성공 시 가장 많이 사용합니다.
* **ResponseEntity 사용법:**
    ```java
    return ResponseEntity.ok(dto);
    // 또는
    return ResponseEntity.status(HttpStatus.OK).body(dto);
    ```

### 2. `201 Created` (생성 성공)
* **의미:** 요청이 성공적이었고, 그 결과로 **새로운 리소스가 생성됨.**
* **언제 쓰는가:** 회원가입, 게시글 작성 등 데이터 등록(POST) 성공 시 사용합니다.
* **특징:** 실무에서는 생성된 객체의 ID나 URI를 함께 보내주기도 합니다.
* **ResponseEntity 사용법:**
    ```java
    return ResponseEntity.status(HttpStatus.CREATED).body(savedDto);
    ```

### 3. `204 No Content` (데이터 없음)
* **의미:** 요청은 성공했으나, **응답 본문(Body)에 보낼 데이터가 없음.**
* **언제 쓰는가:** 게시글 삭제(DELETE) 성공 시 주로 사용합니다. 삭제된 데이터를 굳이 다시 보내줄 필요가 없기 때문입니다.
* **특징:** 바디가 비어있으므로 앞서 물어보신 `.build()`를 반드시 써야 합니다.
* **ResponseEntity 사용법:**
    ```java
    return ResponseEntity.noContent().build(); 
    // status(HttpStatus.NO_CONTENT).build()와 같습니다.
    ```

### 4. `202 Accepted` (접수 완료)
* **의미:** 요청이 접수되었으나, **처리는 아직 완료되지 않음.**
* **언제 쓰는가:** 대용량 파일 업로드, 배치 작업 실행 등 비동기 처리가 필요할 때 사용합니다. "일단 접수는 됐으니 나중에 결과 확인해!"라는 뜻입니다.
* **ResponseEntity 사용법:**
    ```java
    return ResponseEntity.accepted().build();
    ```

---

## 요약 가이드 (어떤 걸 쓸지 고민될 때)



| 상황 | 추천 코드 | ResponseEntity 형태 |
| :--- | :---: | :--- |
| 데이터를 성공적으로 가져왔을 때 | **200** | `.ok(data)` |
| 새로운 데이터를 저장했을 때 | **201** | `.status(CREATED).body(data)` |
| 데이터를 수정(Update)했을 때 | **200** | `.ok(data)` |
| 데이터를 삭제(Delete)했을 때 | **204** | `.noContent().build()` |
| 작업이 오래 걸려서 예약만 했을 때 | **202** | `.accepted().build()` |

---

### 💡 실무 팁
사실 모든 성공을 `200 OK`로만 던져도 프로그램이 죽지는 않습니다. 하지만 **`201`**과 **`204`**만이라도 명확히 구분해 주면, 프론트엔드 쪽에서 "아, 이건 새로 생겨서 ID가 생겼겠구나" 혹은 "삭제됐으니 리스트에서 바로 지우면 되겠구나"라고 훨씬 쉽게 판단할 수 있습니다.

지금 만들고 계신 게시판의 **삭제 기능**에는 `204`를, **글쓰기 기능**에는 `201`을 적용해 보시는 건 어떨까요?

혹시 프로젝트에서 이런 상태 코드 구분이 필요한 구체적인 기능이 또 있으신가요?
