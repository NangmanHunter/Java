MySQL과 MariaDB는 사실상 **한 뿌리에서 나온 형제**와 같습니다. 그래서 사용법이나 구조가 거의 99% 일치하지만, 태생과 지향점에는 약간의 차이가 있습니다.

---

## 1. 관계 (History)
* **MySQL:** 세계에서 가장 많이 쓰이는 오픈소스 데이터베이스입니다. 썬 마이크로시스템즈를 거쳐 현재는 **오라클(Oracle)** 사가 소유하고 있습니다.
* **MariaDB:** 오라클이 MySQL을 인수한 후, "이러다 MySQL이 유료화되거나 폐쇄적으로 변하면 어쩌지?"라는 걱정에 MySQL 개발자들이 나와서 만든 **오픈소스 DB**입니다. 



---

## 2. 주요 차이점

| 구분 | MySQL | MariaDB |
| :--- | :--- | :--- |
| **소유주** | 오라클 (Oracle) | MariaDB 재단 (오픈소스) |
| **호환성** | 업계 표준 수준의 안정성 | MySQL과 완벽 호환 (명령어, 라이브러리 동일) |
| **성능** | 안정적이고 검증된 성능 | 특정 상황(복잡한 쿼리 등)에서 MySQL보다 빠름 |
| **라이선스** | 유료/무료 버전이 나뉨 | 완전 무료 (GPL 라이선스) |
| **기능** | 보수적이고 안정적인 업데이트 | 새로운 기능과 엔진을 더 빠르게 도입 |

---

## 3. 왜 MariaDB를 더 추천하나요?
보통 개인 프로젝트나 게시판 세팅 시 MariaDB를 많이 추천하는 이유는 다음과 같습니다.

1.  **가벼움:** MySQL보다 메모리 점유율이 약간 더 낮아 사양이 낮은 AWS EC2(Free Tier)에서 돌리기에 더 적합합니다.
2.  **완벽한 대체 가능:** `mysql`이라는 명령어를 터미널에서 그대로 사용할 수 있을 정도로 똑같습니다. 나중에 MySQL로 갈아타는 것도 매우 쉽습니다.
3.  **오픈소스 정신:** 특정 기업(오라클)에 종속되지 않는다는 심리적 안정감이 있습니다.

---

## 4. 게시판 구축 시 실제 체감되는 부분
게시판 코딩을 하실 때, 어떤 것을 선택하든 소스 코드(Java/Spring Boot) 레벨에서는 차이가 거의 없습니다.

* **의존성(Dependency):** `mysql-connector-java` 또는 `mariadb-java-client` 중 아무거나 쓰셔도 됩니다.
* **쿼리문:** `SELECT`, `INSERT`, `UPDATE` 같은 표준 SQL 문법은 100% 동일합니다.
* **접속 포트:** 두 DB 모두 기본 포트로 **3306**을 사용합니다.

* **"조금이라도 더 가볍고, 완전한 무료 오픈소스를 쓰고 싶다"** → **MariaDB**
