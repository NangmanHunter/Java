어떤 게시판을 만드느냐에 따라 선택지가 달라지겠지만, 현재 학습 단계와 게시판의 범용성을 고려하면 **관계형 데이터베이스(RDBMS)**, 그중에서도 **MySQL**이나 **MariaDB**를 가장 추천합니다.

AWS 환경에서 게시판용 DB를 선택할 때 고려할 수 있는 세 가지 옵션을 정리해 드릴게요.

---

## 1. MySQL 또는 MariaDB (가장 추천)
가장 표준적인 선택입니다. 게시판은 글 번호, 작성자, 작성일, 본문 등 데이터의 구조가 명확하기 때문에 표(Table) 형태의 RDBMS가 딱 맞습니다.

* **장점:** 커뮤니티가 매우 커서 문제가 생겼을 때 구글링으로 99% 해결 가능합니다. Spring Boot와의 호환성도 최고 수준입니다.
* **추천 상황:** "일단 표준적인 게시판을 만들고 싶다"면 무조건 이것으로 시작하세요.

## 2. PostgreSQL
MySQL보다 더 복잡한 기능(복잡한 쿼리 처리, 데이터 무결성 등)에 강점이 있습니다.

* **장점:** 최근 개발 트렌드에서 인기가 매우 높고, 표준 SQL을 아주 엄격하게 잘 지킵니다. 나중에 통계나 복잡한 데이터 분석 기능을 넣을 때 유리합니다.
* **추천 상황:** "나는 조금 더 정교하고 탄탄한 DB를 공부하면서 써보고 싶다"는 경우에 적합합니다.

## 3. H2 Database (테스트용)
서버에 따로 설치할 필요 없이 메모리 안에서 돌아가는 아주 가벼운 DB입니다.

* **장점:** 설정이 아주 간편해서 로컬에서 개발할 때 금방 띄울 수 있습니다.
* **추천 상황:** 서버에 정식으로 올리기 전, 내 컴퓨터에서 빠르게 기능을 구현해 볼 때만 사용하세요. (서버를 끄면 데이터가 사라지도록 설정될 수 있어 실제 서비스용으로는 부적합합니다.)

---

## 💡 AWS에서 어떻게 세팅할 것인가?
DB 소프트웨어를 정했다면, AWS에서 실행하는 방식은 다음 두 가지 중 하나를 선택해야 합니다.

| 방식 | 설명 | 추천 여부 |
| :--- | :--- | :--- |
| **EC2에 직접 설치** | 웹 서버가 돌아가는 컴퓨터(EC2) 안에 DB도 같이 깔아서 쓰는 방식입니다. | **학습용/비용 절감용 추천.** 사양이 낮은 서버라면 속도가 느려질 수 있지만 관리가 단순합니다. |
| **RDS 서비스 이용** | AWS에서 DB만 전용으로 관리해 주는 별도의 서비스를 빌리는 방식입니다. | **실무형/안정성 중시.** 백업이나 복구가 자동으로 되지만 프리티어 기간이 끝나면 비용이 꽤 나갈 수 있습니다. |



### 결론 제안
**"EC2(Ubuntu) 서버에 MariaDB를 직접 설치"**해서 시작해 보세요. 
MySQL과 거의 똑같으면서도 가볍고 무료로 쓰기에 아주 좋습니다. 설치 방법이나 Spring Boot 설정 파일(`application.properties`)에 넣어야 할 내용이 필요하시면 바로 알려드릴게요!
