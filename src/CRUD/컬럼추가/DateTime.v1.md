SQL 명령어를 공부 중이시군요! `ALTER TABLE` 문법에서 `datetime`은 해당 컬럼에 **어떤 종류의 데이터를 담을지** 결정하는 **데이터 타입(자료형)**입니다.

구체적으로 어떤 의미인지 핵심만 짚어드릴게요.

---

### 📅 DATETIME의 정체
`DATETIME`은 말 그대로 **날짜와 시간**을 동시에 저장하는 타입입니다. 

* **저장 형식:** `YYYY-MM-DD HH:MM:SS` (년-월-일 시:분:초)
* **범위:** 보통 `1000-01-01 00:00:00`부터 `9999-12-31 23:59:59`까지 지원합니다.
* **용도:** 게시글 작성일(`created_at`), 수정일, 예약 시간 등 정확한 시점이 중요할 때 사용합니다.

### 🔍 쿼리 전체 해석하기
작성하신 코드를 한 줄씩 뜯어보면 이렇습니다:

1.  **`ALTER TABLE board`**: `board`라는 테이블의 구조를 바꾸겠다.
2.  **`ADD COLUMN created_at`**: `created_at`이라는 이름의 기둥(컬럼)을 새로 추가하겠다.
3.  **`datetime`**: 그런데 그 기둥에는 **날짜와 시간 데이터만** 넣도록 하겠다.

---

### 💡 참고하면 좋은 유사 타입
DB 종류(MySQL, Oracle, PostgreSQL 등)에 따라 조금씩 다르지만, 보통 다음 타입들과 함께 쓰입니다.

| 데이터 타입 | 저장 내용 | 예시 |
| :--- | :--- | :--- |
| **DATE** | 날짜만 저장 | `2026-04-26` |
| **TIME** | 시간만 저장 | `19:16:50` |
| **DATETIME** | 날짜 + 시간 | `2026-04-26 19:16:50` |
| **TIMESTAMP** | 날짜 + 시간 (타임존 반영) | 시스템의 현재 시간대에 맞춰 자동 변환됨 |

> **Tip:** 보통 게시판의 `created_at`은 직접 입력하기보다, 데이터가 들어올 때 자동으로 현재 시간이 찍히도록 `DEFAULT CURRENT_TIMESTAMP` 같은 설정을 덧붙여서 많이 사용합니다.
