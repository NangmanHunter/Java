##
네, 맞습니다! 실무에서는 **엔티티(Entity)**와 **DTO** 두 곳 모두에서 빌더를 아주 많이 사용합니다. 특히 엔티티에 빌더를 사용하는 것은 거의 **'국룰'**에 가까울 정도로 권장되는데, 그 명확한 이유가 있습니다.

---

### 1. 왜 엔티티에 빌더를 쓰나요? (`Setter` 금지)

가장 큰 이유는 **엔티티의 안정성** 때문입니다.

* **Setter를 지양함:** 엔티티에 `@Setter`를 남발하면 어디서든 객체 값이 수정될 수 있어 데이터가 오염되기 쉽습니다.
* **불변성 유지:** 빌더를 쓰면 객체를 생성하는 시점에만 값을 딱 정해주고, 이후에는 값을 함부로 바꿀 수 없게 만들 수 있습니다. (생성 시점에 모든 책임이 집중됨)
* **가독성:** 아까 보셨듯이 필드가 많은 엔티티를 생성할 때 어떤 값이 제목인지, 어떤 값이 작성자인지 코드로 바로 읽힙니다.

---

### 2. 엔티티에서 빌더를 사용하는 올바른 위치

클래스 위가 아니라 **'생성자' 위에 `@Builder`를 붙이는 것**이 가장 좋습니다.

```java
@Entity
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED) // JPA를 위한 기본 생성자
public class Board {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;
    private String content;

    @Enumerated(EnumType.STRING)
    private BoardType boardType;

    // 클래스 위가 아니라, 필요한 필드만 포함된 생성자 위에 붙입니다.
    @Builder
    public Board(String title, String content, BoardType boardType) {
        this.title = title;
        this.content = content;
        this.boardType = boardType;
    }
}
```

* **주의:** `@NoArgsConstructor`는 JPA가 엔티티를 관리하기 위해 반드시 필요합니다. 하지만 외부에서 함부로 `new Board()`를 못 하게 `PROTECTED`로 막아두고, 생성은 오직 **빌더**를 통해서만 하게 유도하는 것이 정석입니다.

---

### 3. 서비스 코드에서의 변화

이제 아까 만드셨던 서비스 코드를 빌더 방식으로 바꾸면 이렇게 됩니다.

```java
@Transactional
public Long write(Long memberId, BoardType boardType, String title, String content) {
    Member member = memberRepository.findById(memberId)
            .orElseThrow(() -> new IllegalArgumentException("존재하지 않는 회원입니다."));

    // 생성자 대신 빌더를 사용!
    Board board = Board.builder()
            .title(title)
            .content(content)
            .boardType(boardType)
            .build();

    board.setMember(member); // 연관 관계 설정

    boardRepository.save(board);
    return board.getId();
}
```

---

### 4. DTO에서도 쓰나요?

네! 특히 서버가 클라이언트에게 응답을 줄 때 쓰는 **Response DTO**에서 빌더를 쓰면 아주 편합니다.

```java
// 예시: BoardResponse를 record가 아닌 클래스로 만들 때
@Builder
public record BoardResponse(Long id, String title) { }
// (Java 14 이상의 Record는 기본적으로 빌더를 지원하지 않지만, 
// 롬복의 @Builder를 붙여서 똑같이 사용할 수 있습니다.)
```
