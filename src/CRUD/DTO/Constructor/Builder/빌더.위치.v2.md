`Comment` 엔티티에서 `@Builder`가 무시되거나 제대로 작동하지 않는 이유는 크게 두 가지 가능성이 높습니다. 특히 **JPA 엔티티**에서 이 문제가 자주 발생하는데, 하나씩 짚어드릴게요.

---

### 1. @Builder와 @NoArgsConstructor의 충돌 (가장 유력)

JPA를 사용하면 프록시 객체 생성을 위해 기본 생성자(`@NoArgsConstructor`)가 반드시 필요합니다. 그런데 여기서 **자바의 기본 규칙**과 **Lombok의 동작 방식**이 충돌을 일으킵니다.

* **원인:** `@Builder`를 클래스 레벨에 붙이면 Lombok은 원래 "모든 필드를 인자로 받는 생성자"를 자동으로 만듭니다. 하지만 `@NoArgsConstructor`가 이미 존재하면, Lombok은 **"아, 생성자가 이미 있네? 그럼 내가 따로 안 만들게"** 하고 물러나 버립니다. 결과적으로 빌더가 필요로 하는 전용 생성자가 없어서 오류가 나거나 빌더가 동작하지 않게 됩니다.
* **해결책:** `@AllArgsConstructor`를 명시적으로 추가해 주어야 합니다. (이미 코드에 추가되어 있으시다면, 아래 2번을 확인해 보세요!)

### 2. @Builder 위치의 문제 (권장 사항)

클래스 레벨에 `@Builder`를 두면 모든 필드에 대해 빌더가 생성됩니다. 하지만 실무에서는 **필요한 필드만 받는 생성자 위에 `@Builder`를 붙이는 것**이 더 안전하고 명확합니다.

예를 들어, `id`는 `@GeneratedValue`로 자동 생성되는데 빌더에서 굳이 `id`를 넣을 필요가 없겠죠? 다음과 같이 개선해 보세요.

```java
@Entity
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED)
// @AllArgsConstructor // 클래스 레벨 빌더를 쓸 게 아니라면 굳이 필요 없습니다.
public class Comment extends BaseTimeEntity {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String content;
    private boolean isDeleted = false;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "board_id")
    private Board board;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "member_id")
    private Member member;

    // 필요한 필드만 포함한 생성자에 빌더 적용
    @Builder
    public Comment(String content, Board board, Member member) {
        this.content = content;
        this.board = board;
        this.member = member;
    }

    public void updateContent(String content){
        this.content = content;
    }
}
```

---

### 3. IDE 환경 문제 (혹시 모를 체크)

코드상 문제가 없는데도 "Cannot resolve symbol 'builder'" 같은 메시지가 뜬다면 의외로 설정 문제일 수 있습니다.

* **Annotation Processing:** IntelliJ 설정(`Settings > Build, Execution, Deployment > Compiler > Annotation Processors`)에서 **Enable annotation processing**이 체크되어 있는지 확인해 보세요.
* **Lombok Plugin:** IntelliJ 버전이 낮다면 롬복 플러그인이 설치되어 있는지, 최신 버전인지 확인이 필요합니다.

### 4. 사용법 확인
빌더를 호출할 때 `Comment.builder().content("내용").build();` 형태로 사용하고 계신지도 다시 한번 체크해 보세요. 클래스 이름 뒤에 `.builder()`가 붙어야 합니다.

---

**정리하자면:**
지금 코드에서 `@AllArgsConstructor`가 이미 붙어있는데도 안 된다면, **생성자에 직접 `@Builder`를 붙이는 방식**으로 바꿔보시는 걸 강력 추천합니다. `id`나 `isDeleted` 같은 필드를 빌더에서 제외할 수 있어 훨씬 깔끔한 코드가 됩니다.
