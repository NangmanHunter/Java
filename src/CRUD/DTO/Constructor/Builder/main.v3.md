##
`@Builder`를 붙였을 때 롬복이 우리 대신 작성해주는 **"완전히 같은 코드"**를 아주 단순한 예시로 보여드릴게요. 

만약 필드가 두 개인 `Board` 클래스가 있다고 가정해봅시다.


### 3. 왜 이렇게 복잡하게 만들까요?

우리가 실제 사용할 때의 차이를 보세요.

* **생성자 방식:** `new Board("제목", "내용")` 
    * (순서 틀리면 큰일 남, 뭐가 제목인지 코드만 봐선 모름)
* **빌더 방식:** `Board.builder().title("제목").content("내용").build()`
    * (순서 상관없음, `title`에 "제목"이 들어간다는 게 눈에 확 보임)

### 핵심 요약
1.  **`builder()`**: `Board` 안에 숨어있던 **`BoardBuilder`라는 도우미**를 밖으로 꺼내는 버튼입니다.
2.  **`title("...")`**: 도우미의 손에 재료를 쥐여주는 과정입니다. (이때 타입은 여전히 `BoardBuilder`입니다.)
3.  **`build()`**: 도우미가 쥐고 있던 재료들을 싹 모아서 **드디어 진짜 `Board` 객체**를 만들어 우리에게 건네주는 순간입니다.

결국 **`@Builder` = 전체 생성자 + 도우미 클래스 + 조립 로직**을 한 방에 해결해 주는 마법 같은 어노테이션인 셈이죠! 이제 구조가 완전히 잡히시나요?
