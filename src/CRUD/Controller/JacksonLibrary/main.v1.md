##
| Type                  | Json           | ValueDoubleQuotes |
| :-------------------- | :------------- | :---------------- |
| String                | "aaa": "Aaa"   | ✅""              |
| 숫자(int long double) | "number": 10   | ❌""              |
| boolean               | "isㅇㅇ": true | ❌""              |
| null                  | "aaa": null    | ❌""              |



##
JSON 데이터에서 **밸류(Value)값에 쌍따옴표(`""`)가 붙느냐 아니느냐**는 해당 필드의 **자바 데이터 타입**에 따라 결정됩니다.

결론부터 말씀드리면, **`boolean` 타입은 쌍따옴표가 붙지 않습니다.**

---

### 1. 타입별 JSON 표현 방식

JSON 표준 규격에 따라 데이터 타입마다 표기법이 다릅니다.

| 자바 타입 (Java)             | JSON 출력 예시  | 쌍따옴표 여부                       |
| ---------------------------- | --------------- | ----------------------------------- |
| **`boolean` / `Boolean**`    | `true`, `false` | **없음** (예: `"isApproved": true`) |
| **숫자 (int, long, double)** | `10`, `25.5`    | **없음** (예: `"age": 30`)          |
| **문자열 (String)**          | `"Hello"`       | **있음** (예: `"name": "Gemini"`)   |
| **null**                     | `null`          | **없음** (예: `"data": null`)       |



### 2. 왜 `boolean`은 따옴표가 없을까?

JSON에서 `true`와 `false`는 문자열이 아니라 불리언 리터럴(Literal)이라는 독립된 예약어입니다.

* `"true"` (따옴표 있음) → 문자열(String)로 인식
* `true` (따옴표 없음) → 논리값(Boolean)으로 인식

따라서 컨트롤러에서 `boolean` 타입으로 데이터를 보내면, 프론트엔드(자바스크립트 등)에서도 별도의 변환 없이 바로 `if (data.approved)` 처럼 논리 조건문으로 사용할 수 있습니다.

---

### ⚠️ 주의할 점

만약 자바 DTO에서 필드 타입을 `String`으로 선언하고 `"true"`라는 글자를 넣어서 보낸다면, JSON 결과값에도 쌍따옴표가 붙어서 나갑니다. 이 경우 프론트엔드에서는 이게 '글자'인지 '논리값'인지 헷갈릴 수 있으니, 논리 데이터는 반드시 `boolean` 타입을 사용하는 것이 좋습니다.

**요약하자면:**
컨트롤러에서 보낸 불리언 값은 JSON 상에서 **쌍따옴표 없이** `true` 또는 `false`로 깨끗하게 나갑니다! 편하게 조건문으로 쓰시면 돼요.
