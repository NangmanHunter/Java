


##
오, 아주 날카로운 질문이에요! 자바를 처음 배우면 보통 `main` 메서드가 끝나면 프로그램도 죽는 게 당연하다고 생각하게 되죠. 그런데 톰캣(Tomcat) 같은 서버는 한 번 켜두면 내가 끌 때까지 계속 살아있으니 신기할 법합니다.

이 차이는 단순히 "코드가 길어서"가 아니라, **스레드(Thread)의 관리 방식**과 **이벤트 루프(Event Loop)**라는 개념에서 옵니다.

---

## 1. 근본적인 차이: "누가 종료를 결정하는가?"

일반적인 자바 프로그램과 서버 프로그램의 가장 큰 차이점은 **'Non-daemon Thread(사용자 스레드)'**의 생존 여부입니다.

* **일반 프로그램:** `main` 메서드를 실행하는 스레드가 하나뿐입니다. 이 메서드의 중괄호(`}`) 끝에 도달하면 스레드가 종료되고, JVM은 "더 이상 할 일이 없네?" 하고 프로세스를 닫아버립니다.
* **서버 프로그램:** `main` 메서드는 단순히 **'서버를 세팅하고 실행하는 시작점'**일 뿐입니다. 서버가 구동되면 내부적으로 별도의 스레드(Request Listener 등)를 생성하여 무한히 대기하게 만듭니다.

---

## 2. 상태를 유지하는 방법 (메커니즘)

서버가 종료되지 않고 계속 도는 핵심 원리는 크게 세 가지로 요약할 수 있습니다.

### ① 무한 루프와 블로킹 (The Infinite Loop)
가장 원초적인 이유는 코드 어딘가에 **`while(true)`**와 같은 무한 루프가 돌고 있기 때문입니다. 

```java
while (true) {
    // 1. 누군가 접속할 때까지 여기서 멈춰서 기다림 (Blocking)
    Socket clientSocket = serverSocket.accept(); 
    
    // 2. 접속하면 새로운 스레드를 만들어서 일을 시킴
    new Thread(new RequestHandler(clientSocket)).start();
    
    // 3. 다시 위로 올라가서 다음 손님을 기다림
}
```


위의 `accept()` 메서드는 클라이언트가 접속하기 전까지는 다음 줄로 넘어가지 않고 **'블로킹(대기)'** 상태가 됩니다. 이 루프가 `main` 스레드 혹은 별도의 실행 스레드에서 돌고 있기 때문에 프로그램이 종료되지 않는 것입니다.

### ② 논-데몬 스레드 (Non-daemon Threads)
JVM은 **"살아있는 User Thread가 하나라도 있다면 프로그램을 종료하지 않는다"**는 규칙을 가지고 있습니다. 
서버 프레임워크(Spring, Tomcat 등)는 내부적으로 요청을 받는 스레드를 User Thread로 생성하여 계속 유지합니다.


### ③ 이벤트 루프 (Event Loop)
최근의 고성능 서버(Netty 등)는 무작정 스레드를 늘리는 대신, **이벤트 루프** 방식을 씁니다. 운영체제 레벨에서 "데이터 왔어!"라는 신호를 줄 때까지 기다렸다가, 신호가 오면 그걸 처리하는 루프를 계속 돌리는 방식이죠.

---

## 3. 요약: 일반 vs 서버

| 구분 | 일반 Java Application | Server Application (Web, WAS) |
| :--- | :--- | :--- |
| **흐름** | `main` 시작 → 로직 실행 → `main` 종료 | `main` 시작 → **환경 설정 및 스레드 풀 생성** → **무한 대기** |
| **종료 시점** | 모든 코드를 다 읽었을 때 | 관리자가 강제로 종료(Stop) 신호를 줄 때 |
| **핵심 코드** | 순차적 로직 | `while(true)` + `Socket.accept()` |

### 💡 한 줄 요약
서버가 계속 떠 있는 이유는 **"누군가 들어오길 기다리는 무한 루프 스레드"**가 백그라운드에서 죽지 않고 계속 일을 하고 있기 때문입니다!
