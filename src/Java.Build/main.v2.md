빌드와 컴파일은 떼려야 뗄 수 없는 사이지만, 범위로 따지면 **빌드(Build)가 훨씬 큰 개념**입니다. 결론부터 말씀드리면 **"컴파일은 빌드 과정의 핵심 단계 중 하나"**라고 이해하시면 정확합니다.

---

## 1. 컴파일 (Compile)
**"번역" 단계입니다.**

* **정의:** 사람이 작성한 소스 코드(`.java`)를 컴퓨터가 이해할 수 있는 바이너리 코드나 바이트 코드(`.class`)로 변환하는 작업입니다.
* **역할:** 문법 오류가 없는지 체크하고, 기계가 읽을 수 있는 형태로 바꿉니다.
* **도구:** `javac`가 이 역할을 수행합니다.

## 2. 빌드 (Build)
**"제품 출하" 단계입니다.**

* **정의:** 소스 코드를 실행 가능한 소프트웨어 산출물(Artifact, 예: `.jar`, `.war`, `.exe`)로 만드는 **전체 과정**을 말합니다.
* **포함 과정:** 컴파일은 기본이고, 그 외에 아주 많은 일이 일어납니다.



---

## 3. 빌드 안에 포함되는 주요 작업들
빌드 버튼 하나를 누르면 내부적으로 다음과 같은 일들이 순차적으로 일어납니다.

1.  **의존성 해결 (Dependency Resolution):** 프로젝트에 필요한 외부 라이브러리(예: JUnit, Spring)를 인터넷에서 다운로드합니다.
2.  **컴파일 (Compile):** 소스 코드를 `.class` 파일로 번역합니다.
3.  **테스트 (Test):** 작성된 유닛 테스트들이 통과하는지 검증합니다. (하나라도 실패하면 빌드가 중단됩니다.)
4.  **리소스 처리 (Resource Processing):** 이미지, 설정 파일(XML, YAML) 등을 적절한 위치로 복사합니다.
5.  **패키징 (Packaging):** 컴파일된 코드와 리소스를 하나로 묶어 실행 가능한 파일(`.jar`)로 만듭니다.
6.  **정적 분석 (Static Analysis):** 코드에 잠재적인 버그나 스타일 문제가 없는지 검사합니다.

---

## 4. 한눈에 비교

| 구분 | 컴파일 (Compile) | 빌드 (Build) |
| :--- | :--- | :--- |
| **범위** | 부분적 (번역만 함) | 전체적 (번역 + 테스트 + 포장) |
| **결과물** | `.class` 파일 | `.jar`, `.war` 같은 실행 파일 |
| **비유** | 식재료를 손질하는 것 | 요리를 완성해서 접시에 담아 내놓는 것 |

### 💡 정리하자면
"컴파일 완료됐어?"는 **"번역 잘 됐어?"**라는 뜻이고,
"빌드 성공했어?"는 **"테스트 다 통과하고 배포 가능한 파일까지 잘 만들어졌어?"**라는 뜻입니다.

이제 `build.gradle`에서 왜 `testImplementation` 같은 설정을 넣는지 이해가 좀 더 잘 되시나요? 테스트도 빌드 과정의 중요한 일부이기 때문입니다!

혹시 방금 만든 프로젝트에서 빌드 오류가 발생해서 질문하신 건가요? 에러 메시지가 있다면 같이 해결해 드릴 수 있습니다.
