현재 인텔리제이에서는 두 가지 DSL 중 하나를 선택할 수 있습니다.

### ① Groovy DSL (`build.gradle`)
* **특징:** 오래전부터 쓰인 전통적인 방식입니다. 문법이 자유롭고 유연합니다.
* **장점:** 설정 코드가 짧고, 참고할 수 있는 예제 코드가 인터넷에 압도적으로 많습니다.
* **단점:** 코드가 "문자열" 기반인 경우가 많아 오타가 나도 미리 알려주지 않는 경우가 있습니다.


### ② Kotlin DSL (`build.gradle.kts`)
* **특징:** 코틀린 언어를 사용하여 작성합니다. 최신 트렌드이며 구글과 JetBrains가 권장합니다.
* **장점:** **자동 완성**이 기가 막히게 잘 됩니다. 오타를 치면 빨간 줄이 바로 떠서 실수를 줄여줍니다. (인텔리제이와 찰떡궁합!)
* **단점:** 처음 로딩 속도가 Groovy보다 약간 느릴 수 있고, 문법이 조금 더 엄격합니다.




## 한눈에 비교하기

| 구분 | 빌드 시스템 (Build System) | Gradle DSL |
| :--- | :--- | :--- |
| **개념** | 어떤 공장을 돌릴 것인가? | 공장 주문서를 무슨 언어로 쓸 것인가? |
| **선택지** | IntelliJ, Maven, **Gradle** | **Groovy**, **Kotlin** |
| **비유** | 레고를 조립하는 **사람** | 레고 조립 **설명서의 언어** (한국어 vs 영어) |
