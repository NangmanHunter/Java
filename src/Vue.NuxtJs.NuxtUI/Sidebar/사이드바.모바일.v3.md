##
사용자 입장에서 생각해보면, **'어디를 눌러야 닫히는지 명확하게 알려주는 것'**이 가장 중요합니다.

하단 빈 공간을 클릭해서 닫는 기능은 있으면 좋지만, 사용자가 "여기를 누르면 닫히겠구나"라고 미리 짐작하기 어렵습니다. 그래서 **'직관적인 X 버튼'**과 **'당연하게 느껴지는 배경(Overlay) 클릭'** 두 가지를 조합하는 것이 베스트입니다.

제가 추천하는 구성은 다음과 같습니다.

---

### 1. 제가 추천하는 베스트 시나리오
*   **상단에 'X' 버튼 추가:** 모바일에서는 사이드바가 화면을 많이 가리기 때문에, 확실하게 끌 수 있는 버튼이 눈에 보여야 합니다.
*   **배경 오버레이(Overlay) 클릭:** 메뉴 외의 본문(어둡게 처리된 부분)을 누르면 닫히는 기능은 이제 모바일의 '표준'입니다.
*   **하단 빈 공간 클릭:** 이건 `USidebar` 내부의 `div`에 클릭 이벤트를 걸어서 구현할 수 있지만, 굳이 하나만 택하라면 **X 버튼**이 우선입니다.

---

### 2. 코드 구현 방법

#### A. 상단에 'X' 버튼 달기 (모바일 전용)
헤더 부분에 팀 선택 드롭다운 옆에 닫기 버튼을 배치하세요.

```vue
<template #header>
  <div class="flex items-center justify-between w-full">
    <!-- 기존 팀 선택 드롭다운 -->
    <UDropdownMenu ... >
      <UButton ... />
    </UDropdownMenu>

    <!-- [추가] 모바일에서만 보이는 닫기 버튼 -->
    <UButton
      icon="i-lucide-x"
      color="neutral"
      variant="ghost"
      class="lg:hidden" 
      @click="open = false"
    />
  </div>
</template>
```

#### B. 하단 빈 공간 클릭 시 닫기
사이드바의 내용물(Default 슬롯) 하단에 `flex-1`을 가진 빈 공간을 만들고, 거기에 클릭 이벤트를 걸어주면 됩니다.

```vue
<template #default="{ state }">
  <div class="flex flex-col h-full">
    <!-- 실제 메뉴 -->
    <UNavigationMenu
      :key="state"
      :items="getItems(state)"
      ...
    />
    
    <!-- [추가] 하단 남는 공간 - 클릭 시 사이드바 닫힘 -->
    <div 
      class="flex-1 cursor-pointer" 
      @click="open = false"
    >
      <!-- 공간을 채우기 위한 투명 영역 -->
    </div>
  </div>
</template>
```

---

### 3. 결론: "X 버튼"을 추천하는 이유
사이드바 하단 공간을 클릭해서 닫는 방식은 사용자가 의도치 않게 메뉴 아래쪽을 잡으려다 사이드바를 꺼버리는 **'오작동'**을 유발할 수 있습니다.

반면, 상단의 **X 버튼**은:
1.  **명확성:** 누구나 보면 닫는 버튼인 줄 압니다.
2.  **안정성:** 실수로 누를 확률이 적습니다.
3.  **심미성:** 디자인적으로도 모바일 앱 레이아웃의 완성도를 높여줍니다.

**저라면 상단에 X 버튼을 넣고, `USidebar`에 `overlay` 속성을 줘서 배경 클릭 시 닫히게 만드는 조합으로 가겠습니다.** 이게 사용자에게 가장 친숙하고 오류 없는 방식이에요!
