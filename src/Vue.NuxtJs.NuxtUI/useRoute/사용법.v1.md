##
결론부터 말씀드리면, **`useRoute`와 `useRuntimeConfig`는 매번 컴포넌트마다 선언해서 사용하는 것이 원칙이자 정석**입니다.

"전역으로 하나만 만들어서 공유하면 편하지 않을까?"라는 생각이 드실 텐데, 왜 굳이 매번 가져다 쓰는지 그 이유와 실무에서의 활용 패턴을 정리해 드릴게요.

---

### 1. 왜 매번 컴포넌트 안에서 선언할까?

#### ① `useRoute()`는 "현재"를 의미하기 때문
`useRoute`는 현재 **활성화된 컴포넌트의 맥락(Context)**을 담고 있습니다. 
* 페이지가 바뀔 때마다 `route` 정보는 실시간으로 변합니다. 
* 만약 전역 변수 어딘가에 고정해버리면, 페이지 이동 시 값이 제대로 갱신되지 않거나 엉뚱한 페이지의 파라미터를 참조하는 사고가 날 수 있습니다.
* Nuxt는 컴포넌트가 로드될 때 해당 시점의 주소 정보를 주입해주기 때문에, 필요한 곳에서 직접 호출하는 것이 가장 안전합니다.

#### ② `useRuntimeConfig()`는 트리 쉐이킹과 관련
설정값들을 매번 호출하는 게 무거워 보일 수 있지만, 사실 이건 함수를 실행해서 이미 메모리에 올라와 있는 객체에 접근하는 것뿐이라 **성능 부하가 거의 없습니다.** 오히려 명시적으로 선언해서 쓰는 게 유지보수와 타입 추론에 훨씬 유리합니다.

---

### 2. 사람들(실무자)은 보통 이렇게 씁니다

#### 패턴 A: 표준적인 방식 (가장 많이 사용)
유저님이 지금 하고 계신 방식입니다. 필요한 컴포넌트에서만 딱딱 선언합니다.
```javascript
// PostDetail.vue
const route = useRoute()
const config = useRuntimeConfig()

const postId = route.params.id
```

#### 패턴 B: API 요청 전담 함수(Composables)로 분리
매번 `$fetch`를 쓸 때 `config.public.apiBase`를 적기 귀찮으니, 아예 **공통 통신 로직**을 만들어 버립니다.

```javascript
// composables/useApi.ts
export const useApi = (url: string, options = {}) => {
  const config = useRuntimeConfig()
  
  return $fetch(url, {
    baseURL: config.public.apiBase,
    ...options
  })
}

// 컴포넌트에서 사용 시 (config 선언 필요 없음!)
const data = await useApi(`/api/boards/notice/1`)
```
이렇게 하면 컴포넌트 안에서는 `config`를 직접 부를 일이 확 줄어듭니다.

#### 패턴 C: 비즈니스 로직 분리 (Custom Composables)
댓글 등록처럼 반복되는 로직은 아예 '기능 단위'로 묶어서 `route`까지 그 안에 넣어버립니다.

```javascript
// composables/useComment.ts
export const useComment = () => {
  const route = useRoute()
  const config = useRuntimeConfig()

  const submit = async (content: string) => {
    const { boardType, id } = route.params
    // ... 전송 로직
  }

  return { submit }
}

// 컴포넌트에서는 한 줄로 끝!
const { submit } = useComment()
```

### 3. "전역"처럼 쓰고 싶을 때의 꿀팁 (Auto-import)

Nuxt의 정말 낭만적인 기능 중 하나가 **Auto-import**입니다.
* 사실 유저님이 `import { useRoute } from 'nuxt/app'` 이런 코드를 안 써도 `useRoute()`를 바로 쓸 수 있는 이유입니다.
* 이미 Nuxt가 "전역에서 언제든 꺼내 쓸 수 있게" 준비를 다 해놓은 상태입니다.
* 따라서 `const route = useRoute()`를 적는 행위 자체가 "전역에 있는 주소 창고에서 내 정보를 꺼내오겠다"는 선언이라고 보시면 됩니다.
