

##
`row` 객체 때문에 당황스러우셨죠? 특히 `original`이라는 이름이 생소하실 텐데, 사실 원리만 알면 아주 합리적인 구조입니다.

유저님이 사용하시는 **TanStack Table(v8)**에서 `row`는 단순히 데이터 한 줄이 아니라, **"테이블이 데이터를 관리하기 위해 감싸놓은 래퍼(Wrapper) 객체"**입니다.

가장 자주 쓰는 핵심 속성 3가지를 깔끔하게 정리해 드릴게요.

---

### 1. `row.original` (가장 중요 ⭐)
이게 바로 유저님이 **백엔드에서 가져온 "순수 데이터 객체"** 그 자체입니다.
*   **왜 쓰나요?** 테이블 기능과 상관없이 내가 정의한 필드(예: `isApproved`, `loginId`, `email`)에 직접 접근하고 싶을 때 사용합니다.
*   **예시:** `v-model="row.original.isApproved"` (원본 데이터의 값을 직접 바꿀 때)

### 2. `row.getValue('key')`
특정 컬럼의 **"가공된 값"**을 가져올 때 사용합니다.
*   **왜 쓰나요?** 단순히 `original`에서 값을 꺼내는 게 아니라, 테이블 설정에 정의된 `accessorKey`를 기반으로 안전하게 값을 추출합니다.
*   **특징:** 만약 해당 컬럼에 `cell` 가공 로직이 있다면 그 결과값을 가져오는 경우도 있습니다.

### 3. `row.getIsSelected()` / `row.toggleSelected()`
이것은 테이블의 **"행 선택(Selection)"** 상태를 관리합니다.
*   **왜 쓰나요?** 왼쪽 끝에 전체 선택 체크박스를 만들어서 여러 명을 한꺼번에 승인하거나 삭제할 때, 이 행이 체크되었는지 확인하는 용도입니다.

---

### 💡 왜 그냥 데이터를 안 주고 `row`로 감싸서 주나요?

테이블은 단순히 데이터를 보여주는 걸 넘어 **정렬, 필터링, 선택, 그룹화** 같은 복잡한 일을 해야 합니다.



*   **원본 보호:** `original`에 원본을 보관해두어야 나중에 필터링을 하거나 정렬을 해도 데이터가 꼬이지 않습니다.
*   **메타 정보:** 이 줄이 몇 번째 인덱스인지(`row.index`), 지금 펼쳐진 상태인지(`row.getIsExpanded()`) 같은 **상태값**을 보관할 장소가 필요하기 때문입니다.

---

### 🛠️ 실전 팁: 언제 뭘 써야 할까?

*   **"화면에 이 값만 보여줄래"** → `row.getValue('amount')`
*   **"이 버튼 누르면 이 유저 ID로 API 호출할래"** → `row.original.id`
*   **"체크박스랑 유저 승인 상태를 동기화할래"** → `v-model="row.original.isApproved"`

이제 `original`을 보시면 **"아, 이게 내가 만든 진짜 데이터구나!"**라고 반갑게 맞아주시면 됩니다. :)
