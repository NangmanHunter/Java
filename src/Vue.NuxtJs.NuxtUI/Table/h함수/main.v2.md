가장 간단하게 생각하면, `h` 함수는 **"HTML 태그를 자바스크립트 객체로 만드는 도구"**입니다.

문법은 딱 3층 구조만 기억하시면 됩니다.
`h( '태그이름', { 속성 }, '내용' )`

---

### 1. 아주 간단한 3가지 예시

#### ① 텍스트만 있는 태그
```javascript
// <p>안녕하세요</p>
h('p', '안녕하세요') 
```

#### ② 속성(Class, ID 등)이 있는 태그
```javascript
// <button class="btn-primary" id="submit">전송</button>
h('button', { class: 'btn-primary', id: 'submit' }, '전송')
```

#### ③ 자식 요소가 있는 태그 (배열 사용)
```javascript
// <div> <span>안녕</span> </div>
h('div', [
  h('span', '안녕')
])
```

---

### 2. Vue 컴포넌트를 쓸 때 (가장 많이 쓰는 형태)
컴포넌트를 쓸 때는 태그 이름 대신 **컴포넌트 변수**를 넣고, 자식 요소는 **함수(Slot)** 형태로 전달하는 것이 정석입니다.

```javascript
// <UBadge color="green">성공</UBadge>
h(UBadge, { color: 'green' }, () => '성공')
```
*   **두 번째 인자:** `props`(데이터)를 전달합니다.
*   **세 번째 인자:** `() => '내용'` (슬롯). Vue에게 "여기에 '성공'이라는 글자를 끼워 넣어줘"라고 알려주는 방식입니다.

---

### 3. 왜 `() =>` (화살표 함수)를 쓰나요?
이게 제일 잡스러워 보이는 부분이죠? Vue 3에서는 **성능** 때문에 이렇게 합니다.

*   그냥 `'성공'`이라고 적으면 화면이 바뀔 때마다 바로 데이터를 굽습니다.
*   `() => '성공'`이라고 적으면, Vue가 "아, 이 배지는 나중에 화면에 그릴 때 이 함수를 실행해서 내용을 채워야지!"라고 **지연 실행**을 할 수 있어 훨씬 효율적입니다.

3.  **세 번째:** 안에 뭘 넣을지? (텍스트나 자식 노드)
