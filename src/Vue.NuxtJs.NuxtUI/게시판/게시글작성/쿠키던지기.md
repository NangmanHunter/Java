세션(Session)이 넘어가지 않는 이유는 보통 **CORS(Cross-Origin Resource Sharing)** 설정 때문일 확률이 매우 높습니다. 

현재 Nuxt(프론트엔드)는 `localhost:3000`에서 돌고, Spring Boot(백엔드)는 `localhost:8080` 같은 다른 포트에서 돌고 있죠? 브라우저는 보안상의 이유로 **다른 도메인(포트 포함)으로 요청을 보낼 때 쿠키(세션 ID)를 기본적으로 포함하지 않습니다.**

이를 해결하기 위해 확인해야 할 3가지 체크포인트를 짚어드릴게요.

---

### 1. 프론트엔드: `credentials` 옵션 추가
`$fetch`나 `fetch`를 사용하실 때, 브라우저에게 "이 요청에 쿠키를 포함해줘!"라고 명시해야 합니다. `$fetch`에서는 `credentials: 'include'` 옵션을 사용합니다.

```javascript
const responseText = await $fetch('/api/boards', {
  baseURL: config.public.apiBase, 
  method: 'POST',
  // 💡 이 옵션이 있어야 브라우저의 쿠키(JSESSIONID)가 백엔드로 전달됩니다.
  credentials: 'include', 
  body: { 
    title: title.value, 
    content: content.value 
  }
})
```

### 2. 백엔드(Spring): CORS 설정 확인
백엔드에서도 "프론트엔드에서 보낸 쿠키를 내가 허용하겠다"는 설정이 반드시 필요합니다. `WebMvcConfigurer` 설정 클래스에 다음 내용이 있는지 확인해 보세요.

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOrigins("http://localhost:3000") // 프론트엔드 주소 정확히 명시
                .allowedMethods("GET", "POST", "PUT", "DELETE")
                .allowCredentials(true); // 💡 중요: true로 설정해야 세션/쿠키 수신 가능
    }
}
```
> **주의:** `.allowedOrigins("*")`처럼 와일드카드를 쓰면 `allowCredentials(true)`와 함께 사용할 수 없습니다. 반드시 특정 도메인을 명시해야 합니다.

### 3. 로그인 시점 확인
혹시 로그인을 처리하는 컨트롤러에서 세션을 생성할 때, 로그인이 성공했음에도 세션이 유지되지 않는 경우라면 크롬 브라우저의 **'개발자 도구(F12) -> Application -> Cookies'** 탭을 확인해 보세요. 
* 여기에 `JSESSIONID`라는 이름의 쿠키가 들어와 있나요?
* 만약 없다면 로그인 로직 자체에서 세션이 생성되지 않았거나, 응답 시 쿠키가 제대로 전달되지 않은 것입니다.

### 💡 추가 팁: 컨트롤러 디버깅
서버 로그에 `loginMember`가 `null`로 찍히는지 확인하기 위해 컨트롤러 첫 줄에 로그를 찍어보시는 것도 좋습니다. 

```java
@PostMapping
public String write(@RequestBody BoardCreateRequest request,
                    @SessionAttribute(name = "loginMember", required = false) Member loginMember) {
    
    System.out.println("loginMember: " + loginMember); // 로그 찍어서 null 여부 확인
    // ...
}
```

만약 `credentials: 'include'`를 넣었는데도 안 된다면, 현재 사용 중인 서버 도메인 환경(예: IP 주소로 접속 중인지 등)을 알려주세요! 그에 맞춰 더 상세히 봐 드릴게요.
