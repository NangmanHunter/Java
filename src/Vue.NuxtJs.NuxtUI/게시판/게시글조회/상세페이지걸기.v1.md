상세 페이지로 이동하는 링크를 거는 방법은 두 가지가 있습니다. **`columns` 설정 안에서 함수로 처리**하는 방법과 **템플릿 슬롯(Slot)을 사용**하는 방법입니다.

Nuxt 환경에서는 `NuxtLink`의 기능을 온전히 활용하고 HTML 구조를 직관적으로 볼 수 있는 **슬롯 방식**을 가장 추천합니다.

---

### 1. 추천 방식: 템플릿 슬롯 (Slot) 사용
`UTable`은 각 컬럼의 셀을 커스텀할 수 있도록 `#[accessorKey]-cell` 이라는 약속된 슬롯을 제공합니다.

```vue
<template>
  <UTable :data="boards || []" :columns="columns">
    <template #title-cell="{ row }">
      <NuxtLink 
        :to="`/board/${row.original.id}`" 
        class="text-blue-500 hover:text-blue-700 font-medium hover:underline"
      >
        {{ row.original.title }}
      </NuxtLink>
    </template>
  </UTable>
</template>
```

* **`#title-cell`**: `accessorKey`가 `title`인 컬럼의 데이터 칸을 지정합니다.
* **`row.original`**: TanStack Table의 규칙입니다. 현재 줄의 **원본 데이터 객체**(`{id, title, ...}`)에 접근할 때 사용합니다.

---

### 2. 다른 방식: `columns` 정의에서 해결 (`cell` 함수)
템플릿을 건드리지 않고 `script` 단에서 해결하고 싶을 때 사용합니다. 다만 이 방식은 Vue 컴포넌트(NuxtLink)를 직접 넣기가 까다로워 일반 `<a>` 태그를 쓰거나 렌더링 함수를 써야 합니다.

```typescript
const columns = [
  { accessorKey: 'id', header: '번호' },
  { 
    accessorKey: 'title', 
    header: '제목',
    // cell 함수 내에서 직접 가공
    cell: ({ row }: any) => {
      // 이 방식은 단순 텍스트 가공에 유리하며, 
      // 복잡한 링크는 위 1번(슬롯) 방식이 훨씬 유지보수에 좋습니다.
      return row.original.title 
    }
  },
  { accessorKey: 'writerName', header: '작성자' },
  { accessorKey: 'viewCount', header: '조회수' }
]
```

---

### 🧐 왜 `row.original`을 써야 하나요?



TanStack Table에서 `row` 객체는 단순히 데이터만 담고 있는 게 아니라, "이 줄이 선택되었는가?", "이 줄이 확장되었는가?" 같은 **상태 정보**를 잔뜩 가지고 있습니다. 
우리가 백엔드에서 받아온 **진짜 데이터**는 그 안에 `original`이라는 이름의 주머니에 따로 담겨 있기 때문에 `row.original.id`라고 불러줘야 합니다.

### 💡 팁: 경로 설정
현재 위치가 `/board`라면, 위에서 작성한 `:to="`/board/${row.original.id}`"` 처럼 **절대 경로**로 작성하는 것이 가장 안전합니다. (아까 겪으셨던 경로 꼬임 문제를 방지할 수 있습니다.)

이제 제목을 클릭하면 해당 게시글의 ID를 들고 상세 페이지로 잘 넘어갈 거예요!
