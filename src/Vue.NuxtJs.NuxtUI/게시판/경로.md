##
그 현상은 Nuxt(그리고 Vue Router)에서 **상대 경로(Relative Path)**를 해석하는 방식 때문에 발생하는 아주 흔한 혼란입니다!

결론부터 말씀드리면, 현재 주소가 `/board`인 상태에서 `./write`를 호출하면 브라우저는 `/board`를 **파일 이름**처럼 인식하고, 그 부모 디렉토리 수준에서 `write`를 찾으려 하기 때문에 `/write`가 되는 것입니다.

### 🛠 해결 방법 (3가지 중 선택)

가장 확실하고 실수를 방지하는 방법은 **절대 경로**를 쓰는 것입니다.

#### 1. 절대 경로 사용 (가장 권장)
앞에 `/`를 붙여서 주소의 시작점부터 명시하는 방식입니다. 고민할 필요 없이 가장 깔끔합니다.
```vue
<UButton to="/board/write" label="글쓰기" />
```

#### 2. 현재 경로 뒤에 붙이기 (상대 경로의 정석)
만약 반드시 상대 경로 느낌으로 가고 싶다면, `./`를 빼고 이름만 적거나 현재 폴더를 명시해야 합니다. 하지만 현재 `/board` 주소 상태에 따라 결과가 달라질 수 있어 위험합니다.
```vue
<UButton to="board/write" label="글쓰기" />
```

#### 3. 프로그래밍 방식으로 이동 (스크립트 활용)
주소를 동적으로 제어해야 할 때 사용합니다.
```vue
<script setup lang="ts">
const router = useRouter()
const goToWrite = () => router.push('/board/write')
</script>

<template>
  <UButton @click="goToWrite" label="글쓰기" />
</template>
```

---

### 🧐 왜 `./write`가 `/write`가 됐을까요?

웹 브라우저의 경로 해석 규칙 때문입니다.

* **현재 주소:** `http://localhost:3000/board`
* **해석:** 브라우저는 `/board`를 하나의 **파일명**으로 봅니다. (폴더가 아니라)
* **`./write` 요청:** "현재 파일(`board`)이 있는 위치에서 `write`라는 파일을 찾아줘!"
* **결과:** `/` 위치에서 `write`를 찾게 되어 `/write`가 됩니다.

> **💡 팁:** 만약 현재 주소가 `/board/` (끝에 `/`가 붙음)였다면, `./write`는 `/board/write`로 이동했을 거예요. 하지만 Nuxt는 기본적으로 끝에 슬래시를 붙이지 않으므로 **절대 경로(`/board/write`)**를 쓰는 습관을 들이는 게 정신 건강에 훨씬 좋습니다!

이제 버튼 누르면 `/board/write`로 잘 이동하나요? :)
