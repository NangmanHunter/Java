##
- `board/[id].vue`


##
네, 맞습니다! Nuxt의 **파일 기반 라우팅(File-based Routing)** 규칙에 따라 파일명을 지으시면 됩니다.

구체적으로는 아래와 같이 두 가지 방법 중 하나를 선택해서 구조를 잡으시면 됩니다.

---

### 1. 권장하는 방식 (디렉토리 구조)
상세 페이지 외에도 수정 페이지(`edit.vue`) 등이 추가될 예정이라면, **`board`**라는 폴더 안에 **`[id].vue`**를 넣는 방식이 가장 깔끔합니다.

* **경로:** `pages/board/[id].vue`
* **매핑 주소:** `http://localhost:3000/board/123`

### 2. 다른 방식 (파일 중심)
만약 `board`라는 폴더 없이 `pages` 바로 아래에 두고 싶다면 파일명 자체를 대괄호로 감싸면 됩니다.

* **경로:** `pages/board-[id].vue`
* **매핑 주소:** `http://localhost:3000/board-123`

---

### 💡 왜 대괄호 `[]`를 쓰나요?
이것을 **동적 파라미터(Dynamic Params)**라고 부릅니다.
대괄호 안에 들어가는 이름(`id`)은 변수명 역할을 합니다. 아까 코드에서 사용한 `route.params.id`의 **`id`**가 바로 이 파일명에서 오는 것입니다.

만약 파일명을 `pages/board/[no].vue`라고 지었다면, 코드에서는 `route.params.no`로 가져오게 됩니다.

### 📂 최종 추천 구조 (게시판 모듈)
지금 게시판을 만들고 계시니, 나중에 "수정하기" 기능까지 고려하면 아래 구조가 가장 정석입니다.

```text
pages/
└── board/
    ├── index.vue          // 목록 페이지 (/board)
    ├── write.vue          // 작성 페이지 (/board/write)
    ├── [id].vue           // 상세 페이지 (/board/1, /board/2 ...)
    └── edit/
        └── [id].vue       // 수정 페이지 (/board/edit/1)
```

이렇게 구성하면 Nuxt가 알아서 주소를 척척 만들어주니, 복잡한 설정 없이 개발에만 집중하실 수 있습니다!
