- Nuxt UI(혹은 Vue.js 프레임워크 전반)에서 말하는 **Prop(프롭)**은 Properties의 약자로, 부모 컴포넌트가 자식 컴포넌트에게 데이터를 전달하는 통로라고 이해하시면 됩니다.
- 컴포넌트를 하나의 '함수'라고 봤을 때 그 함수에 넣는 **'매개변수(인자)'**와 같은 역할을 합니다.

- Prop의 핵심 개념👉컴포넌트는 재사용이 가능해야 합니다. 예를 들어 Nuxt UI의 버튼 컴포넌트(<UButton />)를 쓸 때, 어떤 때는 빨간색, 어떤 때는 파란색이어야 하죠. 이때 "이번 버튼은 빨간색으로 그려줘!"라고 속성을 넘겨주는 것이 바로 Prop입니다.
- Prop의 핵심 개념👉방향성: 항상 부모에서 자식으로만 흐릅니다 (단방향 데이터 흐름).
- Prop의 핵심 개념👉목적: 컴포넌트의 모습(UI)이나 동작을 동적으로 제어하기 위함입니다.
- Prop의 특징 (주의사항)👉읽기 전용(Read-only): 자식 컴포넌트는 받은 Prop을 직접 수정할 수 없습니다. (데이터를 준 부모가 바꿔야 자식의 값도 바뀝니다.)
- Prop의 특징 (주의사항)👉타입 검사: Nuxt UI 내부적으로는 각 Prop이 숫자(Number)인지, 문자열(String)인지, 객체(Object)인지 미리 정의해 두어 잘못된 값이 들어오면 개발자 도구에서 경고를 띄워줍니다.
