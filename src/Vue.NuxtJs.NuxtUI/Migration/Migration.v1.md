##
HTML과 JavaScript로 된 코드를 Vue/Nuxt 같은 프레임워크 문법으로 옮기는 이 과정, 개발 현장에서는 주로 다음과 같은 용어들로 부릅니다.

### 1. 리팩토링 (Refactoring)
가장 보편적인 용어입니다. **"결과의 동작은 유지하면서, 코드의 구조를 개선하는 작업"**을 뜻합니다. 
사용자님은 기존 '회원가입'이라는 기능은 그대로 두되, 이를 Vue의 컴포넌트 구조와 `v-model` 방식으로 더 유지보수하기 좋게 고쳤으므로 리팩토링의 범주에 들어갑니다.

### 2. 마이그레이션 (Migration)
한 환경에서 다른 환경으로 통째로 옮길 때 씁니다.
* "레거시(HTML/JS) 코드를 **Nuxt 3로 마이그레이션** 중이야."
* 보통 기술 스택을 바꿀 때(예: jQuery -> Vue) 가장 많이 사용하는 단어입니다.

### 3. 컴포넌트화 (Componentization)
통으로 짜여 있던 HTML 코드를 잘게 쪼개서 `.vue` 파일이라는 **독립된 부품(Component)**으로 만드는 과정입니다. 
아까 `SignUpButton.vue`를 따로 만드신 것이 바로 이 컴포넌트화 과정의 핵심입니다.

---

### 💡 실무에서 대화할 때는 이렇게 말해요
> "기존에 구현된 HTML/JS 기반의 가입 페이지를 **Nuxt 3 컴포넌트 방식으로 리팩토링**하고 있습니다." 
> "데이터 바인딩을 **v-model**로 교체하고, 기존 DOM 접근 로직을 **Vue 스타일로 마이그레이션**하는 중이에요."

### 왜 이 과정이 중요한가요? (백엔드 관점)
백엔드에서 스파게티 코드를 Clean Code로 바꾸는 것과 같습니다.
* **명령형(Imperative) → 선언형(Declarative):** "ID가 loginId인 태그를 찾아서 값을 가져와라"라고 명령하는 대신, "이 변수와 저 입력창은 연결되어 있다"라고 선언하는 방식으로 바뀌면서 로직이 훨씬 단순해집니다.



방금 코드를 옮기면서 `document.getElementById`가 사라진 것이 바로 이 **선언형 프로그래밍**으로의 변화인데, 이 맛에 한 번 들리면 다시 생 HTML로 돌아가기 힘들 정도로 생산성이 올라가실 거예요!
