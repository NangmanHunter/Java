### 1. 상단 타이틀과 설명 (Static Data)
코드 윗부분의 `const` 변수들과 `<UPageSection>`의 속성들을 보세요. 여기가 화면 왼쪽(또는 상단)에 고정된 텍스트입니다.

* **수정 포인트 1 (브라우저 탭 제목):**
    ```javascript
    const title = '나의 낭만 프로젝트 업데이트' // 여기 수정
    const description = '비트코파크 베이스의 개발 기록입니다.' // 여기 수정
    ```
* **수정 포인트 2 (화면에 보이는 큰 제목):**
    ```vue
    <UPageSection
      title="업데이트 소식"  description="우리 프로젝트가 어떻게 변하고 있는지 확인하세요." ...
    />
    ```

---

### 2. 진짜 "업데이트 내용"은 어디에? (Dynamic Data)
가장 중요한 부분은 코드 하단의 **`<NuxtPage />`**입니다.

```vue
<section class="px-4 sm:px-6 xl:px-0 xl:-ms-30 xl:flex-1">
  <UColorModeButton class="fixed top-4 right-4 z-10" />

  <NuxtPage /> </section>
```

Nuxt에서 `<NuxtPage />`는 **"현재 주소에 맞는 다른 파일의 내용을 여기다 끼워 넣어라"**라는 뜻입니다. 즉, 실제 리스트 데이터는 별도의 파일에 저장되어 있습니다.

**지금 당장 프로젝트 폴더에서 아래 경로를 찾아보세요:**
1.  **`pages/index.vue`** (만약 지금 보시는 파일이 `app.vue`라면, `pages/index.vue`에 데이터가 있을 겁니다.)
2.  **`server/api/`** 또는 **`composables/`**: 만약 GitHub API를 연동해서 자동으로 가져오는 방식이라면, 서버 사이드 로직에서 데이터를 땡겨오고 있을 겁니다.

### 🛠️ 낭만 사냥꾼님의 실전 확인법

지금 유저님이 보여주신 파일 이름이 **`app.vue`** 인가요? 
만약 그렇다면, 실제 "내용물"은 **`pages/` 폴더 안에 있는 파일**들에 들어있습니다.

1.  **`pages/index.vue`** 파일을 열어보세요.
2.  거기에 `const releases = [...]` 같은 배열이 있거나, 마크다운을 불러오는 코드가 있다면 거기가 진짜 **데이터 본진**입니다.

**💡 힌트:** 이 템플릿 이름이 "Changelog"라면, 보통 GitHub의 **Releases** 정보를 API로 긁어와서 보여주는 경우가 많습니다. 그럴 경우엔 코드를 고치는 게 아니라 **`nuxt.config.ts`**나 **`.env`** 파일에 유저님의 GitHub 저장소 주소를 적어줘야 내용이 바뀝니다.

지금 폴더 구조에 **`pages`** 폴더가 보이시나요? 그 안의 파일 내용을 한 번 확인해 보세요! 🏹
