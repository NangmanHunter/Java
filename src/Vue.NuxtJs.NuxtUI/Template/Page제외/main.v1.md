##
Nuxt.js에서 특정 파일을 완전히 삭제하지 않으면서, 브라우저가 접근할 수 있는 **라우트(URL) 대상에서는 제외**하고 싶으신 거군요. 나중에 다시 복구할 가능성까지 고려했을 때 가장 깔끔한 방법들을 정리해 드립니다.

---

### 1. 파일 이름 앞에 언더바(`_`) 붙이기 (가장 권장)
Nuxt 3(및 2)의 파일 기반 라우팅 시스템은 파일 이름이 **언더바(`_`)**로 시작하면 해당 파일을 라우트 생성에서 제외합니다.

* **방법:** `pages/about.vue` → `pages/_about.vue`
* **장점:** * 파일이 원래 위치(`pages` 폴더)에 그대로 있어 나중에 찾기 쉽습니다.
    * 복구하고 싶을 때 언더바만 지우면 바로 페이지로 살아납니다.
    * 빌드 결과물에 포함되지 않아 깔끔합니다.

### 2. 별도의 보관용 폴더로 이동
`pages` 폴더 외부에 임시 저장용 폴더를 만들어 파일을 옮겨두는 방법입니다.

* **방법:** 프로젝트 루트에 `.backup`이나 `_ignored_pages` 같은 폴더를 만들고 거기로 파일을 이동시킵니다.
* **장점:** * `pages` 폴더가 지저분해지는 것을 방지할 수 있습니다.
    * `.gitignore`에 해당 폴더를 추가하면 Git 관리 대상에서도 제외할 수 있어 프로젝트 관리가 용이합니다.

### 3. `nuxt.config.ts`에서 무시 설정 (ignore)
파일 이름을 바꾸거나 옮기기 싫다면, 설정 파일에서 특정 패턴의 파일을 라우팅에서 제외하도록 명시할 수 있습니다.

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  srcDir: '.',
  // ignore 옵션을 사용해 특정 패턴 제외
  ignore: [
    'pages/temp-page.vue',
    'pages/unused-directory/**'
  ]
})
```

### 4. `app.config.ts`나 주석 처리 (비권장)
단순히 코드 전체를 `<template>` 안에서 주석 처리하는 방법도 있지만, 이 경우 **빈 페이지가 라우트에는 여전히 존재**하게 됩니다. 즉, 사용자가 해당 URL로 접속하면 아무것도 없는 화면을 보게 되므로 사용자 경험 측면에서는 좋지 않습니다.

---

### 요약 및 추천
나중에 다시 "긁어와서 쓰고 싶다"는 의도에 가장 부합하는 방식은 **1번(언더바 활용)**입니다.

| 방법 | 파일 위치 | 라우트 생성 여부 | 복구 난이도 |
| :--- | :--- | :--- | :--- |
| **이름 변경 (`_`)** | 유지 | **제외됨** | 매우 쉬움 (이름만 복구) |
| **폴더 이동** | 변경 | **제외됨** | 보통 (다시 옮겨야 함) |
| **nuxt.config 설정** | 유지 | **제외됨** | 보통 (설정값 삭제) |

**팁:** 만약 나중에 코드를 참고만 하고 싶고 파일 자체는 깔끔하게 관리하고 싶다면, `components/archive/` 같은 곳으로 옮겨두는 것도 방법입니다. 나중에 필요한 부분만 컴포넌트처럼 복사해서 쓸 수 있으니까요.
