##
이 **Changelog 템플릿**은 서비스의 업데이트 소식을 깔끔하게 보여주기에 최적화된 구성입니다.

백엔드 개발자 스타일로 비유하자면, 이 템플릿은 **"잘 짜여진 오픈소스 보일러플레이트(Boilerplate)"**입니다. 
단순히 참고만 할 수도 있고, 통째로 프로젝트의 기반으로 삼을 수도 있죠.



### 전략 1: 통째로 내려받아 "새 프로젝트"로 시작하기 (가장 쉬움)
만약 지금 만드는 기능이 오직 '공지사항'이나 '업데이트 기록' 페이지라면, 이 구조를 그대로 쓰는 게 가장 빠릅니다.

1.  **터미널에서 클론:**
    ```bash
    npx nuxi@latest init -t ui-templates/changelog my-changelog
    ```
    *이 명령어를 쓰면 GitHub에서 일일이 다운로드하지 않아도 Nuxt 공식 명령어로 템플릿만 쏙 뽑아올 수 있습니다.*
2.  **의존성 설치 및 실행:**
    ```bash
    cd my-changelog
    npm install
    npm run dev
    ```

---

### 전략 2: 현재 프로젝트에 "부분 이식"하기 (추천 🏹)
이미 유저님이 진행 중인 프로젝트가 있다면, 템플릿에서 **'아이디어'와 '컴포넌트'만 쏙쏙** 뽑아오는 방식입니다.

1.  **`content/` 폴더 훔쳐오기:** 이 템플릿은 `Nuxt Content` 모듈을 씁니다. `content/index.md` 파일을 열어보시면 마크다운으로 글을 쓰면 자동으로 웹페이지가 되는 구조를 배울 수 있습니다.
2.  **`app.config.ts` 설정 참고:** 템플릿의 `app.config.ts`를 열어서 `header`, `footer`, `socials` 설정을 어떻게 했는지 보고 유저님 프로젝트에 복사하세요.
3.  **UI 배치 배우기:** `app.vue`나 `pages/index.vue`를 열어서 `UContainer`, `UPage`, `ULandingSection` 같은 컴포넌트들을 어떤 순서로 배치했는지 눈으로 익히세요.

---

### 전략 3: "레이아웃"으로 활용하기 (아까 질문하신 내용)
이 템플릿의 디자인이 맘에 든다면, 특정 페이지(예: `/updates`)에만 이 느낌을 적용할 수 있습니다.

1.  템플릿의 `app.vue` 내용을 복사해서 유저님 프로젝트의 `layouts/changelog.vue`로 만듭니다.
2.  해당 레이아웃 안에 `<slot />`을 넣습니다.
3.  업데이트 페이지(`pages/updates.vue`)에서 `definePageMeta({ layout: 'changelog' })`를 선언합니다.

---

### 🧐 이 템플릿의 핵심 포인트 (백엔드 관점)
이 템플릿은 **`Nuxt Content`**라는 강력한 도구를 쓰고 있습니다. 
* **DB가 필요 없음:** `content/` 폴더에 `.md`(마크다운) 파일만 새로 만들면 페이지가 자동으로 생성됩니다. 
* **정적 생성:** 백엔드 API 호출 없이 빌드 타임에 파일이 생성되므로 속도가 미친 듯이 빠릅니다.
