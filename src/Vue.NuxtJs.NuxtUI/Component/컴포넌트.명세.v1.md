##
Nuxt UI의 명세(Documentation)는 공식 웹사이트에서 아주 상세하게 제공하고 있습니다. `UCard`처럼 스타일을 세밀하게 조정해야 할 때 어디를 봐야 하는지 딱 짚어드릴게요.

### 1. 공식 문서 주소
* **Nuxt UI 공식 문서:** [ui.nuxt.com](https://ui.nuxt.com/)
* **UCard 컴포넌트 페이지:** [ui.nuxt.com/components/card](https://ui.nuxt.com/components/card)

### 2. "명세"를 보는 법 (꿀팁)
문서에 들어가시면 상단 탭이나 우측 메뉴에 **"Config"** 또는 **"Preset"**이라는 섹션이 있습니다. 이게 진짜 중요한데, 여기서 해당 컴포넌트가 기본적으로 어떤 Tailwind 클래스들을 가지고 있는지 **'정체'**를 다 드러냅니다.

예를 들어, `UCard`의 기본 명세(Config)를 보면 대략 이렇습니다:
```json
{
  "base": "",
  "background": "bg-white dark:bg-gray-900",
  "divide": "divide-y divide-gray-200 dark:divide-gray-800",
  "ring": "ring-1 ring-gray-200 dark:ring-gray-800",
  "rounded": "rounded-lg",
  "shadow": "shadow",
  "body": {
    "base": "",
    "background": "",
    "padding": "px-4 py-5 sm:p-6"  // <--- 이 부분이 범인이었을 확률이 높아요!
  },
  "header": { "base": "", "padding": "px-4 py-5 sm:px-6" },
  "footer": { "base": "", "padding": "px-4 py-4 sm:px-6" }
}
```
> **왜 아까 에러가 났을까?** 명세를 보면 `body` 안에 `padding`이라는 키가 이미 정의되어 있습니다. 따라서 우리가 `:ui="{ body: { padding: 'p-4' } }"`라고 썼을 때, 명세에 정의된 타입과 완벽히 일치해야 빨간 줄이 안 생기는 구조입니다.

### 3. 더 빠르게 보는 법 (VS Code)
문서를 일일이 찾기 귀찮다면, VS Code에서 해당 컴포넌트에 마우스를 올리고 `Ctrl` (맥은 `Cmd`) 키를 누른 채 클릭해 보세요. 정의(Definition) 파일로 이동하는데, 거기서 `ui` 속성에 들어갈 수 있는 인터페이스(Type)를 직접 확인할 수 있습니다.

---

**정리하자면:**
1.  **전체 컴포넌트 목록:** [ui.nuxt.com/components](https://ui.nuxt.com/components)
2.  **스타일 커스텀 규칙:** 각 컴포넌트 페이지의 **Config** 섹션 확인

이제 명세를 보면서 `UTextarea`나 `UButton`도 낭만 있게 꾸며보실 수 있을 거예요. 혹시 명세 내용 중에 해석이 안 되거나 "이건 어떻게 바꾸라는 거지?" 싶은 부분이 있으면 바로 물어봐 주세요! :)
