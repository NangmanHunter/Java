많이 당황스러우셨죠? 결론부터 말씀드리면 **네, 원래 그렇습니다.** 😅 하지만 이건 Nuxt UI의 버그가 아니라, 그 안에 포함된 **Tailwind CSS의 설계 의도** 때문이에요.

왜 이런 현상이 발생하는지, 그리고 어떻게 해결하면 되는지 깔끔하게 정리해 드릴게요.

---

## 🧐 왜 h1 태그가 안 먹나요?

Nuxt UI는 내부적으로 **Tailwind CSS**를 사용합니다. Tailwind는 브라우저마다 제각각인 기본 스타일을 초기화하기 위해 **Preflight**라는 스타일 리셋 시트를 적용하는데요, 이때 다음과 같은 작업이 수행됩니다.

* `h1`, `h2`, `h3` 등 모든 제목 태그의 **폰트 크기(font-size)를 기본 본문 크기와 동일하게** 맞춥니다.
* **굵기(font-weight)를 보통(normal)**으로 초기화합니다.
* **상하 여백(margin)을 제거**합니다.

결과적으로 `<h1>`을 써도 일반 `<span>`이나 `<p>` 태그와 시각적으로 차이가 없게 느껴지는 것이죠.

---

## ✅ 어떻게 해결하나요?

가장 권장되는 방법은 Tailwind의 **Utility Class**를 직접 활용하는 것입니다.

### 1. 개별적으로 스타일링하기 (권장)
가장 빠르고 명확한 방법입니다. 원하는 크기와 굵기를 클래스로 직접 지정하세요.

```html
<h1 class="text-3xl font-bold">이것은 제목입니다</h1>
```

### 2. Nuxt UI의 Typography 활용
만약 긴 글(블로그 포스트 등)을 작성 중이라 일일이 클래스를 넣기 힘들다면, Tailwind CSS의 **Typography 플러그인**을 사용하는 것이 좋습니다. (Nuxt UI 프로젝트라면 이미 설정되어 있을 확률이 높습니다.)

```html
<article class="prose lg:prose-xl">
  <h1>여기서는 h1 태그가 기본 제목처럼 보입니다!</h1>
  <p>prose 클래스가 하위 HTML 태그에 표준 스타일을 입혀주거든요.</p>
</article>
```

### 3. 글로벌 스타일 설정 (비권장)
모든 `h1` 태그가 예전처럼 크게 나오길 원하신다면 `assets/css/main.css` 같은 파일에 직접 정의할 수도 있습니다. 하지만 이는 Tailwind의 철학(명시적 스타일링)과는 조금 어긋날 수 있어요.

```css
/* 추천하지는 않지만 해결은 가능합니다 */
h1 {
  font-size: 2rem;
  font-weight: bold;
}
```

---

**💡 요약하자면:**
Nuxt UI(Tailwind)는 개발자가 스타일을 완전히 통제할 수 있도록 **"백지 상태"**에서 시작하게 만듭니다. `text-2xl`, `font-bold` 같은 클래스를 적극적으로 활용해 보세요!
