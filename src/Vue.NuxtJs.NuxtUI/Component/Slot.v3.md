##
컴포넌트 슬롯(Slot)은 아주 쉽게 말해서 **"컴포넌트 안에 뚫어놓은 구멍"**입니다.

수학적으로 비유하자면, 값이 정해진 '상수'가 아니라 **함수 $f(x)$에서 $x$라는 자리를 비워두는 것**과 같습니다. $x$에 무엇을 넣느냐에 따라 결과값이 달라지듯, 슬롯에 무엇을 넣느냐에 따라 컴포넌트의 최종 모습이 달라집니다.

---

### 1. 슬롯이 필요한 이유 (문제 상황)

만약 우리가 버튼을 하나 만들었다고 칩시다. 그런데 어떤 때는 **글자**만 넣고 싶고, 어떤 때는 **아이콘**도 같이 넣고 싶습니다.

* **슬롯이 없다면:** `props`로 글자 따로, 아이콘 유무 따로... 사사건건 설정을 넘겨야 합니다. HTML 태그를 넘기기는 거의 불가능하죠.
* **슬롯이 있다면:** 그냥 버튼 태그 사이에 내가 넣고 싶은 걸 다 집어넣으면 됩니다.

---

### 2. 코드 예시 (부품 만들기 vs 가져다 쓰기)

#### 1단계: 부품 만들기 (`MyButton.vue`)
컴포넌트를 만드는 사람은 **"어디가 바뀔지"**를 고민해서 그 자리에 `<slot />` 태그를 적어줍니다.

```vue
<template>
  <button class="bg-blue-500 text-white p-2 rounded">
    <slot /> 
  </button>
</template>
```

#### 2단계: 가져다 쓰기 (`App.vue`)
컴포넌트를 쓰는 사람은 `<MyButton>` 태그 사이에 **원하는 HTML 조각**을 마음대로 넣습니다.

```vue
<template>
  <MyButton>
    그냥 확인 버튼
  </MyButton>

  <MyButton>
    <UIcon name="i-heroicons-check" />
    <span>완료했습니다!</span>
  </MyButton>
</template>
```



---

### 3. "가리키는 대상"의 물리적 흐름

이 부분이 헷갈리실 텐데, 딱 이렇게 정리하세요.

1.  **`<MyButton>` 과 `</MyButton>` 사이의 내용**: 이게 **재료**입니다.
2.  **`MyButton.vue` 내부의 `<slot />`**: 이게 **그릇**입니다.

사용자가 `<MyButton>` 태그 사이에 적은 모든 내용은, 컴포넌트 내부의 `<slot />` 태그가 있는 위치로 **순간이동**해서 렌더링됩니다.

---

### 4. 왜 레이아웃 슬롯이랑 헷갈릴까?

* **레이아웃의 `<slot />`**: "이 화면의 **메인 페이지 내용**이 들어올 커다란 구멍"입니다. (Nuxt가 `pages/` 폴더의 내용을 자동으로 채워줌)
* **컴포넌트의 `<slot />`**: "이 작은 부품의 **일부분**을 사용자가 취향껏 채울 작은 구멍"입니다. (개발자인 당신이 직접 태그 사이에 내용을 채움)
