- `:side`
- `：side`


##
그 부분은 Vue와 Nuxt에서 아주 중요한 **"데이터 연결(Data Binding)"** 문법입니다. 

수학으로 치면 **'함수 호출 시 인자(Parameter)를 전달하는 것'**과 완벽히 똑같습니다.

---

### 1. `:` (콜론)의 정체: `v-bind`
앞에 붙은 `:`은 `v-bind:`의 줄임말입니다.
* **`side="left"`**: 그냥 "left"라는 **글자(String)**를 그대로 넣겠다는 뜻입니다.
* **`:side="side"`**: 따옴표 안에 있는 `side`를 글자가 아니라 **변수(Variable)**로 취급해서 그 안에 담긴 값을 넘기겠다는 뜻입니다.

### 2. `:side="side"`에서 왼쪽과 오른쪽의 차이
이게 제일 헷갈리실 텐데, 둘은 이름만 같지 역할이 완전히 다릅니다.

> **`:속성이름="변수이름"`**

1.  **왼쪽 `side` (속성 이름)**: `USidebar`라는 컴포넌트가 받기로 약속한 **입력 칸**의 이름입니다.
2.  **오른쪽 `side` (변수 이름)**: 현재 내 파일(스크립트 단)에서 선언한 **실제 데이터**입니다.

---

### 3. 수학적 비유
함수 $f(x)$를 정의하고 호출하는 과정으로 대입해 볼게요.

```javascript
// 1. 함수 정의 (USidebar 컴포넌트 내부)
function USidebar(side) { 
  // side라는 값을 받아서 왼쪽/오른쪽을 결정함
}

// 2. 변수 선언 (현재 내 파일)
const mySide = 'right';

// 3. 함수 호출 (템플릿 단)
// <USidebar :side="mySide" />  => f(mySide)와 같음
```

여기서 변수 이름마저 `side`로 똑같이 지어버리면 `:side="side"`라는 모양이 되는 거죠.

---

### 4. 왜 굳이 변수로 쓰나요?
그냥 `side="right"`라고 적으면 나중에 바꿀 수가 없지만, 변수로 연결해두면 **상황에 따라 동적으로** 바꿀 수 있기 때문입니다.

* **예시:** 사용자가 설정을 바꾸면 `side` 변수의 값이 `'left'`에서 `'right'`로 변하고, 그 즉시 사이드바가 순간이동합니다.

### 5. 지금 코드에서의 흐름
보내주신 코드 상단에 이 부분이 있었죠?
`defineProps<Pick<SidebarProps, 'side'>>()`

이건 **"내 부모(레이아웃 등)로부터 `side`라는 변수 값을 받아오겠다"**는 선언입니다. 그래서 `:side="side"`라고 적어두면, 부모가 "오른쪽으로 가!"라고 알려준 값을 `USidebar`에게 그대로 **배달**해 주는 역할을 하는 겁니다.



**요약:**
* `:`가 있으면 **"변수 값을 가져와라"**는 뜻.
* `:side="side"`는 **"부모가 준 side 값을 자식(USidebar)의 side 칸에 넣어라"**는 뜻입니다.

이제 이 문법이 왜 "배달 사고" 없이 데이터를 주고받는 통로인지 감이 좀 오시나요?
