##
일상적인 대화에서는 두 단어를 섞어서 쓰기도 하지만, 프로그래밍(특히 자바) 세계에서는 **시점과 목적**에 따라 명확히 구분됩니다.

아주 쉽게 비유하자면 이렇습니다.
* **초기화(Initialization):** 빈 집에 **처음으로** 가구를 들여놓는 것 (생애 첫 데이터 입력)
* **대입(Assignment):** 살고 있는 집에 가구를 **바꾸거나 새로 넣는 것** (데이터 덮어쓰기나 추가)

---

### 1. 초기화 (Initialization)
변수가 태어나서 **처음으로** 값을 갖게 되는 순간을 말합니다.
* **자바의 특징:** 자바는 "초기화되지 않은 변수"는 위험하다고 판단해서 읽는 것을 막습니다.
* **전역 변수:** `int a;`라고만 적어도 자바가 몰래 `0`을 집어넣습니다. (이게 **자동 초기화**)
* **지역 변수:** 자바가 자동으로 안 해주기 때문에, 개발자가 직접 `int a = 10;` 또는 `a = 10;`을 해서 첫 값을 넣어줘야 합니다. (이게 **명시적 초기화**)

### 2. 대입 (Assignment)
이미 만들어진 변수에 값을 **할당**하는 모든 행위를 말합니다.
* 초기화도 대입의 한 종류입니다. (첫 번째 대입 = 초기화)
* 하지만 초기화 이후에 값을 바꾸는 것은 그냥 '대입' 또는 '재할당'이라고 부릅니다.

---

### 3. 코드 영역과 데이터 영역의 관점에서 보기



이전에 우리가 배운 메모리 구조와 연결하면 더 명확해집니다.

```java
int a;    // 1. 메모리에 공간 확보 (이름표만 붙임)
a = 10;   // 2. 초기화 (비어있던 공간에 첫 데이터를 채움)
a = 20;   // 3. 대입 (이미 있던 10을 지우고 20으로 덮어씀)
```

1.  **Data/BSS/Stack 영역:** `int a;`를 하는 순간 메모리 어딘가에 4바이트짜리 땅(주소)이 배정됩니다.
2.  **초기화 시점:** 그 땅에 처음으로 `10`이라는 기계어 비트가 채워집니다. 이제 이 변수는 "살아있다"라고 간주되어 다른 코드에서 읽어갈 수 있습니다.
3.  **대입 시점:** 이미 데이터가 들어있는 그 주소로 찾아가서 값을 `20`으로 바꿉니다.

### 4. 왜 구분하는 게 중요한가요?

특히 **`final`** 키워드를 쓸 때 이 차이가 극명하게 드러납니다.

```java
final int b; // 선언
b = 100;     // '초기화' (첫 대입이라 허용됨)
// b = 200;  // 에러! (초기화가 이미 끝난 변수에 '대입'을 시도했기 때문)
```
