##
Spring Boot 환경에서 회원가입을 '승인제'로 운영하고 싶을 때, 기술적으로 **"가로채는 지점"**과 **"승인 워크플로우"**에 대한 몇 가지 아이디어를 제안해 드립니다.

---

## 1. 데이터베이스 상태값을 이용한 제어 (가장 권장)
가입 요청 자체를 막는 것이 아니라, 가입 직후의 **상태(Status)**를 가로채는 방식입니다.

* **구현 방법:** * `Member` 엔티티에 `status` 필드(예: `PENDING`, `ACTIVE`, `REJECTED`)를 추가합니다.
    * 회원가입 시 기본값을 `PENDING`으로 저장합니다.
* **가로채는 지점:** **`UserDetailsService` (Spring Security)**
    * 로그인 시점에 DB에서 사용자를 조회할 때, `status`가 `ACTIVE`가 아니면 `DisabledException`이나 `LockedException`을 던지도록 설정합니다.
* **장점:** 가입 폼은 열어두되 실제 서비스 이용은 관리자가 승인(DB 상태 변경)한 후에만 가능하게 하므로 관리가 유연합니다.



## 2. '가입 코드(Invitation Code)' 방식
아무나 가입 버튼을 누르지 못하게 입구에서부터 가로채는 방식입니다.

* **구현 방법:** * 관리자가 미리 난수 형태의 '초대 코드'를 생성하여 DB에 저장해둡니다.
    * 회원가입 폼에 `joinCode` 입력란을 추가합니다.
* **가로채는 지점:** **`Service Layer` (Validation)**
    * `join()` 메서드 시작 부분에서 입력받은 코드가 유효한지, 이미 사용된 것은 아닌지 체크합니다.
* **장점:** 로직이 단순하고, 허락된 사람(코드를 전달받은 사람)만 가입 시도가 가능합니다.

## 3. Spring Security Filter를 이용한 가로채기
가입 요청 API 자체를 물리적으로 차단하고 싶을 때 사용합니다.

* **가로채는 지점:** **`OncePerRequestFilter` 커스텀 필드**
    * `/api/signup` 경로로 들어오는 POST 요청을 필터에서 먼저 가로챕니다.
    * 요청 헤더나 파라미터에 관리자가 발급한 임시 토큰이 포함되어 있지 않으면 바로 403 Forbidden을 응답합니다.
* **장점:** 비즈니스 로직(Service)에 도달하기 전에 보안 계층에서 미리 걸러낼 수 있습니다.

## 4. 관리자 전용 '회원 생성' API
회원가입 폼 자체를 없애는 방식입니다.

* **구현 방법:** * 일반 사용자가 접근 가능한 `/signup` 페이지를 만들지 않습니다.
    * 대신 관리자 권한(`ROLE_ADMIN`)만 접근 가능한 `/admin/create-member` API를 만듭니다.
* **워크플로우:** 1. 사용자가 이메일 등으로 가입 요청을 보냄.
    2. 관리자가 관리자 페이지에서 해당 정보를 입력해 계정을 직접 생성함.
    3. 시스템이 사용자에게 임시 비밀번호를 메일로 발송함.

---

### 추천하는 구성
만약 혼자 사용하거나 소규모로 운영할 계획이라면 **1번(상태값 제어) + 이메일 알림** 조합을 추천합니다.

1.  사용자가 가입을 완료하면 DB에 `ENABLED = false` 상태로 저장.
2.  가입과 동시에 관리자(본인)에게 "가입 요청이 들어왔습니다"라고 **메일이나 슬랙 알림**이 오게 설정.
3.  관리자 페이지에서 '승인' 버튼을 누르면 `ENABLED = true`로 변경.
4.  Spring Security의 `UserDetails` 구현체에서 `isEnabled()`가 이 값을 반환하게 하여 로그인 차단.

어떤 방식이 현재 만드시는 프로젝트 성격에 가장 잘 맞을 것 같나요? 구체적인 코드 예시가 필요하면 말씀해 주세요.
