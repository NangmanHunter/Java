##
이 방식은 서버가 켜질 때마다 **"혹시 관리자 계정 없니? 그럼 내가 만들어줄게!"**라고 알아서 체크해주기 때문에 개발 단계에서 정말 편리합니다.

만드는 방법은 아주 간단합니다. 프로젝트 구조 내에 적당한 패키지를 정해서 클래스를 생성하시면 됩니다.

---

### 1. 클래스 생성 위치
보통 `global.config` 또는 `global.init` 같은 패키지를 만들어 관리합니다.

*   **경로 예시:** `src/main/java/your/project/global/init/AdminInitializer.java`

### 2. 코드 작성 (복사해서 바로 쓰셔도 됩니다)

이 코드가 작동하려면 `Member` 엔티티에 `Role` 필드와 `isApproved` 필드가 있어야 합니다.

```java
package your.project.package.global.init; // 본인의 패키지 경로에 맞게 수정하세요

import lombok.RequiredArgsConstructor;
import org.springframework.boot.CommandLineRunner;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.stereotype.Component;
import your.project.package.domain.member.entity.Member;
import your.project.package.domain.member.entity.Role; // Role이 Enum일 경우
import your.project.package.domain.member.repository.MemberRepository;

@Component // 스프링이 서버 뜰 때 이 클래스를 찾아내게 합니다
@RequiredArgsConstructor
public class AdminInitializer implements CommandLineRunner {

    private final MemberRepository memberRepository;
    private final PasswordEncoder passwordEncoder;

    @Override
    public void run(String... args) {
        // 1. 이미 admin이라는 계정이 있는지 확인 (중복 생성 방지)
        if (memberRepository.findByLoginId("admin").isEmpty()) {
            
            // 2. 관리자 계정 생성 (엔티티 구조에 맞게 builder 채우기)
            Member admin = Member.builder()
                .loginId("admin")
                .password(passwordEncoder.encode("1234")) // 비밀번호는 반드시 암호화!
                .name("최고관리자")
                .role(Role.ADMIN) // 가입 시 Role.ADMIN 권한 부여
                .isApproved(true) // 관리자는 본인이 본인을 승인한 셈이니 true
                .build();

            memberRepository.save(admin);
            System.out.println("★ 초기 관리자 계정(admin)이 생성되었습니다. ★");
        }
    }
}
```

---

### 3. 작동 원리 (왜 여기서 만드나요?)

*   **`@Component`**: 스프링 부트가 시작될 때 "어? 여기 내가 관리해야 할 컴포넌트가 있네?" 하고 자동으로 객체를 생성합니다.
*   **`CommandLineRunner`**: 이 인터페이스를 구현(implements)하면, 스프링 부트가 모든 설정(DB 연결, 시큐리티 등)을 다 마친 직후에 딱 한 번 `run()` 메서드를 실행해줍니다.
*   **`passwordEncoder`**: 시큐리티 설정에서 등록해두신 `BCryptPasswordEncoder` 등을 주입받아 사용합니다. DB에 생으로 `1234`를 넣으면 시큐리티가 로그인을 거부하기 때문에 꼭 필요합니다.

### 💡 주의사항 & 팁

1.  **로그인 아이디 확인:** `memberRepository`에 `findByLoginId`라는 메서드가 미리 정의되어 있어야 합니다. (없다면 `MemberRepository` 인터페이스에 추가해 주세요!)
2.  **데이터 소스:** 만약 실무 서버(운영 환경)로 넘어갈 때는 이 코드를 지우거나, `@Profile("dev")` 같은 어노테이션을 붙여서 **개발용**으로만 돌아가게 하는 것이 안전합니다. 안 그러면 운영 서버 뜰 때마다 매번 체크하게 되니까요.
3.  **Member 엔티티 수정:** 혹시 엔티티에 `isApproved` 필드가 아직 없다면, 지금 바로 추가해 주세요. 그래야 관리자가 나중에 "승인"을 해줄 수 있습니다.
