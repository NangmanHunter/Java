
## 관리자기능
- 상태 관리(Pinia/useState)
- 라우터 미들웨어(Middleware)





##
Nuxt3 환경에서 관리자 기능을 구현하려면 **"상태 관리(Pinia/useState)"**와 **"라우터 미들웨어(Middleware)"**를 조합하는 것이 정석입니다.

이 방식은 단순히 버튼만 숨기는 게 아니라, 주소창에 직접 `/admin`을 치고 들어오는 행위까지 타이트하게 막아줍니다.

---

### 1. 로그인 응답 데이터 저장 (로그인 컴포넌트)

로그인이 성공하면 서버가 주는 JSON 응답(`role` 포함)을 받아 저장합니다. 여기서는 간단하게 `useState`를 사용하겠습니다. (더 규모가 커지면 **Pinia**를 추천합니다.)

```javascript
// composables/useAuth.js (전역 상태 관리)
export const useAuth = () => {
  const user = useState('user', () => null) // { loginId: string, role: string }

  const setAuth = (userData) => {
    user.value = userData
  }

  const clearAuth = () => {
    user.value = null
  }

  return { user, setAuth, clearAuth }
}
```

**로그인 로직 부분:**
```javascript
const { setAuth } = useAuth()

const handleLogin = async () => {
  const data = await $fetch('http://localhost:8080/api/auth/login', {
    method: 'POST',
    body: { loginId: '...', password: '...' }
  })
  
  // 서버에서 받은 { role: "ROLE_ADMIN", ... } 저장
  setAuth(data)
}
```

---

### 2. 관리자 버튼 노출 제어 (GNB/내비게이션 바)

화면 상단 메뉴에서 `user.role`이 `ROLE_ADMIN`일 때만 버튼을 보여줍니다.

```vue
<template>
  <nav>
    <NuxtLink to="/">홈</NuxtLink>
    
    <!-- 관리자만 보이는 낭만 버튼 -->
    <NuxtLink 
      v-if="user?.role === 'ROLE_ADMIN'" 
      to="/admin" 
      class="admin-btn"
    >
      관리자 콘솔
    </NuxtLink>
  </nav>
</template>

<script setup>
const { user } = useAuth()
</script>
```

---

### 3. 경로 보호 (Nuxt Middleware) - 중요! ⭐

버튼을 숨겨도 주소창에 `localhost:3000/admin`을 직접 치면 들어갈 수 있습니다. 이를 방지하기 위해 **미들웨어**를 만듭니다.

**`middleware/admin-only.ts` 생성**
```typescript
export default defineNuxtRouteMiddleware((to, from) => {
  const { user } = useAuth()

  // 1. 로그인이 안 되어 있거나
  // 2. 권한이 ROLE_ADMIN이 아니면
  if (!user.value || user.value.role !== 'ROLE_ADMIN') {
    // 홈으로 튕겨내거나 에러 페이지로 보냄
    alert('관리자 권한이 필요합니다.')
    return navigateTo('/')
  }
})
```

---

### 4. 관리자 페이지에 미들웨어 적용

관리자 관련 페이지들(예: `pages/admin/index.vue`) 상단에 방금 만든 미들웨어를 선언합니다.

```vue
<!-- pages/admin/index.vue -->
<script setup>
definePageMeta({
  middleware: 'admin-only'
})
</script>

<template>
  <div>
    <h1>관리자 전용 대시보드</h1>
    <p>여기는 ROLE_ADMIN만 들어올 수 있는 낭만 구역입니다.</p>
  </div>
</template>
```

### 💡 한 걸음 더: 새로고침 대응

Nuxt(SSR) 특성상 새로고침을 하면 `useState`에 담긴 유저 정보가 날아갑니다. 이를 방지하기 위해 보통 다음 두 가지를 병행합니다.

1.  **쿠키 사용:** `role` 정보를 쿠키에 저장해두고 앱이 로드될 때 다시 읽어옵니다.
2.  **초기 인증 체크:** `app.vue`나 `plugin`에서 서버의 `/api/auth/me`(현재 로그인 정보 확인 API)를 호출하여 상태를 복구합니다.

### 🛡️ 다시 강조하는 보안 원칙
프론트엔드 미들웨어가 막아주는 건 **사용자 경험(UX)** 차원의 방어입니다. 설령 이 미들웨어를 뚫고 들어오더라도, 유저님이 서버(Spring)에 설정한 `.requestMatchers("/api/admin/**").hasRole("ADMIN")`이 **최종 판결**을 내리므로 데이터가 유출될 걱정은 안 하셔도 됩니다.

이렇게 구성하면 아주 깔끔하고 타이트한 관리자 시스템이 완성됩니다! :)
