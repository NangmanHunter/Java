- [x] 게시글페이지


##
보통 로그아웃 후 이동하는 페이지는 **서비스의 성격**에 따라 결정됩니다. 정해진 정답은 없지만, 가장 많이 사용하는 패턴 2가지를 비교해 드릴게요.

---

### 1. 메인 페이지 (홈)로 이동
가장 보편적인 방식입니다. 사용자가 로그아웃을 했다고 해서 서비스를 아예 떠나는 것이 아니라, 비회원 상태로 콘텐츠를 계속 구경할 수 있게 배려하는 것입니다.

* **주요 사용처:** 커뮤니티(게시판), 쇼핑몰, 뉴스 사이트 등
* **장점:** 서비스의 첫 화면을 보여줌으로써 이탈률을 줄이고 다시 로그인을 유도할 수 있습니다.
* **사용 예시:** `MapsTo('/')`

### 2. 로그인 페이지로 이동
보안이 중요하거나, 로그인하지 않으면 아예 기능을 쓸 수 없는 서비스에서 주로 선택합니다.

* **주요 사용처:** 사내 시스템(ERP/인트라넷), 뱅킹 서비스, 개인 메모장 등
* **장점:** 로그아웃이 확실히 되었음을 시각적으로 바로 알 수 있고, 다시 로그인해야 한다는 목적을 명확히 전달합니다.
* **사용 예시:** `MapsTo('/login')`

---

### 💡 추천하는 방식 (사용자 경험 중심)

현재 **게시판 모듈**을 만들고 계시다면, 저는 **메인 페이지(`/`)**나 **게시판 목록(`/board`)**으로 보내는 것을 추천합니다.

* 사용자가 로그아웃을 하더라도 게시글 목록은 읽을 수 있는 경우가 많기 때문입니다.
* 만약 로그아웃 후에 바로 로그인 페이지로 튕겨나가면, 사용자는 "내가 보던 글이 어디 갔지?"라고 당황할 수 있습니다.

> **작은 팁 (UX 개선):** > 로그아웃 후 메인으로 보낼 때 상단에 **"로그아웃되었습니다."**라는 작은 안내 메시지(Toast 알림 등)를 띄워주면 사용자가 훨씬 친절하다고 느낍니다.

지금 만드시는 프로젝트는 로그아웃 후에 사용자가 어떤 화면을 보는 게 더 자연스러울까요? 현재 흐름상으로는 `MapsTo('/')`가 가장 무난해 보이긴 합니다! 

---
**구현 시 주의사항:**
로그아웃 후 페이지를 이동시킬 때, 브라우저 메모리에 남아있을 수 있는 **사용자 정보(전역 상태)**를 완전히 초기화하는 것을 잊지 마세요! (앞서 말씀드린 `isLoggedIn.value = false` 처리)

```javascript
// 추천하는 로그아웃 마무리
isLoggedIn.value = false;
await navigateTo('/'); // 또는 '/board'
```
