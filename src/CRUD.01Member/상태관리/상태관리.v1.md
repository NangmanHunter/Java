Nuxt.js에서 상태 관리(State Management)는 애플리케이션의 여러 컴포넌트 간에 공유되는 데이터를 효율적으로 관리하는 것을 의미합니다. 쉽게 말해, **"어떤 데이터를 한곳에 저장해두고, 필요한 모든 곳에서 꺼내 쓰거나 수정할 수 있게 하는 시스템"**입니다.

Nuxt 3를 기준으로 핵심 개념들을 정리해 드릴게요.

---

### 1. 왜 상태 관리가 필요한가요?
일반적으로 부모-자식 컴포넌트 사이에는 `props`와 `emit`을 통해 데이터를 주고받습니다. 하지만 서비스가 커지면 다음과 같은 문제가 생깁니다.
*   **Prop Drilling:** 아주 먼 조상 컴포넌트의 데이터를 증손자 컴포넌트에 전달하기 위해 중간 컴포넌트들이 의미 없는 전달자 역할을 해야 함.
*   **데이터 동기화:** 여러 화면에서 같은 사용자 정보(프로필 이미지 등)를 보여줄 때, 한곳에서 수정하면 모든 곳이 동시에 바뀌어야 함.

---

### 2. Nuxt 3의 기본: `useState`
Nuxt 3에는 별도의 라이브러리 설치 없이 사용할 수 있는 **`useState`**라는 내장 컴포저블이 있습니다. 이는 SSR(서버 사이드 렌더링) 환경에 최적화된 상태 관리 도구입니다.

*   **특징:** 서버에서 클라이언트로 데이터가 전달될 때 상태가 유지(hydration)됩니다.
*   **사용법:**
    ```javascript
    // composables/states.ts (공통 저장소로 활용)
    export const useCounter = () => useState<number>('counter', () => 0)
    ```



---

### 3. 대세는 Pinia (피니아)
현재 Nuxt와 Vue 생태계에서 가장 권장되는 상태 관리 라이브러리는 **Pinia**입니다. 과거의 Vuex를 대체하며, 훨씬 가볍고 TypeScript 지원이 강력합니다.

#### **Pinia의 3대 요소:**
1.  **State:** 실제 데이터 (예: `user: { name: 'Nuxtie' }`)
2.  **Getters:** 데이터를 계산해서 가져올 때 (Computed와 유사, 예: `isLoggedIn`)
3.  **Actions:** 데이터를 변경하는 로직이나 비동기 작업 (예: `login()`, `fetchData()`)

---

### 4. 언제 무엇을 쓸까요?

| 구분 | `useState` | Pinia |
| :--- | :--- | :--- |
| **규모** | 소규모, 단순한 상태 공유 | 중대규모, 복잡한 로직 |
| **설치** | 내장 (설치 필요 없음) | 별도 설치 필요 |
| **복잡성** | 매우 단순함 | 구조적임 (State, Action 등 구분) |
| **추천 상황** | 단순한 UI 토글, 테마 변경 등 | 장바구니, 유저 인증, 게시판 데이터 등 |
