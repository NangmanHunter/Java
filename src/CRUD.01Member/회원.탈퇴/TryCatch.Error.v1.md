JavaScript의 `try...catch` 구문에서 `catch`가 "어떤 에러를 잡느냐"는 질문은 아주 좋은 포인트입니다. 보통 **네트워크 요청(API 호출)** 상황과 연결해서 생각하게 되는데, 결론부터 말씀드리면 **JS 엔진 입장에서는 '예외(Exception)'가 발생했느냐 아니냐**가 기준입니다.

---

## 1. `try...catch`의 기본 규칙
`try...catch`는 기본적으로 **런타임 에러**가 발생했을 때 작동합니다. 
* 문법 오류, 존재하지 않는 변수 참조, 함수가 아닌 것을 호출할 때 등 '코드 실행이 불가능한 상황'에서 `catch`로 넘어갑니다.
* **HTTP 상태 코드(300, 400, 500번대)와는 직접적인 상관이 없습니다.**

## 2. HTTP 상태 코드와 `catch` (fetch vs axios)
많은 분이 헷갈려 하시는 부분인데, 어떤 라이브러리를 쓰느냐에 따라 300~500번대 응답을 `catch`에서 잡을지 말지가 결정됩니다.

### ① `fetch` API를 쓸 때 (브라우저 기본)
`fetch`는 꽤 관대합니다. **404(Not Found)나 500(Internal Server Error)이 발생해도 `catch`로 가지 않습니다.**
* **성공으로 간주:** 200번대뿐만 아니라 300, 400, 500번대도 일단 '응답'이 왔으므로 `try` 블록 안에서 정상 처리됩니다.
* **실패(`catch`)로 가는 경우:** 네트워크 연결 자체가 끊겼거나, DNS 설정 오류 등 아예 서버에 닿지 못했을 때만 `catch`로 이동합니다.

### ② `axios` 라이브러리를 쓸 때
`axios`는 조금 더 엄격합니다.
* **성공:** 200번대(정확히는 2xx) 응답일 때만 `try`에서 처리됩니다.
* **실패(`catch`):** **300번대(리다이렉트 설정에 따라 다름), 400번대, 500번대** 응답이 오면 자동으로 에러라고 판단하여 `catch` 블록으로 던집니다.

---

## 3. 요약 및 규칙 테이블

| 상황 | `try` 블록 유지 (성공 처리) | `catch` 블록 이동 (에러 처리) |
| :--- | :--- | :--- |
| **순수 JS 코드** | 에러 없는 정상 코드 실행 | 변수 오타, 타입 에러 등 발생 시 |
| **`fetch` 호출** | **200, 300, 400, 500번대 모두** | 네트워크 단절, 주소 오타 등 |
| **`axios` 호출** | **200번대만** | **400, 500번대 등 (설정에 따라 300 포함)** |

---

## 💡 팁: 300번대는 왜 안 잡힐까?
질문하신 **300번대(Redirection)**는 보통 브라우저가 알아서 처리합니다. 예를 들어 서버가 "이 주소 말고 저 주소로 가!"라고 301 응답을 주면, 브라우저가 자동으로 새 주소로 다시 요청을 보내기 때문에 최종적으로는 200번대 성공 결과값을 받게 되는 경우가 많습니다. 그래서 `catch`에 걸릴 일이 거의 없는 것이죠.

만약 `fetch`를 쓰면서 400, 500번대 에러를 따로 처리하고 싶다면 아래처럼 수동으로 체크해줘야 합니다.

```javascript
try {
  const response = await fetch('/api/data');
  
  if (!response.ok) { // 200~299가 아니면 실행
    throw new Error(`상태 코드 에러: ${response.status}`);
  }
  
  const data = await response.json();
} catch (error) {
  console.log("여기서 에러를 잡아요:", error.message);
}
```
