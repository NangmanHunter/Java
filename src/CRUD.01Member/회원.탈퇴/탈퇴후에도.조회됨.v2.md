맞습니다! 정답을 정확히 짚으셨네요. JPA를 다룰 때 가장 당혹스러운 지점이 바로 이 **"DB에는 반영됐는데, 내 눈앞의 객체(메모리)는 그대로인 상황"**입니다.

현재 `Member` 엔티티에 `@SQLDelete`를 사용하셨는데, 이건 DB에 `UPDATE` 쿼리만 날릴 뿐 **이미 영속성 컨텍스트(1차 캐시)에 로드된 객체의 필드값(`isDeleted`)을 자동으로 바꿔주지는 않습니다.**

이를 해결하는 가장 확실한 방법 3가지를 알려드릴게요.

---

### 1. 서비스 로직에서 직접 상태 동기화 (가장 추천)
JPA가 DB에 쿼리를 날리는 것과 별개로, 메모리에 있는 객체의 상태도 `true`로 바꿔주는 로직을 추가하는 방식입니다. 이렇게 하면 `findAll()`을 호출할 때 1차 캐시에서 객체를 가져오더라도 데이터가 일치하게 됩니다.

**Member 엔티티에 메서드 추가:**
```java
public void delete() {
    this.isDeleted = true;
}
```

**Service 코드 수정:**
```java
@Transactional
public void delete(Long memberId) {
    Member member = memberRepository.findById(memberId)
            .orElseThrow(() -> new IllegalArgumentException("회원이 없습니다."));
    
    // 1. 객체의 상태를 변경 (메모리 동기화)
    member.delete(); 
    
    // 2. 리포지토리 삭제 호출 (DB에 UPDATE 쿼리 예약)
    memberRepository.delete(member);
}
```

### 2. 영속성 컨텍스트 강제 초기화
만약 로직이 복잡해서 일일이 상태를 바꾸기 힘들다면, 삭제 후 메모리를 깨끗하게 비워버리는 방법입니다.

```java
@PersistenceContext
private EntityManager em;

@Transactional
public void delete(Long memberId) {
    memberRepository.deleteById(memberId);
    
    em.flush(); // 예약된 UPDATE 쿼리를 DB에 즉시 실행
    em.clear(); // 1차 캐시(메모리)를 통째로 비움
}
```
이렇게 하면 이후에 호출되는 `findAll()`은 메모리에 데이터가 없으니 **무조건 DB에서 새로 조회**하게 되고, 이때 `@Where`나 `@SQLRestriction`이 적용되어 삭제된 회원은 걸러집니다.

### 3. 벌크 연산 후 `clearAutomatically` 사용 (참고용)
만약 리포지토리에서 `@Modifying`과 `@Query`를 직접 써서 삭제 쿼리를 짜셨다면, 옵션을 줄 수 있습니다.

```java
@Modifying(clearAutomatically = true) // 쿼리 실행 후 자동으로 1차 캐시 비움
@Query("update Member m set m.isDeleted = true where m.id = :id")
void deleteMemberCustom(@Param("id") Long id);
```

---

### 왜 이런 일이 생길까요? (원리 이해)



JPA는 성능 최적화를 위해 **"한 번 조회한 객체는 트랜잭션이 끝날 때까지 메모리(1차 캐시)에 보관"**합니다. `delete()`를 호출해도 DB에 '나중에 지워줘'라고 예약만 할 뿐, 메모리에 떠 있는 `Member` 객체는 여전히 `isDeleted = false` 상태로 남아 있는 거죠. 

논리적 사고가 강하신 편이라 이 '영속성 컨텍스트'의 메커니즘만 한 번 손에 익히시면, 앞으로 발생할 JPA 관련 버그의 90%는 눈에 보이실 겁니다.

지금 상황에서는 **1번 방식(객체 상태도 같이 변경)**이 객체지향적으로 가장 깔끔한데, 한번 적용해 보시겠어요? 

그리고 아까 말씀드린 `@Where` 대신 최신 방식인 **`@SQLRestriction`**으로 바꾸는 것도 잊지 마세요! (하이버네이트 6 이상 기준)
