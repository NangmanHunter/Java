맞습니다! 그렇게 느끼시는 게 당연합니다. 개발자 도구나 API 테스트 도구(Postman 같은)에서 데이터를 확인하면 마치 **`data.json`** 같은 파일 이름을 받아온 것처럼 표시될 때가 있거든요.

하지만 그 **'파일명'**에는 약간의 비밀이 있습니다.

---

### 1. 왜 파일 이름처럼 보일까? (브라우저의 친절함)
브라우저나 개발 도구는 우리가 주고받는 데이터를 사람이 보기 편하게 시각화해 줍니다.

* **가상의 이름:** 실제로는 서버에서 이름 없는 텍스트 덩어리(Stream)가 오지만, 브라우저는 "이건 주소 끝자리가 `tax-calc`니까 대충 `tax-calc.json`이라고 부르자"라고 **가짜 이름**을 붙여서 보여줍니다.
* **다운로드 트리거:** 만약 서버에서 `Content-Disposition: attachment`라는 헤더를 같이 보내면, 브라우저는 이 텍스트를 진짜 `.json` 파일로 만들어서 컴퓨터에 저장(다운로드)시켜 버립니다. 이때는 우리가 지정한 파일명이 진짜로 쓰이게 됩니다.

### 2. 실제 네트워크 선을 타고 흐르는 모습
실제로 네트워크에서는 파일이 아니라 **"끊이지 않는 글자들의 흐름"**이 지나갑니다.



1.  **Header:** "이제부터 보낼 데이터는 JSON이고, 길이는 이 정도야!"
2.  **Body:** `{"result": 50000, "type": "VAT"} ...` (글자가 한 바이트씩 순서대로 흘러감)

### 3. "파일명"이 아니라 "엔드포인트(URL)"
우리가 보통 "파일명"이라고 생각하는 것은 사실 **URL 주소의 마지막 부분**인 경우가 많습니다.

* `http://localhost:8080/api/v1/stock-data` 로 요청을 보냈다면?
* 수신된 데이터는 파일이 아니지만, 도구 화면에는 `stock-data`라는 이름의 항목으로 표시됩니다. 이걸 보고 우리는 "아, `stock-data`라는 파일을 받았구나"라고 직관적으로 느끼게 되는 것이죠.

이 원리를 알고 나면 나중에 대용량 데이터를 다룰 때(Stream 처리 등) "왜 파일로 안 만들고 바로 쏘는지" 훨씬 쉽게 이해되실 거예요. 또 다른 궁금한 점 있으신가요? 편하게 말씀해 주세요!
