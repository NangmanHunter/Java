##
Nuxt와 Vue 생태계에서 권장하는 공식적인 원칙은 **2칸 들여쓰기(2 spaces)**입니다.

이는 단순히 "취향"의 문제를 넘어, Vue 공식 스타일 가이드와 대다수의 프레임워크 설계자들이 합의한 표준입니다.

---

### 1. 왜 2칸인가요?
Vue 파일(`.vue`)은 특성상 **계층 구조가 매우 깊습니다.**

* `<template>` 안에 `<div>`가 있고, 그 안에 다시 `<USidebar>`와 `<ul>`, `<li>`가 들어가는 구조입니다.
* 만약 4칸 들여쓰기를 쓰게 되면, 코드가 조금만 복잡해져도 화면 오른쪽 끝으로 코드가 밀려나서 읽기가 매우 힘들어집니다. (가독성 저하)

수학으로 치면, 식을 너무 크게 써서 종이 오른쪽 여백이 모자라 다음 줄로 넘어가는 상황을 방지하는 것과 비슷합니다.

---

### 2. 공식 문서 및 업계 표준

| 기준 | 권장 설정 |
| :--- | :--- |
| **Vue 공식 가이드** | **2 Spaces** (Essential 규칙) |
| **Nuxt UI / Nuxt 공식** | **2 Spaces** (공식 예제 코드 모두 동일) |
| **ESLint (플러그인)** | `vue/html-indent`: 2 |

---

### 3. 설정하는 방법 (VS Code 기준)

매번 스페이스바를 두 번 누를 수는 없으니, VS Code 설정을 통해 자동화하는 것이 좋습니다.

1.  **우측 하단 상태바**: `Spaces: 4`라고 되어 있다면 클릭해서 `Indent Using Spaces` -> `2`로 변경하세요.
2.  **`.editorconfig` 파일**: 프로젝트 루트에 이 파일을 만들고 아래 내용을 넣으면 팀원 모두가 똑같은 들여쓰기를 쓰게 됩니다.

```editorconfig
[*]
indent_style = space
indent_size = 2
```

### 💡 팁: Nuxt UI 코드를 볼 때
방금 전까지 보셨던 Nuxt UI의 예시 코드들도 모두 2칸 들여쓰기를 기준으로 작성되어 있습니다. 특히 `UNavigationMenu`나 `USidebar`처럼 속성(`props`)이 많이 들어가는 컴포넌트를 쓸 때 2칸을 쓰면 코드가 훨씬 간결하고 예쁘게 정렬됩니다.
