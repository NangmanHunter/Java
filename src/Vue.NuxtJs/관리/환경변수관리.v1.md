##
Nuxt에서 `runtimeConfig`를 사용해 API Base URL을 관리하는 게 처음엔 조금 복잡해 보일 수 있습니다. 하지만 한 번 구조를 잡아두면 환경별(로컬, 스테이징, 운영)로 관리하기가 매우 편해집니다.

가장 깔끔하고 표준적인 관리 방법을 단계별로 정리해 드릴게요.

---

### 1. `.env` 파일 설정

먼저 프로젝트 루트에 환경 변수 파일을 만듭니다. 보안이나 환경별 전환을 위해 직접적인 URL은 여기에 적는 것이 원칙입니다.

```bash
# .env
NUXT_PUBLIC_API_BASE_URL=https://api.example.com

```

> **Tip:** `NUXT_PUBLIC_`으로 시작하면 Nuxt가 자동으로 인식해서 `runtimeConfig.public`으로 매핑해줍니다.

---

### 2. `nuxt.config.ts` 설정

이제 설정 파일에서 이 변수를 등록합니다. `public` 영역에 넣어야 브라우저(Client-side)에서도 접근이 가능합니다.

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    // public 안에 넣어야 클라이언트에서도 사용 가능합니다.
    public: {
      apiBase: process.env.NUXT_PUBLIC_API_BASE_URL || 'http://localhost:3000'
    }
  },
  
  // Nuxt UI 설정 (필요한 경우)
  modules: ['@nuxt/ui'],
})

```

---

### 3. `$fetch` 또는 `useFetch` 공통화 (핵심)

매번 `useRuntimeConfig()`를 불러와서 URL을 합치는 건 번거롭습니다. `composables`를 활용해 **커스텀 fetch**를 만드는 것이 가장 세련된 방법입니다.

```typescript
// composites/useApi.ts
export const useApi = (url: string, options = {}) => {
  const config = useRuntimeConfig()
  
  return useFetch(url, {
    baseURL: config.public.apiBase,
    ...options,
    // 여기에 공통 인터셉터(토큰 삽입 등)를 추가하면 더 좋습니다.
    onRequest({ options }) {
      // 예: Authorization 헤더 추가
      // options.headers = options.headers || {}
      // options.headers.Authorization = `Bearer ...`
    }
  })
}

```

---

### 4. 실제 컴포넌트에서 사용하기

이제 복잡한 설정 없이 아주 간단하게 호출할 수 있습니다.

```vue
<script setup>
// 알아서 baseURL이 붙은 상태로 호출됩니다.
const { data, pending } = await useApi('/users')
</script>

<template>
  <UContainer>
    <UCard>
      <div v-if="pending">로딩 중...</div>
      <pre v-else>{{ data }}</pre>
    </UCard>
  </UContainer>
</template>

```

### 💡 요약 및 꿀팁

| 단계 | 방법 | 이유 |
| --- | --- | --- |
| **1. 변수화** | `.env` 사용 | 보안 및 환경 분리 (Dev/Prod) |
| **2. 노출** | `runtimeConfig.public` | 클라이언트 사이드 접근 허용 |
| **3. 추상화** | `useApi` 커스텀 컴포저블 | 중복 코드 방지 및 유지보수 용이 |

**왜 `process.env`를 직접 안 쓰나요?**
Nuxt는 보안을 위해 빌드 타임과 런타임 환경 변수를 엄격히 구분합니다. `runtimeConfig`를 사용해야 빌드 후에 서버를 실행할 때도 환경 변수를 유연하게 바꿀 수 있습니다.

이렇게 설정해두면 나중에 API 주소가 바뀌거나 서버 환경이 추가되어도 `.env` 파일 하나만 수정하면 끝납니다!
