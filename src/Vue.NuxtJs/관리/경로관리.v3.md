결론부터 말씀드리면, 단순한 "값"이라면 상수(Constants)로, 로직과 상태가 포함된 "기능"이라면 컴포저블(Composables)로 관리하는 것이 정석입니다.

`bitcopark-base` 프로젝트의 유지보수 효율을 높여줄 명확한 구분 기준을 정리해 드릴게요.

---

### 1. 상수로 관리해야 하는 것 (Static)

변하지 않는 고정된 값들입니다. 주로 `assets/`나 `utils/` 폴더 내에 정의합니다.

* **대상:** 에러 메시지 모음, 게시판 카테고리 목록, 설정값, 고정된 API 경로 등.
* **장점:** 가볍고, 어디서든 임포트(Import)만 하면 즉시 사용 가능합니다.
* **예시:**
```typescript
// constants/board.ts
export const BOARD_TYPES = {
  NOTICE: 'notice',
  FREE: 'free',
  QNA: 'qna'
} as const

```



### 2. 컴포저블로 관리해야 하는 것 (Dynamic)

Vue의 반응형 시스템(Ref, Computed)이나 Nuxt 전용 기능(useRoute, useFetch)을 써야 한다면 무조건 컴포저블입니다.

* **대상:** 현재 선택된 게시판 ID 꺼내기, 로그인 상태 체크, API 호출 로직 등.
* **장점:** `route`가 바뀌면 자동으로 값이 업데이트되는 "반응성"을 가집니다.
* **예시:**

```typescript
    // composables/useBoard.ts
    export const useBoard = () => {
      const route = useRoute()
      // route.params가 바뀔 때마다 이 값도 같이 변함 (반응형)
      const boardId = computed(() => route.params.id)
      
      return { boardId }
    }
```

---

### 3. 한눈에 비교하기

| 구분 | 상수 (Constants) | 컴포저블 (Composables) |
| :--- | :--- | :--- |
| **성격** | 정적 (Static) | 동적 (Dynamic / Reactive) |
| **핵심 도구** | `export const` | `ref`, `computed`, `useRoute` 등 |
| **데이터 흐름** | 한 번 정해지면 끝 | 주소나 상태가 바뀌면 같이 변함 |
| **메모리** | 매우 가벼움 | 함수가 실행될 때마다 인스턴스 생성 |

---

### 💡 실전 "낭만" 가이드

지금 고민하시는 **`route.params` 관리**는 무조건 **컴포저블**이 낫습니다. 

왜냐하면 `route.params`는 고정된 상수가 아니라, 사용자가 클릭해서 페이지를 이동할 때마다 실시간으로 변하는 **상태(State)**이기 때문입니다. 상수로 관리하려고 하면 페이지가 바뀌었을 때 값이 갱신되지 않는 치명적인 문제가 생길 수 있어요.

**추천 구조:**
1.  **`composables/useBoardRoute.ts`**: 현재 페이지의 `id`, `type` 등을 반응형으로 추출.
2.  **`constants/boardConfig.ts`**: 각 게시판 타입별 한글 이름이나 권한 설정 같은 고정 데이터.

이렇게 두 개를 조합해서 쓰면 "로직은 컴포저블이, 데이터는 상수가" 담당하는 아주 깔끔한 구조가 됩니다. 어때요, 딱 감이 오시나요? 🚀
