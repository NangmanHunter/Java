## Global
### 1. 전역 미들웨어 (Global Middleware) 활용

가장 간단한 방법입니다. 파일 이름 뒤에 `.global.ts`만 붙이면 **모든 페이지**에서 자동으로 실행됩니다.

* **경로:** `middleware/auth.global.ts`
* **특징:** 별도의 설정 없이 프로젝트의 모든 라우트 이동 시 실행됩니다.

```typescript
// middleware/auth.global.ts
export default defineNuxtRouteMiddleware((to, from) => {
  // 특정 페이지(예: 로그인)는 제외하고 체크
  if (to.path === '/login') return

  const { loggedIn } = useAuth() // 혹은 다른 인증 상태 체크
  if (!loggedIn) {
    return navigateTo('/login')
  }
})
```


### 2. 서비스 전체에 적용할 때: `.global.ts` (자동형)

로그인 여부를 모든 페이지에서 체크해야 하거나, 모든 페이지 진입 시 로그를 남겨야 한다면 이 방식이 제일 속 편합니다.

* **장점:** 파일만 만들어두면 끝입니다. 신경 쓸 게 없습니다.
* **추천 상황:** "로그인 안 한 사람은 무조건 로그인 창으로 보내야 한다"는 대전제가 있을 때.

```typescript
// middleware/auth.global.ts
export default defineNuxtRouteMiddleware((to) => {
  // 예외 처리가 핵심! (로그인 페이지까지 막으면 무한 루프에 빠집니다)
  const publicPages = ['/login', '/signup', '/']
  if (publicPages.includes(to.path)) return

  const { loggedIn } = useAuth()
  if (!loggedIn) return navigateTo('/login')
})
```


## RouteRules
### 2. 라우트 규칙 (Route Rules) 설정

`nuxt.config.ts`에서 경로 패턴에 따라 미들웨어를 통으로 걸어버릴 수 있습니다. 페이지 파일은 건드릴 필요가 없어서 관리가 아주 편합니다.

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  routeRules: {
    '/admin/**': { appMiddleware: 'admin-only' },
    '/mypage/**': { appMiddleware: 'auth' },
    '/board/**': { appMiddleware: ['auth', 'log-check'] } // 여러 개도 가능
  }
})

```


### 1. 가장 추천하는 방식: `routeRules` (중앙 집중형)

현재 게시판이나 어드민 페이지처럼 경로(Path)가 확실히 구분되어 있다면 `nuxt.config.ts`에서 관리하는 것이 가장 "관리자답고" 깔끔합니다.

* **장점:** 페이지마다 `definePageMeta`를 적을 필요가 없고, 어떤 페이지에 어떤 보호가 걸려있는지 한눈에 보입니다.
* **추천 상황:** `/admin/`, `/mypage/` 처럼 특정 폴더 전체를 막고 싶을 때.

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  routeRules: {
    // 관리자 페이지는 전부 'admin-auth' 미들웨어 적용
    '/admin/**': { appMiddleware: 'admin-auth' },
    // 글쓰기, 수정 등 특정 액션 페이지는 'auth' 적용
    '/board/**/write': { appMiddleware: 'auth' },
    '/board/**/edit': { appMiddleware: 'auth' },
  }
})

```

### 3. 커스텀 라우터 (Router) 설정 (가장 유연한 방식)

만약 조건이 복잡하다면(예: 특정 `boardType`에 따라서만 미들웨어를 다르게 걸고 싶을 때), 서버 미들웨어와 비슷하게 **전역 미들웨어 하나**에서 지도를 그리듯 관리하는 방식입니다.

```typescript
// middleware/router-guard.global.ts
export default defineNuxtRouteMiddleware((to) => {
  const path = to.path

  // 1. 관리자 구역 관리
  if (path.startsWith('/admin') && !isAdmin()) {
    return navigateTo('/')
  }

  // 2. 게시판 상세 페이지 중 'private' 게시판만 체크
  if (path.includes('/board/secret') && !hasAccess()) {
    return abortNavigation('권한이 없습니다.')
  }
})

```

### 💡 어떤 방식을 추천하나요?

| 방식 | 추천 상황 | 낭만 포인트 |
| --- | --- | --- |
| **`.global.ts`** | 서비스 전체에 공통으로 적용할 때 (예: 로그 수집, 기본 인증) | "설정이 필요 없는 깔끔함" |
| **`routeRules`** | 특정 디렉토리(`admin/`, `api/`) 단위로 끊고 싶을 때 | "한 곳에서 내려다보는 관리자 뷰" |
| **커스텀 가드** | 비즈니스 로직에 따라 동적으로 막아야 할 때 | "세밀한 제어에서 오는 완벽함" |

**실전 팁:**
지금 게시판을 만들고 계시니까, 일반적인 인증은 `auth.global.ts`로 처리하시고, `/admin` 같은 특수 경로는 `nuxt.config.ts`의 `routeRules`로 관리하시는 게 가장 깔끔할 것 같습니다.

이렇게 하면 페이지 코드(`[boardtype].vue` 등)에는 비즈니스 로직만 남게 되어 코드가 훨씬 읽기 좋아집니다! 어떤 구역을 한꺼번에 막고 싶으신가요?


### 3. 복합 전략 (실무에서 가장 많이 쓰는 낭만 조합)

저는 개인적으로 **이 조합**을 가장 추천합니다.

1. `.global.ts`로 **공통 로직** 처리 (예: 유저 정보 복구, 간단한 접근 로그)
2. `routeRules`로 **구역별 권한** 처리 (예: 어드민 구역, 마이페이지 구역)

혹시 지금 바로 막아야 하는 특정 경로(예: `/admin`이나 특정 게시판)가 있으신가요?
