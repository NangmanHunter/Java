##
Route(경로) 관리가 널브러지는 이유는 보통 **'하드코딩된 문자열'** 때문입니다. `'/user/profile/settings'` 같은 경로를 코드 곳곳에 직접 타이핑하다 보면, 나중에 주소가 살짝만 바뀌어도 일일이 찾아 고쳐야 하는 지옥이 펼쳐지죠.

Nuxt에서 Route를 깔끔하게 관리하는 **3단계 전략**을 추천해 드립니다.

---

### 1. Route 상수화 (Object Mapping)

가장 먼저 할 일은 프로젝트의 모든 경로를 하나의 객체로 관리하는 것입니다. `constants/` 폴더를 만들고 경로 파일을 생성하세요.

```typescript
// constants/routes.ts
export const ROUTES = {
  HOME: '/',
  AUTH: {
    LOGIN: '/auth/login',
    SIGNUP: '/auth/signup',
  },
  USER: {
    PROFILE: (id: string | number) => `/user/${id}`,
    SETTINGS: '/user/settings',
  },
  POSTS: {
    LIST: '/posts',
    DETAIL: (slug: string) => `/posts/${slug}`,
  },
} as const

```

* **장점:** 오타 방지, 자동 완성 가능, 경로 변경 시 이 파일만 수정하면 끝.
* **동적 경로:** 위 예시처럼 함수형태로 만들면 인자를 받아 유연하게 대응할 수 있습니다.

---

### 2. 컴포넌트에서 사용 (NuxtLink)

하드코딩 대신 위에서 만든 `ROUTES` 객체를 사용합니다.

```vue
<script setup>
import { ROUTES } from '~/constants/routes'
</script>

<template>
  <nav>
    <!-- 고정 경로 -->
    <UButton :to="ROUTES.AUTH.LOGIN">로그인</UButton>
    
    <!-- 동적 경로 -->
    <NuxtLink :to="ROUTES.USER.PROFILE(user.id)">프로필 보기</NuxtLink>
  </nav>
</template>

```

---

### 3. 구조적 미들웨어 활용 (Access Control)

경로가 많아지면 "로그인한 사람만 들어가야 하는 페이지" 관리가 힘들어집니다. 이때는 `middleware/`를 폴더 구조로 관리하세요.

* **Global Middleware:** 모든 페이지에 적용 (ex: 로그 체크)
* **Named Middleware:** 특정 페이지 그룹에만 적용

```typescript
// middleware/auth.ts
export default defineNuxtRouteMiddleware((to, from) => {
  const auth = useAuthStore() // 예시 스토어

  if (!auth.isLoggedIn && to.path.startsWith('/user')) {
    return navigateTo(ROUTES.AUTH.LOGIN)
  }
})

```

## Typed Router
```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  experimental: {
    typedPages: true // 이 옵션을 켜면 navigateTo, useRoute 등에서 경로 자동완성이 됩니다.
  }
})
```
- Nuxt 3에서는 설정을 통해 **실제 존재하는 파일 기반으로 타입을 자동 생성**할 수 있습니다. `nuxt.config.ts`에서 아래 옵션을 켜보세요.
- 이걸 켜면 `navigateTo('/')`를 입력할 때 현재 `pages/` 폴더 내에 있는 실제 경로들을 IDE가 추천해 줍니다.
