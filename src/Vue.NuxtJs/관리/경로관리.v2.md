##
페이지마다 `useRoute()`를 선언하고 `params.type`, `params.id`를 일일이 꺼내 쓰는 건 확실히 노가다죠. 특히 게시판처럼 주소 구조가 비슷한 페이지가 많을수록 더 그렇습니다.

개발자들이 이 번거로움을 해결하기 위해 사용하는 **3가지 레벨의 관리법**을 알려드릴게요.

---

### 1. Level 1: 구조 분해 할당 (가장 쉬운 개선)

매번 변수를 따로 선언하지 않고, 한 줄로 깔끔하게 가져오는 방식입니다.

```typescript
const { params: { type: boardType, id: boardId } } = useRoute()

```

* **장점:** 코드가 짧아집니다.
* **단점:** 여전히 페이지마다 이 줄을 써줘야 합니다.

---

### 2. Level 2: 커스텀 컴포저블 (추천: "낭만" 방식)

`bitcopark-base` 프로젝트 전용 도구를 만드는 겁니다. `useBoardRoute`라는 이름으로 게시판 관련 파라미터만 쏙 뽑아주는 함수를 만드세요.

* **파일 위치:** `composables/useBoardRoute.ts`

```typescript
export const useBoardRoute = () => {
  const route = useRoute()
  
  // 파라미터가 없을 때를 대비한 기본값 설정도 한 번에!
  return {
    boardType: route.params.type as string,
    boardId: route.params.id as string,
    query: route.query, // 검색어 같은 게 필요할 때를 대비
    fullPath: route.path
  }
}

```

**사용할 때는?**

```typescript
const { boardType, boardId } = useBoardRoute()

```

* **장점:** 페이지 코드가 극도로 깨끗해집니다. 나중에 파라미터 이름이 `category`로 바뀌어도 이 파일 한 곳만 고치면 됩니다.



---

### 3. Level 3: 자동 타입 지정 (Typed Router)

최근 Nuxt 3에서는 `nuxt-typed-router` 같은 모듈을 쓰거나, 최신 버전의 설정을 통해 주소 자체를 타입으로 관리합니다. 하지만 이건 설정이 좀 복잡할 수 있으니, 지금은 **Level 2** 방식이 가장 효율적입니다.

---

### 💡 보너스 팁: `watch`와 결합하기

게시판 상세 페이지에서 다른 글로 이동할 때(주소는 바뀌는데 컴포넌트는 그대로일 때), 파라미터가 바뀌는 걸 감지해야 할 때가 있죠? 그럴 때도 컴포저블이 빛을 발합니다.

```typescript
// useBoardRoute.ts 내부에 추가 가능
const onBoardChange = (callback: (id: string) => void) => {
  const route = useRoute()
  watch(() => route.params.id, (newId) => {
    if (newId) callback(newId as string)
  })
}

```

### 요약: 어떻게 하는 게 제일 나을까?

지금 바로 **`composables/useBoardRoute.ts`** 파일을 만드세요.

거기서 `boardType`과 `boardId`를 꺼내서 리턴해주는 로직을 짜두면, 앞으로 만드는 모든 게시판 관련 `.vue` 파일에서 단 한 줄로 모든 정보를 가져올 수 있습니다. 이게 진정한 "중앙 집중형 관리"의 낭만이죠!

어떤 파라미터를 가장 자주 쓰시나요? `type`이랑 `id` 말고 또 필요한 게 있을까요?
