Nuxt와 Vue는 양이 방대하지만, 실무에서 **"이걸 모르면 개발이 안 된다"** 싶은 핵심 줄기들이 있습니다. 굵직한 개념들 위주로 정리해 드릴게요.

## 1. Vue.js 핵심 개념 (뿌리)

Nuxt는 Vue를 기반으로 하기 때문에, 아래 개념들이 흔들리면 Nuxt 기능을 제대로 쓰기 어렵습니다.

*   **컴포넌트 통신 (Props & Emit):** 부모는 자식에게 데이터를 주고(`Props`), 자식은 부모에게 사건을 알리는(`Emit`) 기본 흐름입니다.
*   **반응성 기초 (ref & reactive):** Vue 3 Composition API의 핵심입니다. 데이터가 변할 때 화면이 자동으로 바뀌게 만드는 "반응형 변수" 선언 방식입니다.
*   **생명주기 훅 (Lifecycle Hooks):** 컴포넌트가 생길 때(`onMounted`), 사라질 때(`onUnmounted`) 등 특정 시점에 코드를 실행하는 법을 알아야 합니다.
*   **가상 DOM (Virtual DOM):** Vue가 왜 빠른지, 실제 DOM을 직접 건드리지 않고 어떻게 효율적으로 화면을 그리는지에 대한 메커니즘입니다.

## 2. Nuxt.js 핵심 개념 (줄기)

Nuxt는 Vue에 "규칙"과 "강력한 기능"을 더한 프레임워크입니다. 다음 4가지가 가장 굵직합니다.

### ① 렌더링 모드 (SSR & CSR)
Nuxt를 쓰는 가장 큰 이유입니다. 
*   **SSR (Server Side Rendering):** 서버에서 미리 HTML을 만들어 보내서 SEO(검색 엔진 최적화)에 유리합니다.
*   **Hydration:** 서버에서 받은 정적 페이지에 Vue의 반응성을 입혀 동적 페이지로 만드는 과정을 이해해야 합니다.

### ② 파일 기반 라우팅 (File-based Routing)
별도의 설정 파일 없이 `pages/` 폴더 내의 파일 구조가 그대로 브라우저 주소(URL)가 됩니다. 
*   `pages/index.vue` → `/`
*   `pages/user/[id].vue` → `/user/1` (동적 라우팅)

### ③ 오토 임포트 (Auto-imports)
Nuxt의 생산성 핵심입니다. `components/`, `composables/`, `utils/` 폴더에 있는 파일들은 `import` 구문 없이 바로 사용할 수 있습니다. 처음엔 마법 같지만, 이 규칙을 알아야 코드가 깔끔해집니다.

### ④ 데이터 페칭 (useFetch & useAsyncData)
클라이언트와 서버 양쪽에서 데이터를 안전하게 가져오는 방식입니다. SSR 환경에서 데이터가 중복으로 호출되는 것을 방지하는 Nuxt만의 전용 훅입니다.

## 3. 구조적 차이 한눈에 보기

| 개념 | Vue.js (Library) | Nuxt.js (Framework) |
| :--- | :--- | :--- |
| **라우팅** | `vue-router` 직접 설정 | `pages/` 폴더 구조로 자동 생성 |
| **상태 관리** | `Pinia` 등 별도 설치 | `useState` 기본 내장 (Pinia 권장) |
| **SEO** | 어려움 (기본 CSR) | 매우 쉬움 (기본 SSR) |
| **설정** | 자유롭지만 번거로움 | 'Convention over Configuration' (규칙 중심) |

## 공부 전략 제언
처음부터 모든 API를 외우려 하지 마세요. 
1.  **Vue**에서는 `ref`, `props/emit`으로 데이터 흐름을 먼저 잡으시고, 
2.  **Nuxt**에서는 `pages/`를 통한 화면 이동과 `useFetch`로 데이터 가져오는 것부터 익숙해지시는 걸 추천합니다.
