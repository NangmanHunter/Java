## 고민해볼것
- 중복명칭사용 vs. 개별떨구기
- 중복명칭사용<<개별떨구기-근래
- 중복명칭사용<<개별떨구기
- 개별떨구기>>중복명칭사용


- 중복명칭사용
  - aaa/Aaa
  - aaa/AaaBbb
  - aaa/AaaCcc
  - 이런식으로가는데.
  - 이게 최초A부분은 상당좋음.
  - 그러나 B C 이런것들 하나두개정도는 ㅇㅋ.
  - 그러다가도 좀너무비대해지거나하면은 또 어설퍼지는경우있음.

  - aaa/AaaBbb
  - aaa/Bbb
  - 이러면 이거 중복날거같은데?!. 
  - 맞네 중복이다. `aaa/AaaBbb`>>`aaa/Bbb` 이렇게가버리네.
  - 그러면 애당초 규칙성을 싹잡아주는게 더맞다.
  - 어설프게 `aaa/Aaa` 이런거 붙여버리면 규칙성이 뒷라인까지 싹다붙음. 이건또별로임. 너무잡스러움.

- 중복명칭없애기
  - 그러면 아싸리없애자. 쓰지말자. 
  - 불가피하게해서 뭐쓸순있다. 딱그정도. 근데 정렬틀어진다. 규칙성틀어진다. 이런건또 내스타일아니다. 비최적이다.
  - 아싸리 정렬화피할수있게ㄱㄱ.
  - 명칭을 기본으로 Base 이런식으로 붙여버리기ㄱㄱ.

## `Main.vue`
- Main.vue 또는 Index.vue
- Root.vue 또는 Base.vue
- Wrapper.vue / Container.vue

- Section.vue
  - 역할 중심의 구체적 명명 (가장 추천)
  - 사실 comment/ 폴더 안에 있다는 것 자체가 이미 '댓글'임을 설명하므로, "그 댓글이 구체적으로 무엇인가"를 뒤에 붙여주는 게 가장 깔끔합니다.
  - comment/List.vue (`<CommentList />`): 댓글 목록 전체
  - comment/Item.vue (`<CommentItem />`): 댓글 한 개 단위
  - comment/Section.vue (`<CommentSection />`): 게시글 하단의 댓글 구역 전체
  - comment/Group.vue (`<CommentGroup />`): 대댓글 등을 포함한 덩어리
