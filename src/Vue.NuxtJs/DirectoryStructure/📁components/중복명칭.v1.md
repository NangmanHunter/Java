- 이질감느끼기보단. 철저활용ㄱㄱ.
- 이양반들도 너무도 고심해서 만든규칙이긴함.
- 내가봐도그럼. 그렇기에 이양반들 겹치는것 어떻게든 최소화해서. 철저히 사람직감적 사람중심적 바로직감적 이것살리고자 만든규칙임. 
- 중복되는부분은 그부분은 줄여주자 이패턴.
- 심지어 중복부분도 파스칼케이스에서 앞단중복부분만 짤라서 딱잡아줌. 너무잘만들었음. 오히려 누군가가 항상고민하던 그런부분이기에.
- 디렉토리ㆍ파일 이런거는 철저히 비중복위해 정렬위해 중복형태로가주는게 고유성 살려주면서. 정렬성도 살려줘서. 직감력 극화로올라감.

- 그러나
- 실상쓸때는 가져다쓸때는 중복부분 싹도려내서 최소화로만 최소성으로만 진행되게 만들어주면되는것. 이거임.
- 이형태로서 싹만들놓음. 너무도대단하고 너무도감사함. 고심함이 엄청난고민이 너무도녹아져있음.
- 파일=고유성 + 사용=유일성
- 파일=고유성(중복표현) + 사용=유일성(비중복표현)
- 파일=고유성(중복표현) + 사용=유일성(유일표현)
- 파일=고유성(접두표현) + 사용=유일성(유일표현)
- 파일=고유성(접두표현) + 사용=유일성(비접두표현)

```text
components/

      ├── Comment.vue        <-- 메인 컴포넌트 (사용: <Comment />)
      ├── CommentList.vue    <-- 목록 담당
      └── CommentItem.vue    <-- 개별 댓글 아이템
```


- 댓글 섹션은 앞으로 기능이 많아질 테니(수정, 삭제, 답글 등), 폴더 구조를 아래처럼 가져가는 게 가장 깔끔합니다.
- 이렇게 하면 `components/comment/Comment.vue` 파일이 자동으로 **`<Comment />`**라는 이름으로 등록됩니다. 


##
Nuxt의 **자동 컴포넌트 스캔** 규칙 때문에 발생하는 현상이군요! Nuxt는 기본적으로 디렉토리 경로를 컴포넌트 이름에 포함시킵니다.

`components/comment/index.vue`라고 만들면, Nuxt는 이 컴포넌트의 이름을 **`Comment`**가 아니라 **`CommentIndex`**로 인식하게 됩니다. (상위 디렉토리 이름 + 파일 이름)

이를 해결하고 **`<Comment />`**라는 깔끔한 이름으로 사용하고 싶다면 아래 세 가지 방법 중 하나를 선택하시면 됩니다.

---

### 1. 가장 간단한 방법: 파일명 변경
디렉토리를 만들지 않고 파일명을 바로 짓는 것입니다.
* **경로:** `components/Comment.vue`
* **사용:** `<Comment />`
* **장점:** 가장 표준적이고 설정이 필요 없습니다.

### 2. 디렉토리를 유지하고 싶을 때 (추천)
댓글 관련 컴포넌트가 많아질 것 같아(`CommentItem.vue`, `CommentList.vue` 등) 폴더를 유지하고 싶다면, 메인 파일의 이름을 디렉토리명과 똑같이 맞추면 됩니다.
* **경로:** `components/comment/Comment.vue`
* **사용:** `<Comment />`
* **원리:** Nuxt는 디렉토리 이름과 파일 이름이 중복되면 중복된 부분을 제거하고 이름을 붙여줍니다. (`CommentComment`가 아니라 `Comment`가 됨)

### 3. `index.vue`를 유지하고 싶을 때 (`pathPrefix` 끄기)
구조상 반드시 `index.vue`를 써야겠다면, `nuxt.config.ts`에서 설정을 바꿔야 합니다. 하지만 이 방법은 프로젝트 전체의 컴포넌트 규칙에 영향을 주므로 신중해야 합니다.

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  components: [
    {
      path: '~/components',
      pathPrefix: false, // 디렉토리 이름을 접두어로 붙이지 않음
    },
  ],
})
```
* **주의:** 이걸 끄면 `components/board/List.vue`도 그냥 `<List />`가 됩니다. 이름 충돌이 날 확률이 높아서 권장하지는 않습니다.
