##
아쉽게도 Nuxt의 **파일 기반 라우팅 시스템(File-based Routing)** 자체에서 `?type=A`와 같은 쿼리 스트링을 폴더나 파일 이름으로 지정해서 매핑하는 기능은 지원하지 않습니다.

파일 시스템 라우팅은 어디까지나 **경로(Path)**, 즉 `/`로 구분되는 구조를 기반으로 설계되었기 때문입니다. 하지만 질문하신 의도처럼 **"특정 주소로 접속했을 때 쿼리 스트링에 따라 알아서 페이지가 연결되는 느낌"**을 구현할 수 있는 설정 방법이 두 가지 있습니다.

---

### 1. Middleware(미들웨어)를 이용한 자동 분기
폴더 구조를 물리적으로 만들 수는 없지만, 사용자가 `/boards`에 들어왔을 때 쿼리값에 따라 다른 로직을 타게 강제할 수 있습니다.

* **설정:** `middleware/route-guard.global.ts` (혹은 특정 페이지용 미들웨어)
* **동작:** 사용자가 `/boards?type=NOTICE`로 접속하면 미들웨어에서 이를 감지해 로직을 수행합니다.

### 2. 커스텀 라우터 설정 (Router Options)
만약 꼭 파일 구조가 아니더라도 **설정 파일**을 통해 쿼리 스트링이 포함된 주소를 특정 파일에 강제로 연결하고 싶다면 `app/router.options.ts`를 활용할 수 있습니다. 하지만 이 역시 쿼리 스트링은 보통 '동적 데이터'로 취급되어 추천되는 방식은 아닙니다.

---

### 💡 가장 근접한 해결책: 동적 임포트 (Dynamic Import)
파일 이름으로 지원되진 않지만, `pages/boards.vue` 하나만 만들고 내부에서 파일 구조처럼 관리하는 가장 효율적인 방법은 다음과 같습니다.

```vue
<template>
  <component :is="currentPage" />
</template>

<script setup>
const route = useRoute()

// 쿼리 값에 따라 components/board/ 폴더 내의 파일을 동적으로 가져옴
const currentPage = computed(() => {
  const type = route.query.type || 'DEFAULT'
  // 예: ?type=NOTICE 이면 BoardNotice.vue 컴포넌트를 렌더링
  return defineAsyncComponent(() => import(`@/components/board/Board${type}.vue`))
})
</script>
```

**이렇게 설정하면 생기는 장점:**
1.  **구조적 관리:** `components/board/` 폴더 안에 `BoardNOTICE.vue`, `BoardFREE.vue` 식으로 파일을 나눠서 관리할 수 있습니다.
2.  **주소 유지:** 브라우저 주소창에는 `?type=NOTICE`가 그대로 유지되면서 화면만 파일별로 갈아끼워집니다.

**결론:**
Nuxt 파일 시스템에서 `pages/?type=A.vue` 같은 파일명은 불가능하지만, **`pages/boards.vue` 내부에서 위와 같이 작성**하면 원하시는 "쿼리에 따른 페이지 분리" 효과를 완벽하게 구현할 수 있습니다.
