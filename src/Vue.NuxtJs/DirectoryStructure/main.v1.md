

##
`app.vue`는 Nuxt 프로젝트의 **"뿌리(Root)"**일 뿐입니다. 하지만 Nuxt의 진짜 힘은 폴더를 만들고 파일을 추가하면, **알아서 라우팅(URL 경로 설정)과 구조를 잡아주는 '규칙(Convention)'**에 있습니다.




---

### 1. Nuxt의 핵심 원리: 파일 시스템 기반 라우팅
Nuxt는 `pages/` 폴더 안의 파일 이름을 기준으로 URL 경로를 자동으로 만듭니다. 이걸 이해하는 게 제일 중요합니다.

* `pages/index.vue` → `http://localhost:3000/` (홈)
* `pages/about.vue` → `http://localhost:3000/about` (소개)
* `pages/stocks/index.vue` → `http://localhost:3000/stocks` (주식 목록)

즉, **페이지를 늘리고 싶으면 `pages/` 안에 파일을 만들기만 하면 됩니다.** 따로 설정 파일에서 경로를 등록할 필요가 없습니다.

---

### 2. 폴더별 역할 (이것만 알면 됩니다)


| 폴더명 | 역할 | 언제 쓰는가? |
| :--- | :--- | :--- |
| **`pages/`** | **URL 경로(라우트)** | 각 화면마다 하나씩 파일 생성 |
| **`layouts/`** | **페이지 공통 틀** | 헤더, 푸터가 고정된 전체 레이아웃 |
| **`components/`** | **재사용 가능한 UI 조각** | 버튼, 차트, 입력폼 등 반복되는 UI |
| **`assets/`** | **원본 소스** | CSS, 이미지 등 빌드 과정이 필요한 파일 |
| **`public/`** | **정적 파일** | 로고, 폰트 등 수정 없이 바로 서빙되는 파일 |
| **`composables/`** | **공통 로직 (핵심!)** | 세금 계산, 데이터 조회 등 로직을 모아둠 |

### 3. 프로젝트가 커지는 과정 (예시)

질문자님이 만드시는 **주식 데이터 대시보드**를 예로 들어볼게요.

1.  **처음:** `app.vue`와 `pages/index.vue`만 있다.
2.  **성장:** "주식 차트" 컴포넌트가 필요해지면 `components/StockChart.vue`를 만든다.
3.  **확장:** "주식 상세 페이지"가 필요해지면 `pages/stocks/[id].vue` 파일을 만든다.
4.  **로직 분리:** API 호출 코드가 너무 길어지면 `composables/useStockData.js`로 빼서 재사용한다.

이런 식으로 폴더를 **'늘려가는'** 것이 아니라, **'역할에 맞게 파일을 배치'**하면 Nuxt가 알아서 종속성을 잡고 최적화해 줍니다. 

### 💡 팁: 초보자를 위한 조언
처음부터 폴더를 너무 세분화하지 마세요. 일단 `pages/` 폴더에 화면을 쭉 나열해 보고, **"어? 이 코드는 다른 데서도 쓰겠는데?"** 싶을 때 `components/`로 옮기고, **"이 로직은 복잡한데?"** 싶을 때 `composables/`로 옮기는 것이 가장 안전한 확장법입니다.

지금 `app.vue`에 모든 코드를 다 때려 넣고 계신 건 아니죠? 혹시 `pages/` 폴더를 만들어 보셨나요? 어떤 기능을 먼저 페이지로 분리해 보고 싶으신가요?
