현재 구조에서 데이터가 갱신되지 않는 이유는 **`comments` 상태(State)의 공유 방식** 때문입니다.

`useComment` 내부에서 `const comments = ref([])`를 정의하면, 각 컴포넌트가 `useComment()`를 호출할 때마다 **각자 독립된 `comments` 주머니**를 가지게 됩니다.

1.  **`Post.vue`**가 `fetchComments`를 실행해서 데이터를 가져오면, 자기만의 `comments` 주머니만 채웁니다.
2.  하지만 화면에 목록을 뿌려주는 **`List.vue`**는 자기가 처음 가졌던 **빈 주머니**만 보고 있습니다.

---

### 🛠️ 해결책: 상태 공유하기 (싱글톤 패턴)

`composables/useComment.ts` 파일에서 `comments` 변수를 **함수 밖으로** 빼내야 합니다. 이렇게 하면 모든 컴포넌트가 단 하나의 주머니를 공유하게 됩니다.



```typescript
// composables/useComment.ts

// 1. 함수 밖에서 선언 (딱 한 번만 생성되어 공유됨)
const comments = ref<any[]>([])

export const useComment = () => {
  const config = useRuntimeConfig()
  const route = useRoute()

  const fetchComments = async () => {
    // URL 분석 로직
    const pathParts = route.path.split('/')
    const boardType = pathParts[2]
    const boardId = pathParts[3]

    try {
      const data = await $fetch<any[]>(`/api/boards/${boardType}/${boardId}/comments`, {
        baseURL: config.public.apiBase,
        credentials: 'include'
      })
      // 전역 변수인 comments에 데이터를 넣음
      comments.value = data
    } catch (error) {
      console.error('댓글 로드 실패:', error)
    }
  }

  return { 
    comments, // 이제 모든 컴포넌트가 같은 comments를 봅니다.
    fetchComments 
  }
}
```

### ✅ 컴포넌트별 체크리스트

#### 1. CommentList.vue
여기서 반드시 `useComment`로부터 `comments`를 받아와서 `v-for`를 돌려야 합니다.

```vue
<script setup>
const { comments, fetchComments } = useComment()

onMounted(() => {
  fetchComments() // 초기 데이터 로드
})
</script>

<template>
  <div>
    <div v-for="comment in comments" :key="comment.id">
      {{ comment.content }}
    </div>
    <slot />
  </div>
</template>
```

#### 2. CommentPost.vue
유저님의 코드에서 `await fetchComments()`를 호출하는 시점에, 이제 **전역 `comments`**가 업데이트되므로 `List.vue` 화면이 즉시 반응합니다.

---

### 💡 왜 안됐었는지 요약
* **기존:** `Post.vue`의 주머니 != `List.vue`의 주머니 (서로 남남)
* **수정:** `Post.vue`와 `List.vue`가 **하나의 주머니**를 같이 사용함.
