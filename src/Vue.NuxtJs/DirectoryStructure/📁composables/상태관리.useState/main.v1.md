##
Nuxt를 공부하다 보면 `useState`가 정말 자주 보이죠! React를 써보셨다면 익숙할 이름이지만, Nuxt(Vue)에서의 `useState`는 그 목적이 아주 명확합니다.

한마디로 정의하자면 "SSR(서버 사이드 렌더링) 환경에서도 데이터가 깨지지 않게 안전하게 공유되는 상태 관리 도구"라고 이해하시면 됩니다.

---

### 1. 왜 그냥 `ref`를 안 쓰고 `useState`를 쓸까?

Vue 개발자라면 보통 데이터를 만들 때 `ref`나 `reactive`를 씁니다. 하지만 Nuxt 같은 **SSR 프레임워크**에서는 치명적인 문제가 하나 있어요.

* **문제 상황:** 서버에서 데이터를 생성해서 사용자에게 보내줄 때, 일반 변수는 서버와 클라이언트 사이에서 연결이 끊깁니다. 최악의 경우, 여러 사용자가 접속했을 때 서버 메모리에 있는 변수를 공유해버리는 **데이터 오염**이 발생할 수 있어요.
* **해결책:** `useState`는 서버에서 생성한 상태를 '직렬화(Serialization)'해서 클라이언트로 고스란히 전달합니다. 덕분에 새로고침을 해도 데이터가 안전하게 유지되고, 컴포넌트 사이에서 데이터를 쉽게 공유할 수 있죠.

---

### 2. 주요 특징

1. **전역 상태 공유 (Global State):** 굳이 Pinia 같은 무거운 라이브러리를 설치하지 않아도, 앱 전체에서 공유할 수 있는 변수를 만들 수 있습니다.
2. **Hydration 친화적:** 서버에서 렌더링된 초기 데이터와 클라이언트의 데이터를 일치시켜 줍니다.
3. **반응성:** `ref`와 똑같이 작동합니다. 값이 바뀌면 화면도 즉시 바뀝니다.

---

### 3. 어떻게 사용하나요?

가장 기본적인 사용법은 다음과 같습니다.

```javascript
// 사용법: const state = useState(고유키, 초기값함수)

// 1. 상태 선언 (어느 컴포넌트에서나 똑같이 접근 가능)
const counter = useState('counter', () => 0)

// 2. 값 변경
const increment = () => {
  counter.value++
}

```

* **첫 번째 인자 (`'counter'`):** 상태를 식별할 수 있는 유니크한 이름(Key)입니다. 이 키가 같으면 다른 파일에서도 같은 데이터를 바라보게 됩니다.
* **두 번째 인자:** 데이터의 초기값을 설정하는 함수입니다.

---

### 4. 언제 사용하면 좋을까?

* **간단한 전역 상태:** 로그인한 유저 정보, 현재 테마(다크모드), 언어 설정 등.
* **Props Drilling 방지:** 부모->자식->손자 컴포넌트로 데이터를 계속 넘겨주기 귀찮을 때.
* **SSR 데이터 유지:** 서버에서 미리 계산된 값을 클라이언트에서도 그대로 쓰고 싶을 때.
