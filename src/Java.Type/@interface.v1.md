## @interface
- 이거는 약간 메타데이터 쪽이긴하네.


##
`interface`와 `@interface`는 이름은 비슷하지만, 자바에서 **"무엇을 정의하느냐"**라는 목적 자체가 완전히 다른 개념입니다. 

질문하신 '엄밀한 구분' 관점에서 보면, 하나는 **규격(Interface)**이고 하나는 **메타데이터(Annotation)**라고 볼 수 있습니다.

---

### 1. `interface` (인터페이스)
우리가 흔히 아는 그 인터페이스입니다. 객체가 **"어떤 동작을 할 수 있는지"** 정의하는 일종의 규격서 또는 계약서입니다.

* **목적:** 다형성 구현, 클래스 간의 결합도 낮추기.
* **특징:** * 추상 메서드(`abstract method`)를 가집니다.
    * 다른 클래스가 이를 `implements` 하여 실제 동작을 구현해야 합니다.
    * "이 클래스는 이 기능을 반드시 가지고 있어!"라고 보장하는 역할입니다.

### 2. `@interface` (어노테이션)
자바에서 **어노테이션(Annotation)** 타입을 정의할 때 사용하는 키워드입니다. 코드에 붙이는 **'태그'나 '설명 포스트잇'**이라고 생각하면 쉽습니다.

* **목적:** 코드에 부가적인 정보를 제공 (컴파일러 체크, 프레임워크 설정 등).
* **특징:**
    * 메서드 형태의 필드를 가질 수 있지만, 로직을 가질 순 없습니다.
    * `implements` 하는 것이 아니라, 클래스나 메서드 위에 `@MyAnnotation` 식으로 붙여서 사용합니다.
    * 직접 동작하기보다는, **Spring 같은 프레임워크나 컴파일러가 이 태그를 보고 특정 작업을 수행**하게 합니다.

---

### 3. 한눈에 비교하기

| 구분 | `interface` | `@interface` (Annotation) |
| :--- | :--- | :--- |
| **정의** | 인터페이스 (규격) | 어노테이션 (메타데이터) |
| **사용 방식** | `class A implements B` | `@MyTag` (클래스/메서드 위에 부착) |
| **핵심 역할** | "무엇을 해야 하는가" (행위 강제) | "이것은 무엇인가" (정보 제공) |
| **상속/구현** | 다른 클래스가 구현해야 의미가 있음 | 붙여놓기만 해도 툴이나 프레임워크가 읽음 |



---

### 4. 왜 이름이 비슷할까?
사실 `@interface`는 내부적으로 `java.lang.annotation.Annotation`이라는 **인터페이스를 상속받는 특수한 형태의 인터페이스**입니다. 그래서 문법적으로 메서드 선언 방식 등이 인터페이스와 닮아있어 `@`를 붙여 구분하는 방식을 택한 것입니다.

**실제 예시로 보면 더 명확합니다:**

```java
// 1. interface: "걷다"라는 기능을 강제함
public interface Walkable {
    void walk(); 
}

// 2. @interface: "이 코드는 내가 만든 거야"라고 표시함
public @interface CreatedBy {
    String author();
    String date();
}

// 적용 예시
@CreatedBy(author = "User", date = "2024-05-22")
public class Human implements Walkable {
    @Override
    public void walk() {
        System.out.println("걷는 중...");
    }
}
```

결국 아까 말씀하신 **"설계도 지점"**에 올 수 있는 것들은 `class`, `interface`, `enum`, 그리고 이 `@interface`까지 포함된다고 보시면 됩니다. 

혹시 스프링에서 `@Service`나 `@RestController` 같은 걸 보면서 이게 어떻게 동작하는지 궁금하셨던 건가요?
