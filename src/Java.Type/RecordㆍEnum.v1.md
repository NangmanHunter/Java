
##
`enum`과 `record` 모두 자바에서 **"데이터를 다루기 위해 특화된 특별한 형태의 클래스"**라는 공통점이 있지만, 그 본질적인 목적은 완전히 다릅니다.

간단히 요약하자면, **`enum`은 '선택지'를 정의**하는 것이고, **`record`는 '데이터 그 자체'를 담는 상자**를 정의하는 것입니다.

---

### 1. Enum (열거 타입): "정해진 상수들의 집합"
`enum`은 변할 수 없는 고정된 값들을 정의할 때 사용합니다.

* **핵심:** 인스턴스의 개수가 정해져 있습니다 (싱글톤 보장).
* **용도:** 요일, 상태 코드(결제완료, 배송중), 사용자 권한(ADMIN, USER).
* **특징:** 외부에서 `new`를 통해 새로운 객체를 절대 만들 수 없습니다.

```java
public enum Status {
    READY, PROCESSING, DONE; // 이 세 가지만 존재함
}
```

### 2. Record (레코드): "불변 데이터를 담는 투명한 객체"
자바 14(정식은 16)에서 도입된 `record`는 오직 **데이터를 전달(DTO)**하기 위해 만들어졌습니다.

* **핵심:** 데이터를 저장하는 필드와 그에 따른 메서드(Getter, toString, equals 등)를 자동으로 생성합니다.
* **용도:** API 응답 객체, DB 조회 결과, 단순히 값만 모아놓은 DTO.
* **특징:** `new` 키워드로 인스턴스를 무한히 생성할 수 있지만, 한 번 생성된 데이터는 수정할 수 없습니다(Immutable).

```java
public record UserDto(String name, int age) { }
// 단 한 줄로 생성자, getter, equals, hashCode, toString이 자동 생성됨
```



---

### 3. 엄밀한 차이점 비교

| 구분 | **Enum** | **Record** |
| :--- | :--- | :--- |
| **주 목적** | **선택지 제한** (Categorization) | **데이터 운반** (Data Carrier) |
| **인스턴스 생성** | 내부적으로 정의된 상수만 존재 (`new` 불가) | 필요한 만큼 `new`로 생성 가능 |
| **상태 값** | 모든 인스턴스가 동일한 상태를 가짐 (상수) | 각 인스턴스마다 서로 다른 데이터 보유 가능 |
| **상속** | `java.lang.Enum` 상속 (추가 상속 불가) | `java.lang.Record` 상속 (추가 상속 불가) |
| **사용 예시** | `Direction.NORTH` | `new User("Gemini", 20)` |

---

### 4. 언제 무엇을 쓸까?

* **"이 변수에는 A, B, C 중 하나만 들어올 수 있어"**라고 제약하고 싶다면 👉 **`enum`**
* **"이 객체는 이름, 나이, 주소 정보를 묶어서 전달하는 역할이야"**라고 하고 싶다면 👉 **`record`**

재밌는 점은 `record` 안에서도 `enum`을 쓸 수 있다는 것입니다. 예를 들어 `UserRecord`라는 데이터를 만들 때, 그 안의 `Level` 필드를 `enum`으로 정의해서 "사용자 등급은 딱 3가지만 가능해"라고 설계하는 식이죠.
