클래스, 인터페이스, 레코드, 그리고 열거형(Enum) 같은 것들을 통칭해서 부르는 용어는 크게 두 가지 관점으로 나뉩니다.

가장 정확한 기술적 명칭은 **타입(Type)** 또는 **참조 타입(Reference Type)**입니다.

---

## 1. 자바의 4가지 "타입(Type)"
자바에서 우리가 소스 파일(`.java`)을 만들 때 선언하는 이 구조체들을 묶어서 **타입**이라고 부릅니다. 자바 14 이후부터는 이들을 구분할 때 다음과 같이 분류하곤 합니다.

| 명칭 | 특징 |
| :--- | :--- |
| **클래스 (Class)** | 가장 일반적인 객체의 설계도. 상태와 행위를 모두 가짐. |
| **인터페이스 (Interface)** | 행위의 규격(규칙)을 정의함. |
| **레코드 (Record)** | **데이터 전달**에 최적화된 특수 클래스 (Java 14+). |
| **열거형 (Enum)** | 연관된 상수들의 집합. |

---

## 2. 왜 "타입"이라고 부를까?
우리가 `String name;`이라고 쓸 때 `String`이 변수의 **타입(형태)**이 되는 것처럼, 내가 만든 `UserRecord`나 `UserService` 인터페이스도 결국 변수의 자료형으로 쓰이기 때문입니다.

> **참고:** 가끔 개발자들 사이에서 "사용자 정의 자료형"이나 "객체 타입"이라는 말로 섞어 쓰기도 하지만, 공식 문서나 전문 서적에서는 보통 **`Reference Type` (참조 타입)** 혹은 **`Kind of Type`**이라는 표현을 주로 사용합니다.

---

## 3. "레코드"의 정확한 정체
질문하신 **레코드**는 엄밀히 말하면 **"클래스의 일종"**입니다.
하지만 일반 클래스와 용도가 명확히 다르기 때문에, 자바 언어 스펙에서는 이를 구분하기 위해 **`Record Type`**이라는 용어를 사용합니다.

* **Class:** 일반적인 설계도
* **Interface:** 규격서
* **Record:** 데이터 상자

---
