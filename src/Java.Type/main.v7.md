질문의 의도를 완벽히 이해했습니다! 단순히 "참조 타입"이라는 넓은 범주 말고, 코드를 짤 때 `class` 키워드로 직접 정의하는 그 **'설계도'**들을 따로 부르고 싶으신 거군요.

엄밀하게 구분하자면, 자바에서 이를 지칭하는 공식적인 용어와 실무적인 용어는 다음과 같습니다.

---

### 1. 사용자 정의 타입 (User-Defined Types)
가장 정확한 표현입니다. `int`, `double` 같은 기본 타입이나 자바 언어 자체에서 제공하는 배열(Array)과 달리, 개발자가 직접 `class`, `interface`, `enum` 키워드를 사용하여 **새롭게 정의한 타입**을 의미합니다.

### 2. 클래스 타입 (Class Types)
가장 일반적으로 쓰이는 용어입니다. 비록 배열도 내부적으로는 `Object`를 상속받는 클래스 구조를 갖지만, 보통 자바 프로그래밍 맥락에서 **"클래스 타입"**이라고 하면 `public class MyClass { ... }`처럼 선언된 형태를 지칭합니다.

### 3. 참조 타입의 세부 분류 (공식 명칭)
자바 언어 명세(Java Language Specification)에서는 참조 타입을 아래와 같이 네 가지로 엄격히 구분합니다.

| 구분 | 설명 | 예시 |
| :--- | :--- | :--- |
| **Class Types** | `class` 키워드로 선언된 타입 | `String`, `Scanner`, `MyService` |
| **Interface Types** | `interface` 키워드로 선언된 타입 | `List`, `Runnable` |
| **Array Types** | 데이터의 집합체인 배열 타입 | `int[]`, `String[]` |
| **Type Variables** | 제네릭에서 사용하는 타입 변수 | `<T>`, `<E>` |

---

### 왜 배열은 `class` 지점에 들어가는 것들과 다르게 느껴질까?

질문하신 대로 배열도 내부적으로는 클래스처럼 동작하지만, 결정적인 차이가 있습니다.

* **배열(Array):** 자바 가상 머신(JVM)이 실행 시점에 직접 생성하며, 개발자가 소스 코드(`Array.java` 같은 파일)로 직접 구조를 정의할 수 없습니다. 
* **클래스(Class):** 개발자가 직접 `.java` 파일을 만들고 멤버 변수와 메서드를 정의합니다.

> **결론:** > `public class ...` 자리에 오는 것들만 딱 집어서 말하고 싶다면 **"사용자 정의 클래스(User-defined Class)"** 혹은 **"클래스 타입(Class Type)"**이라고 부르는 것이 가장 적절합니다.
