

## Modifier
- abstract
- final
- sealed


##
맞습니다! `class`라는 키워드 앞에 붙는 `abstract` 같은 **제어자(Modifier)**나, 아예 `class` 대신 사용하는 **다른 키워드**들까지 포함하면 그 범위가 더 넓어지죠.

자바에서 `public class ...`와 같은 선언부 위치에 올 수 있는 모든 "타입의 종류"를 통칭하는 가장 정확한 용어는 **참조 타입 선언(Reference Type Declaration)**입니다.

자세히 분류해 보면 다음과 같습니다.

---

### 1. 선언할 때 사용하는 키워드들 (Type Identifiers)
단순한 `class` 외에도 설계의 목적에 따라 다음과 같은 키워드들이 그 자리에 올 수 있습니다.

* **`class`**: 일반적인 설계도입니다.
* **`interface`**: 규격(Spec)을 정의합니다. 자바 8 이후로는 구현 메서드(`default`)도 가질 수 있습니다.
* **`enum`**: 열거형 타입입니다. 상수의 집합을 정의할 때 사용합니다.
* **`record`**: (Java 14+) 순수하게 데이터를 담기 위한 불변(Immutable) 객체를 정의할 때 씁니다.
* **`@interface`**: 어노테이션 타입을 정의할 때 사용합니다.

### 2. 타입을 수식하는 제어자 (Modifiers)
말씀하신 `abstract`처럼 클래스의 성격을 규정하는 예약어들입니다.

* **`abstract class`**: 추상 클래스입니다. 직접 객체를 생성(`new`)할 수 없고, 반드시 상속받아 구현해야 합니다.
* **`final class`**: 최종 클래스입니다. 더 이상 상속(Extends)할 수 없도록 제한합니다.
* **`sealed class`**: (Java 17+) 상속받을 수 있는 하위 클래스를 미리 지정하여 제한하는 봉인된 클래스입니다.

---

### 3. 왜 이것들을 한데 묶어 부르는 용어가 필요할까요?
보통 자바 메타 프로그래밍(Reflection)이나 라이브러리를 설계할 때 이들을 하나로 묶어 처리해야 할 일이 많기 때문입니다.



이 모든 것들은 공통적으로 다음과 같은 특징을 갖습니다.
1.  **참조 타입(Reference Type)**: 기본형이 아닌 객체로서 주소값을 가집니다.
2.  **타입 이름(Type Name)**: `Array`와 달리 개발자가 고유한 이름을 부여합니다.
3.  **메타데이터**: `Class` 객체를 통해 런타임에 정보를 조회할 수 있습니다.

---

### 요약하자면
질문하신 "그 지점에 들어가는 타입들"을 프로그래밍 언어적으로 부르고 싶다면 다음 용어 중 상황에 맞는 것을 골라 쓰시면 됩니다.

* **"타입 선언부(Type Declarations)"**: 소스 코드상의 구조를 강조할 때
* **"참조 자료형(Reference Types)"**: 데이터 저장 방식을 강조할 때
* **"사용자 정의 타입(User-defined Types)"**: 기본형과 대비하여 직접 만든 설계도임을 강조할 때

혹시 지금 작성 중이신 **세금 계산기 프로젝트**나 **데이터 수집기**에서 특정 타입을 동적으로 판별하거나 관리해야 하는 상황인가요? 어떤 맥락에서 이 명칭이 궁금하셨는지 알려주시면 더 실무적인 팁을 드릴 수 있습니다.
