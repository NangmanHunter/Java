
## Reflection
- 리플렉션
- 클래스 파일(.class)이 메모리에 로드된 후, 그 클래스의 설계도를 열어보는 행위라고 보시면 됩니다.
- 일반적인 자바 코드는 "정적"입니다. 컴파일 시점에 타입이 결정되죠. 반면 리플렉션은 **"동적"**입니다.
- **구체적인 클래스 타입을 몰라도** 메서드, 타입, 변수들에 접근할 수 있습니다.
- **private** 멤버(필드나 메서드)조차 `setAccessible(true)`를 통해 강제로 접근할 수 있습니다.


- 사실 여러분이 직접 비즈니스 로직을 짤 때는 거의 쓸 일이 없습니다. 
- 리플렉션은 주로 프레임워크나 라이브러리를 만드는 사람들이 사용합니다.


##
자바의 **리플렉션(Reflection)**은 쉽게 비유하자면 **'거울(Reflection)'을 보고 내 모습을 객관적으로 관찰하거나 조작하는 기술**입니다.

보통 우리는 코드를 짤 때 `Person p = new Person();`처럼 클래스 이름을 미리 알고 객체를 생성합니다. 하지만 리플렉션을 사용하면 **클래스의 이름이나 메서드를 미리 알지 못해도, 런타임(프로그램 실행 중)에 클래스 정보를 캐내어 강제로 실행**할 수 있습니다.



---

## 2. 왜 이름이 '리플렉션'인가요?
거울을 보면 내 눈이 어디 있는지, 입이 어디 있는지 '나 자신'을 객관적으로 파악할 수 있죠? 자바 객체가 런타임에 **"아, 내 클래스 이름은 `Member`고, `id`라는 필드가 있고, `login`이라는 메서드가 있구나"**라고 스스로를 들여다보는 것과 같아서 리플렉션이라고 부릅니다.

---

## 3. 언제 사용하나요? (왜 배우나요?)
사실 여러분이 직접 비즈니스 로직을 짤 때는 거의 쓸 일이 없습니다. 리플렉션은 주로 **프레임워크나 라이브러리**를 만드는 사람들이 사용합니다.

* **Spring:** `@Service`, `@RestController` 같은 어노테이션이 붙은 클래스를 찾아내서 자동으로 객체를 생성해 줄 때 리플렉션을 씁니다. (스프링이 여러분이 만든 클래스 이름을 미리 알 방법이 없으니까요!)
* **JPA/Hibernate:** 엔티티 객체의 필드 값을 읽어서 데이터베이스에 매핑할 때 씁니다.
* **Jackson (JSON 변환):** DTO 객체를 JSON으로 바꿀 때, 객체 안에 어떤 필드가 있는지 리플렉션으로 훑어서 변환합니다.

---

## 4. 간단한 코드 예시
클래스 이름만 알아도 객체를 생성하고 메서드를 호출하는 방식입니다.

```java
// 1. 클래스 정보 가져오기 (이름만으로)
Class<?> clazz = Class.forName("com.example.Member");

// 2. 기본 생성자로 객체 생성하기
Object member = clazz.getDeclaredConstructor().newInstance();

// 3. 메서드 찾아서 실행하기 (메서드 명: "getName")
Method method = clazz.getMethod("getName");
String name = (String) method.invoke(member); 
```

---

## 5. 주의할 점 (치명적인 단점)
리플렉션은 강력하지만 그만큼 대가가 따릅니다.

1.  **성능 저하:** 컴파일 시점이 아니라 실행 중에 정보를 찾기 때문에 일반적인 메서드 호출보다 훨씬 느립니다.
2.  **타입 안전성 없음:** 코드상에서는 오류가 안 나는데, 실행했을 때 해당 이름의 메서드가 없으면 바로 에러(`RuntimeException`)가 발생합니다.
3.  **캡슐화 위반:** `private` 필드를 억지로 열어버리기 때문에 객체 지향의 설계 원칙을 깨뜨릴 위험이 있습니다.

### **💡 요약**
> **"리플렉션은 소스코드를 컴파일할 때는 정체를 알 수 없는 클래스나 객체를, 프로그램 실행 중에 동적으로 분석하고 조작할 수 있게 해주는 마법의 돋보기다."**

지금 만들고 계신 **DTO 설계**와도 관련이 있습니다. 나중에 JSON 데이터를 DTO로 변환할 때, 라이브러리가 리플렉션을 사용해 DTO의 생성자를 호출하거든요. 그래서 DTO에 **기본 생성자**가 꼭 필요한 이유이기도 합니다.
