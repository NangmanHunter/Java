- 사용자가 질문하신 Nginx가 바로 이 경우입니다.
- 리버스 프록시는 그 반대(뒤쪽), 즉 서버와 같은 방향에 서서 클라이언트를 맞이합니다.
- 클라이언트 입장에서 보면 서버가 바로 앞에 있는 것 같지만, 실제로는 Nginx라는 '서버 측 대리인'이 반대 방향에서 응답을 주고 있기 때문에 Reverse(반대의/거꾸로의)라는 이름이 붙은 것입니다.
- 리버스 프록시: "우리 서버가 누군지 클라이언트한테 비밀로 해줘" (서버 보호)
- 누구를 대신하나: 서버를 대신합니다.
- 왜 쓰나: 서버가 누군지 클라이언트가 모르게 보호하고, 여러 대의 서버에 일을 나눠주고(로드 밸런싱), 암호화(SSL) 처리를 대신 하려고 씁니다.
- 비유: 대기업의 '대표 번호' 혹은 '안내 데스크'와 같습니다. 고객(클라이언트)은 안내 데스크하고만 통화하지만, 실제 업무는 뒤에 있는 수많은 부서(백엔드/프론트엔드 서버)가 처리하는 방식입니다.
