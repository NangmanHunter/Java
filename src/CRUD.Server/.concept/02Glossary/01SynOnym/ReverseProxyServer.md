- 리버스프록시서버
- Nginx.리버스프록시.리버스 프록시 (Reverse Proxy)
- Nginx.리버스프록시.가장 대표적인 명칭입니다. 클라이언트(사용자)의 요청을 대신 받아서 내부 서버(프론트엔드나 백엔드)로 전달해주는 역할을 합니다. 보안상 내부 서버의 IP를 숨길 수 있고, 요청을 효율적으로 분배할 수 있습니다.
- Nginx.로드밸런서.로드 밸런서 (Load Balancer)
- Nginx.로드밸런서.만약 백엔드 서버가 여러 대 있을 때, Nginx가 요청을 여러 서버에 골고루 나눠준다면 이를 **부하 분산 서버** 또는 로드 밸런서라고 부릅니다.
- Nginx.웹서버.웹 서버 (Web Server)
- Nginx.웹서버.Nginx 자체는 원래 정적 파일(HTML, CSS, JS, 이미지 등)을 서빙하는 웹 서버입니다. 프론트엔드 빌드 파일을 Nginx가 직접 호스팅하고 있다면 웹 서버로서의 역할을 수행하고 있는 것입니다.
- Nginx.API게이트웨이.API 게이트웨이 (API Gateway)
- Nginx.API게이트웨이.경로에 따라 요청을 다르게 보낼 때(예: `/api`는 백엔드로, `/`는 프론트엔드로) 입구 역할을 한다고 해서 API 게이트웨이 혹은 **엔트리 포인트(Entry Point)**라고 부르기도 합니다.
- 일반적인 웹 개발 아키텍처 문맥에서는 **"Nginx를 리버스 프록시로 앞단에 뒀다"**라고 표현하는 것이 가장 정확하고 흔한 표현입니다.
- 프론트엔드와 백엔드 서버 앞에 Nginx를 두는 구조에서 Nginx는 보통 **리버스 프록시(Reverse Proxy) 서버**라고 부릅니다.
