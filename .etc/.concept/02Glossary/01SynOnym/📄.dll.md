- DLL은 Dynamic Link Library(동적 연결 라이브러리)의 약자입니다.
- 쉽게 말해, 프로그램이 실행될 때 필요한 '공용 부품 상자'라고 생각하시면 됩니다. 윈도우 운영체제에서 아주 흔히 볼 수 있는 파일 형식이죠.
- Dynamic Link Library
- DLL
- .dll
- 📄.dll
- `📄.dll`
- 예전에는 프로그램을 만들 때 필요한 모든 기능을 하나의 실행 파일(.exe)에 다 집어넣었습니다. 하지만 그러면 파일 덩치가 너무 커지고, 똑같은 기능을 여러 프로그램이 중복해서 가지게 되는 낭비가 생겼죠.
- 그래서 윈도우는 "자주 쓰는 기능은 따로 떼서 공유하자!"라고 결정했고, 그 결과물이 바로 DLL입니다.
- 비유: 레고 성(.exe)을 조립할 때, 특수 브릭들(바퀴, 창문 등)을 성 안에 미리 넣어두는 게 아니라 옆에 부품 상자(.dll)로 따로 둬서 필요할 때마다 꺼내 쓰는 방식입니다.
- DLL의 핵심 특징 ① 공유 경제 (메모리 절약)👉예를 들어 '화면에 글자를 그리는 기능'이 담긴 graphics.dll이 있다면, 그림판도, 워드도, 게임도 이 파일 하나만 같이 쓰면 됩니다. 덕분에 컴퓨터 메모리를 아낄 수 있습니다.
- DLL의 핵심 특징 ② 업데이트가 쉬움👉프로그램 전체를 새로 설치할 필요 없이, 문제가 생긴 특정 부품(.dll)만 새 버전으로 갈아 끼우면 프로그램 성능이 좋아지거나 버그가 고쳐집니다.
- DLL의 핵심 특징 ③ 실행될 때 연결됨 (Dynamic Link)👉컴퓨터 전원을 켜자마자 실행되는 게 아니라, 프로그램(.exe)이 돌아가다가 "어? 이 기능이 필요하네?" 하고 부르는 순간에만 메모리에 올라와서 연결됩니다.
- 아까 질문하신 jvm.dll이 바로 이 개념의 핵심입니다.
- java.exe라는 실행 파일은 사실 텅 빈 껍데기에 가깝습니다.
- 사용자가 자바 프로그램을 실행하면, java.exe가 실행되면서 얼른 jvm.dll이라는 부품 상자를 가져와서 내 몸에 합체시킵니다.
- 그 합체가 완료된 상태가 바로 우리가 부르는 '가상 머신(JVM)' 환경인 것입니다.
- `.dll`
- `.dll`.`.exe`라이브러리
- `.exe`.`.dll`
- "DLL 파일이 없습니다" 오류는 왜 뜰까요?👉게임을 실행할 때 가끔 이런 메시지가 뜨는 이유는, 프로그램을 돌리려는데 필요한 부품 상자가 제 자리에 없기 때문입니다. 보통 해당 라이브러리를 다시 설치하거나(Visual C++ 재배포 가능 패키지 등), 프로그램 설치 폴더에 DLL을 직접 넣어주면 해결되곤 하죠.
- .exe: 직접 실행되는 .dll: 본체가 필요할 때 빌려 쓰는 부품 도서관
