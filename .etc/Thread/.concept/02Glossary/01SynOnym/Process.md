- 프로세스 (Process)
- 프로세스는 실행 중인 프로그램의 인스턴스로, 운영체제로부터 독립된 메모리 영역을 할당받습니다.
- 독립성: 각 프로세스는 자신만의 Code, Data, Stack, Heap 영역을 가집니다.
- 안정성: 한 프로세스가 오류로 종료되어도 다른 프로세스에 직접적인 영향을 주지 않습니다.
- 통신 비용: 프로세스 간에 데이터를 주고받으려면 IPC(Inter-Process Communication)라는 복잡한 과정을 거쳐야 하므로 비용이 큽니다.

- 프로세스: 설계도를 바탕으로 공장을 세우고 재료(메모리)를 준비한 상태 (실행 중)
- 프로세스.실행프로그램
- 프로세스는 실행 중인 프로그램 그 자체를 의미합니다. 운영체제는 프로그램을 실행할 때마다 독립된 메모리 공간(코드, 데이터, 힙, 스택)을 통째로 할당해 줍니다.
- 특징: 각 프로세스는 서로의 영역을 침범할 수 없습니다. 하나가 죽어도 다른 프로세스에 직접적인 영향을 주지 않는 '독립된 섬' 같은 존재입니다.
- 비유: 하나의 '회사'라고 생각하면 쉽습니다. 회사마다 사무실, 자금, 장비가 따로 배정되는 것과 같습니다.

- 실행 중인 프로그램
- 독립적 (공유하려면 복잡한 통신 필요)
- 무거움 (생성/종료 시 비용이 큼)
- 하나가 죽어도 나머지는 안전함
- 프로세스 (동적인 상태): 사용자가 프로그램을 실행해서 메모리에 적재되고, CPU가 그 명령어를 읽기 시작한 상태입니다. 즉, '생명력이 부여된 프로그램'입니다.
- 단순히 "실행 중인 프로그램"이라고 안 하고 "프로세스"라는 어려운 단어를 쓰는 이유는, 하나의 프로그램으로 여러 개의 프로세스를 만들 수 있기 때문입니다.
- 프로세스: 하지만 작업관리자를 열어보면 실행 중인 '크롬 프로세스'는 **3개(혹은 그 이상)**가 돌아가고 있습니다.
- 1프로그램3프로세스
- 1프로그램N프로세스
- 프로그램은 하나지만 그 프로그램에서 파생되어 실제로 일하고 있는 개체들은 여러 개일 수 있는 것이죠.
- 프로세스: "나 지금 메모리 차지하고 CPU 써서 일하는 중이야!" (활동 상태)
- 프로세스 (자식): 그 프로그램이 메모리에 올라가서 실체화된 '인스턴스'입니다.
