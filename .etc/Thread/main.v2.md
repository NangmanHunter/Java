### 3. 주요 차이점 요약
| 구분 | 프로세스 (Process) | 스레드 (Thread) |
| :--- | :--- | :--- |
| 정의 | 실행 중인 프로그램의 단위 | 프로세스 내의 실행 흐름 단위 |
| 자원 할당 | OS로부터 독립된 메모리 할당 | 프로세스의 자원을 공유 |
| 통신 (IPC) | 어려움 (별도 메커니즘 필요) | 쉬움 (공유 메모리 사용) |
| 영향도 | 독립적임 (안전) | 프로세스 전체에 영향 (위험) |
| 비용 | 생성 및 전환 비용이 높음 | 생성 및 전환 비용이 낮음 |

---

### 4. 왜 구분이 중요한가요?
백엔드 개발이나 시스템 설계를 할 때 이 차이를 아는 것이 중요합니다. 
예를 들어, 멀티 프로세스 방식은 안정성이 중요한 작업(예: 크롬 브라우저의 각 탭)에 유리하고, 
멀티 스레드 방식은 자원을 효율적으로 쓰면서 빠른 응답이 필요한 서비스(예: 웹 서버의 요청 처리)에 자주 사용됩니다.

공부하시다가 자바의 멀티스레딩이나 동기화(Synchronization) 문제에 대해 더 궁금한 점이 생기면 말씀해 주세요!
