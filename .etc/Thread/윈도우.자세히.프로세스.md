윈도우 작업 관리자의 **[자세히]** 탭에 표시되는 목록은 기본적으로 **프로세스(Process)**입니다.

사용자가 실행 중인 각 프로그램과 백그라운드에서 돌아가는 서비스들의 개별 인스턴스를 보여주는 곳입니다. 하지만 그 안에서 스레드 정보를 확인하거나 구분하는 방법도 있습니다.

---

### 1. 왜 [자세히] 탭은 프로세스인가요?
* **PID (프로세스 ID):** 목록을 보면 각 항목마다 고유한 숫자 번호가 붙어 있는데, 이것이 프로세스 식별 번호입니다.
* **독립적 자원:** 각 항목은 CPU, 메모리(작업 세트) 등 운영체제로부터 할당받은 독립적인 자원 사용량을 보여줍니다.
* **강제 종료:** 특정 항목을 선택해 '작업 끝내기'를 하면 그 프로세스에 속한 모든 스레드가 한꺼번에 종료됩니다.

### 2. 그럼 스레드는 어디서 보나요?
[자세히] 탭에서 마우스 오른쪽 버튼을 클릭하여 스레드 정보를 추가로 띄울 수 있습니다.

1.  **[자세히]** 탭의 열 이름(이름, PID 등) 위에서 **마우스 우클릭**을 합니다.
2.  **[열 선택]**을 누릅니다.
3.  목록에서 **[스레드]** 항목을 찾아 체크하고 확인을 누릅니다.
4.  이제 각 프로세스 옆에 숫자가 나타나는데, 이 숫자가 바로 해당 프로세스가 내부적으로 부리고 있는 **일꾼(스레드)의 수**입니다.



---

### 3. [프로세스] 탭 vs [자세히] 탭
작업 관리자에는 탭이 여러 개라 헷갈릴 수 있는데, 차이점은 다음과 같습니다.

* **[프로세스] 탭:** 앱, 백그라운드 프로세스 등으로 그룹화해서 보여주는 '사용자 친화적' 화면입니다. (예: 크롬 창 10개가 하나로 묶여 보임)
* **[자세히]** 탭:** 그룹화 없이 실제 OS가 관리하는 실제 실행 단위(인스턴스)를 하나하나 나열한 '관리자용' 화면입니다. (예: 크롬의 메인 프로세스, GPU 프로세스 등이 각각 따로 보임)


### 요약
작업 관리자의 **[자세히]** 탭 목록은 **프로세스**이며, 각 프로세스 내부에 몇 개의 **스레드**가 실행 중인지는 열 설정을 통해 숫자로 확인할 수 있습니다. 보통 웹 브라우저나 IDE 같은 무거운 프로그램들은 수십, 수백 개의 스레드를 동시에 사용하곤 합니다.
