**SDK**는 **Software Development Kit**의 약자로, 우리말로는 **'소프트웨어 개발 키트'**라고 부릅니다. 

쉽게 비유하자면, SDK는 **'특정 가구를 만들기 위해 필요한 모든 도구와 도면이 들어있는 종합 공구 상자'**와 같습니다.

---

### 1. SDK 안에는 무엇이 들어있나요?
개발자가 밑바닥부터 모든 것을 만들지 않아도 되도록, 핵심적인 재료들이 패키지로 묶여 있습니다.

*   **라이브러리 (Libraries):** 미리 만들어진 코드 조각들입니다. (예: "화면에 버튼을 그려라", "데이터를 압축해라" 같은 기능을 직접 짤 필요 없이 불러다 씁니다.)
*   **API (Application Programming Interface):** 프로그램과 프로그램이 소통하는 규칙입니다.
*   **컴파일러 (Compiler):** 개발자가 짠 코드(Java, Rust 등)를 컴퓨터가 이해할 수 있는 기계어로 번역해주는 도구입니다.
*   **디버거 (Debugger):** 코드에서 어디가 잘못되었는지 찾아주는 탐지기입니다.
*   **문서 및 예제 코드:** 이 도구들을 어떻게 쓰는지 설명해주는 설명서입니다.



---

### 2. JDK와의 관계 (이미 사용하고 계신 것)
질문하셨던 **JDK(Java Development Kit)**가 바로 자바용 SDK입니다. 자바 프로그램을 개발하기 위해 필요한 자바 라이브러리, 자바 컴파일러(`javac`), 그리고 실행 환경(JRE) 등을 하나로 묶어놓았기 때문에 SDK의 일종이라고 부르는 것입니다.

---

### 3. SDK와 API의 차이점
가장 많이 헷갈려 하는 부분인데, 이렇게 이해하면 명확합니다.

*   **API:** 식당의 **'메뉴판'**입니다. 손님(개발자)이 주방(시스템)에 무엇을 주문할 수 있는지 알려주는 인터페이스입니다.
*   **SDK:** 식당을 차릴 수 있게 해주는 **'프랜차이즈 패키지'**입니다. 메뉴판(API)은 물론이고, 식재료(라이브러리), 조리 기구(툴), 레시피(설명서)까지 통째로 제공하는 개념입니다.

> 즉, **SDK 안에는 보통 API가 포함되어 있습니다.**

---

### 4. 왜 SDK를 쓰나요?
결론은 **생산성** 때문입니다. 

- 예를 들어, "구글 지도를 활용한 앱"을 만든다면 구글이 제공하는 **Google Maps SDK**를 내려받으면 됩니다. 지도 데이터를 어떻게 가져오고 화면에 어떻게 띄울지 고민할 시간에, SDK가 제공하는 기능을 조립해서 핵심 서비스 로직에만 집중할 수 있게 해줍니다.
