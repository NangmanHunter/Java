자바 백엔드 개발을 공부하다 보면 **스레드(Thread)**와 **커넥션(Connection)**이라는 용어가 계속 같이 나와서 헷갈릴 수 있습니다. 둘 다 '통로'나 '일꾼' 같은 느낌이라 비슷해 보이지만, **어디서 무엇을 하느냐**에 따라 역할이 명확히 갈립니다.

쉽게 비유하자면, **스레드는 '요리사'**이고, **커넥션은 '식재료를 실어 나르는 도로'**라고 생각하시면 됩니다.

---

### 1. 스레드 (Thread)
스레드는 프로세스 내에서 **실제로 작업을 실행하는 주체**입니다. 자바 어플리케이션(JVM) 안에서 CPU를 점유하고 코드를 한 줄 한 줄 읽으며 계산을 수행합니다.

*   **역할:** 비즈니스 로직 처리, 데이터 계산, API 호출 등 실질적인 '일'을 함.
*   **비유:** 식당에서 주문을 받고 요리를 하는 **요리사**.
*   **자바에서의 관리:** 주로 `Thread Pool`을 통해 미리 여러 명의 요리사를 대기시켜 놓고 재사용합니다.



---

### 2. 커넥션 (Connection)
커넥션은 주로 **데이터베이스(DB)와의 연결 통로**를 의미합니다. 자바 서버가 DB에 있는 데이터를 가져오거나 넣으려면 물리적인 연결이 필요한데, 그 연결선 자체를 커넥션이라고 부릅니다.

*   **역할:** 서버와 DB 사이의 데이터 전송로.
*   **비유:** 식재료 창고(DB)와 식당(서버)을 잇는 **배달 도로**.
*   **자바에서의 관리:** DB 연결은 생성 비용이 비싸기 때문에 `DBCP (Database Connection Pool)`라는 곳에 도로(연결선)를 미리 깔아두고 사용합니다.



---

### 3. 핵심 차이점 요약

| 구분 | 스레드 (Thread) | 커넥션 (Connection) |
| :--- | :--- | :--- |
| **주체** | **WAS(서버) 내부**의 일꾼 | **서버와 외부(DB)** 사이의 통로 |
| **목적** | CPU를 써서 명령어를 실행함 | 데이터를 주고받기 위한 통신망 유지 |
| **리소스** | 메모리와 CPU를 주로 사용 | 네트워크 소켓과 포트를 사용 |
| **비유** | 요리사 (Work) | 배달 도로 (Transport) |

---

### 4. 둘은 어떻게 상호작용하나요? (Flow)

일반적인 웹 요청이 들어오면 다음과 같은 순서로 움직입니다.

1.  **스레드 할당:** 사용자의 요청이 들어오면 **스레드 풀**에서 요리사(스레드) 한 명이 배정됩니다.
2.  **커넥션 획득:** 요리사가 요리를 하려는데 재료가 부족하면, **커넥션 풀**에서 도로(커넥션)를 하나 빌려 DB에 접근합니다.
3.  **데이터 처리:** DB에서 가져온 데이터를 가지고 요리사(스레드)가 로직을 처리합니다.
4.  **반납:** 작업이 끝나면 요리사는 스레드 풀로, 도로는 커넥션 풀로 각각 **반납**됩니다.

> **주의할 점:** 
> 만약 요리사는 100명인데 도로는 10개뿐이라면? 90명의 요리사는 도로가 비기를 기다리며 손을 놓고 있게 됩니다. 반대로 도로는 많은데 요리사가 적으면 효율이 떨어지겠죠. 그래서 이 두 설정값의 **밸런스**를 맞추는 것이 성능 최적화의 핵심입니다.
