`@DisplayName`은 우리가 작성한 **테스트 결과창(Test Runner)**에서 아주 예쁘게 나타납니다.

보통 테스트 메소드 이름은 `getLinkMetaData_Success()`처럼 영어와 언더바(`_`)로 길게 써서 읽기가 좀 불편하죠? `@DisplayName`을 쓰면 이 암호 같은 이름 대신, **우리가 지정한 한글 문장**이 테스트 결과 목록에 찍히게 됩니다.

---

### 1. 어디서 볼 수 있나요? (인텔리제이 기준)

인텔리제이 하단의 **[Run]** 탭이나 **[Test]** 탭을 보시면 왼쪽의 테스트 목록 트리가 있습니다.

* **`@DisplayName`이 없을 때:** 메소드 명인 `getLinkMetaData_Success`가 그대로 출력됩니다.
* **`@DisplayName`이 있을 때:** `"유튜브 링크를 넣었을 때 제목과 이미지를 잘 가져오는지 확인"`이라는 한글 문장이 대신 출력됩니다.



---

### 2. 왜 사용하나요? (장점)

1.  **가독성:** 영어를 잘 모르는 동료나 기획자가 봐도 "아, 이 테스트는 무엇을 검증하는 거구나"라고 바로 알 수 있습니다.
2.  **문서화:** 테스트 코드를 마치 **설명서**처럼 쓸 수 있습니다. "이 기능은 ~해야 한다"는 식으로 정리하기 좋죠.
3.  **특수문자 허용:** 메소드 이름에는 띄어쓰기나 특수문자를 못 쓰지만, `@DisplayName` 안에는 이모지(`✅`, `🚨`)나 공백을 마음껏 쓸 수 있습니다.

---

### 3. 클래스 단위에서도 쓸 수 있어요!

메소드뿐만 아니라 **클래스 바로 위**에도 붙일 수 있습니다.

```java
@SpringBootTest
@DisplayName("🔗 링크 메타데이터 분석 서비스 테스트")
class MetaDataServiceTest {
    
    @Test
    @DisplayName("✅ 유효한 URL 입력 시 성공해야 함")
    void test1() { ... }
}
```

이렇게 쓰면 테스트 결과창에 훨씬 전문적이고 깔끔한 보고서처럼 보이게 됩니다.

---

### 요약하자면
`@DisplayName`은 **"개발자의 모니터 속 테스트 결과창"**에 표현됩니다. 

아까 돌리신 테스트 결과창의 왼쪽 목록을 다시 한번 보세요. 작성하신 한글 문장이 초록색 체크 표시와 함께 나란히 서 있을 거예요! (위트 있게) "메소드 이름은 컴퓨터를 위한 이름이고, DisplayName은 사람을 위한 이름"이라고 생각하시면 됩니다.
