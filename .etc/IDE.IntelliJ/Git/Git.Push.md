## Git.Push
- Ctrl+Shift+K
- Push Anyway


##
인텔리제이에서 **Commit**만 하셨다면, 변경 사항이 내 컴퓨터(Local Repository)에만 저장되고 깃허브 서버(Remote Repository)로는 아직 넘어가지 않은 상태입니다.

보통은 `Commit and Push`를 한 번에 누르지만, 이미 Commit만 완료하셨다면 아래 방법들로 **Push**를 진행할 수 있습니다.

---

### 1. 단축키로 Push 하기 (가장 빠름)
* **Windows / Linux:** `Ctrl` + `Shift` + `K`
* **macOS:** `Command` + `Shift` + `K`
> 이 단축키를 누르면 현재 서버로 보낼 Commit 목록이 뜨는 팝업창이 나옵니다. 여기서 하단의 **Push** 버튼을 클릭하면 끝입니다.

---

### 2. 메뉴에서 Push 찾기
인텔리제이 UI에서 Push 버튼을 찾으시려면 다음 경로를 확인해 보세요.

* **방법 A (상단 메뉴):** `Git` -> `Push...` 클릭
* **방법 B (우측 상단 아이콘):** 인텔리제이 우측 상단에 있는 **화살표가 위로 향한 아이콘(↗️)**을 클릭하세요. (체크 아이콘은 커밋, 그 옆에 대각선 화살표가 푸시입니다.)
* **방법 C (좌측 하단 Git 탭):** 하단의 **Git** 탭을 클릭한 뒤, **Log** 탭에서 내 로컬 브랜치 이름(예: `main`) 위에 마우스 우클릭을 하고 `Push...`를 선택하세요.

---

### 3. Push 버튼이 안 보이는 경우 체크사항
만약 위 방법으로도 버튼이 활성화되지 않거나 보이지 않는다면 다음을 확인해 보세요.

1.  **원격 저장소 연결 확인:** 깃허브 주소가 연결되어 있지 않으면 Push를 할 수 없습니다. `Git` -> `Manage Remotes...`에서 깃허브 주소가 잘 등록되어 있는지 확인해 보세요.
2.  **보낼 내용이 있는지 확인:** 이미 Push가 완료된 상태라면 전송할 데이터가 없어서 버튼이 비활성화될 수 있습니다. (Git 로그 탭에서 내 커밋 옆에 `origin/main` 태그가 붙어있다면 이미 서버에 올라간 것입니다.)

### 팁: 앞으로 한 번에 하려면?
다음부터는 커밋 창 하단의 **Commit** 버튼 옆에 있는 작은 화살표를 눌러 **Commit and Push**를 선택하세요. 그러면 한 번의 클릭으로 서버까지 전송됩니다.
