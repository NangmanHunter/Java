##
IntelliJ IDEA나 Git을 사용할 때 가장 헷갈리는 부분 중 하나죠. 핵심만 말씀드리면 **'이미 커밋을 원격 저장소(GitHub 등)에 올렸느냐(Push)'**에 따라 선택이 달라집니다.

---

### 1. Undo Commit (방금 한 실수를 되돌리기)
"아, 방금 커밋 메시지 잘못 썼다!" 혹은 "파일 하나 빼먹고 커밋했네?" 할 때 사용합니다.

* **동작 방식:** 마지막 커밋을 취소하고, 해당 내용을 커밋하기 전 상태(Staging Area)로 되돌려 놓습니다.
* **특징:** 커밋 기록 자체가 **삭제**됩니다. 
* **언제 쓰나요?:** 아직 **Push하기 전**, 내 로컬 컴퓨터에만 커밋이 남아있을 때 씁니다.
* **주의사항:** 이미 Push한 커밋에 대해 `Undo`를 하고 다시 Push하려고 하면, 원격 저장소와 이력이 맞지 않아 `Force Push`가 필요해지며 팀원들의 이력이 꼬일 수 있습니다.

### 2. Revert Commit (이미 공유된 기록을 반대로 되돌리기)
"어제 올린 기능에 버그가 있네? 일단 이전 상태로 되돌려놓자!" 할 때 사용합니다.

* **동작 방식:** 대상 커밋의 내용을 정반대로 수행하는 **새로운 커밋**을 만듭니다. (예: A 문장을 추가한 커밋을 Revert하면, A 문장을 삭제하는 커밋이 생성됨)
* **특징:** 기존 커밋 기록은 **그대로 유지**되며, 그 뒤에 '취소용 커밋'이 하나 더 붙습니다.
* **언제 쓰나요?:** 이미 **Push를 완료**해서 팀원들과 공유된 커밋을 되돌려야 할 때 씁니다.
* **장점:** 히스토리를 파괴하지 않기 때문에 협업 시 가장 안전한 방법입니다.

---

### 한눈에 비교하기

| 구분 | Undo Commit | Revert Commit |
| :--- | :--- | :--- |
| **핵심 목적** | 커밋 자체를 **삭제**하고 수정하기 | 커밋은 두고 **반대 내용**을 추가하기 |
| **히스토리 변화** | 커밋 기록이 사라짐 | 새로운 'Revert' 기록이 생성됨 |
| **권장 상황** | Push 하기 전 (로컬 작업 중) | Push 한 후 (협업 중) |
| **안전성** | 혼자 작업 시 편리함 | 협업 시 매우 안전함 |

---

### 💡 상황별 가이드

1.  **커밋 버튼 누르자마자 오타를 발견했다면?**
    * `Undo Commit`을 하고 수정해서 다시 커밋하세요.
2.  **어제 Push까지 다 끝냈는데, 그 기능이 서버를 터뜨리고 있다면?**
    * `Revert Commit`을 실행하여 즉시 원복 커밋을 생성하고 다시 Push하세요.

**팁:** IntelliJ의 Git 로그 탭에서 커밋을 우클릭하면 두 메뉴를 모두 볼 수 있습니다. 현재 내 커밋이 원격(Origin)에 올라가 있는지 표시를 먼저 확인하는 습관을 들이면 실수를 줄일 수 있습니다.
