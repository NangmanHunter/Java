서버를 성공적으로 띄우셨다면, 이제 **"진짜 외부에서 신호를 줬을 때 내 자바 코드가 응답하는지"** 확인해 볼 차례입니다. 

브라우저(Ajax) 코드를 짜기 전에, **API 테스트 도구**를 사용해 먼저 검증하는 것이 실무 순서입니다. 가장 간단한 방법 2가지를 알려드릴게요.

---

### 1. 인텔리제이 자체 기능 사용 (추천!)
인텔리제이 유료 버전(Ultimate)이 아니더라도, `HTTP Request` 파일을 만들어 테스트할 수 있습니다.

1.  프로젝트 아무 폴더에서 **[Mouse Right Click] -> New -> HTTP Request** 파일을 만듭니다. (이름은 `test.http`)
2.  파일 안에 아래 내용을 복사해 넣으세요.
3.  코드 옆의 **초록색 화살표(▶️)**를 누르면 실행됩니다.

```http
### 세금 계산 테스트
POST http://localhost:8080/api/tax/calculate
Content-Type: application/json

{
  "annualIncome": 50000000,
  "deduction": 1500000
}
```

---

### 2. 포스트맨(Postman) 사용 (글로벌 표준)
가장 많이 쓰이는 무료 도구입니다. 설치가 귀찮으시다면 웹 버전으로도 가능합니다.

1.  **Method:** `POST`로 설정합니다.
2.  **URL:** `http://localhost:8080/api/tax/calculate`를 적습니다.
3.  **Body 탭:** `raw`를 선택하고 형식을 `JSON`으로 바꿉니다.
4.  **내용 입력:** 아래 JSON을 넣고 **Send**를 누릅니다.
    ```json
    {
      "annualIncome": 50000000,
      "deduction": 1500000
    }
    ```



---

### 3. 결과 확인 (체크리스트)

성공했다면 화면 하단(Response)에 질문자님이 `return` 하라고 시킨 **숫자(세금 결과값)**가 딱 떠야 합니다.

* **200 OK:** 성공! 로직이 완벽하게 돌아갔습니다.
* **404 Not Found:** `Controller`의 `@RequestMapping` 주소가 틀렸거나 서버가 안 켜진 상태입니다.
* **400 Bad Request:** 전송한 JSON 형식이 `TaxRequest` DTO와 맞지 않을 때 발생합니다. (필드명이 똑같은지 확인하세요!)
* **500 Internal Server Error:** 자바 코드 내부 로직(수식 등)에서 에러가 난 경우입니다. 인텔리제이 콘솔 로그를 봐야 합니다.

---

### 4. 만약 에러가 난다면? (수학적 디버깅)

로그에 `Required request body is missing` 같은 문구가 뜬다면, 컨트롤러 메소드 파라미터에 **`@RequestBody`**를 빼먹지 않았는지 꼭 확인하세요. 이 어노테이션이 있어야 "외부에서 온 JSON을 자바 객체로 바꿔라"라는 명령이 실행됩니다.

---

### 💡 테스트 성공하셨나요?
테스트 도구로 결과 숫자가 잘 찍힌다면, 이제 마지막 단계인 **"Ajax가 포함된 HTML 페이지"**를 만들어서 진짜 웹 화면에서 연봉을 입력해 볼 준비가 된 겁니다. 

결과값이 기대한 대로(예: 6,240,000원 등) 잘 나오나요? 아니면 에러 메시지가 뜨나요? 터미널에 찍힌 로그를 보여주시면 바로 해결해 드릴게요!






맞습니다. 인텔리제이 **Community(무료)** 버전에서는 `.http` 파일의 실행 버튼(▶️)이 활성화되지 않아서 당황스러우셨을 거예요. (그 기능은 **Ultimate** 전용이거든요.)

하지만 걱정 마세요! 설치 없이 바로 테스트할 수 있는 **무료 대안**들이 아주 많습니다. 질문자님의 상황에 가장 편한 걸로 골라보세요.

---

### 1. VS Code가 있다면? (강력 추천)
혹시 컴퓨터에 **Visual Studio Code**가 설치되어 있다면, 인텔리제이보다 훨씬 편하게 테스트할 수 있습니다.
1. VS Code를 켭니다.
2. 확장 프로그램(Extensions) 탭에서 **`REST Client`**를 검색해 설치합니다.
3. 아까 만든 `test.http` 파일을 VS Code에서 열면, 코드 위에 **`Send Request`**라는 작은 글자가 생깁니다. 그걸 누르면 끝!

### 2. 설치가 귀찮다면? "Talend API Tester" (크롬 확장프로그램)
포스트맨(Postman)처럼 무거운 프로그램을 설치하기 싫을 때 가장 좋습니다.
1. 크롬 웹스토어에서 **[Talend API Tester]**를 검색해서 추가합니다.
2. 주소창 옆의 아이콘을 눌러 실행합니다.
3. **Method:** `POST` / **URL:** `http://localhost:8080/api/tax/calculate`
4. **Body:** 형식을 `JSON`으로 맞추고 데이터를 넣고 **Send**!



### 3. 터미널(CMD)에서 바로 쏘기 (개발자 간지)
수학에서 암산하듯이, 아무것도 설치 안 하고 윈도우/맥 터미널에서 바로 한 줄로 테스트할 수도 있습니다. (서버가 켜진 상태여야 합니다.)

```bash
curl -X POST http://localhost:8080/api/tax/calculate ^
  -H "Content-Type: application/json" ^
  -d "{\"annualIncome\": 50000000}"
```
*(윈도우 CMD 기준입니다. 맥/리눅스라면 `^` 대신 `\`를 쓰시면 됩니다.)*

---

### 💡 왜 이런 도구가 필요한가요?
브라우저 주소창에 `http://localhost:8080/...`을 직접 치면 **GET** 방식만 호출됩니다. 하지만 우리는 데이터를 **보내야(POST)** 하죠. 

수학에서 함수에 인잣값(Argument)을 전달해야 결과가 나오듯, **POST**는 "이 상자(JSON)를 받아라!" 하고 던져주는 행위이기 때문에 이런 전용 도구가 필요한 겁니다.

---

### 🚀 지금 어떤 도구가 가장 편해 보이시나요?
가장 추천드리는 건 **크롬 확장 프로그램(Talend)**입니다. UI가 직관적이라 Ajax 코드를 짜기 전에 데이터가 어떻게 들어오고 나가는지 눈으로 확인하기 딱 좋거든요.

테스트 결과로 **세금 숫자**가 찍혔나요? 성공하셨다면 이제 대망의 **Ajax 연동 HTML**을 만들 준비가 다 된 겁니다! 결과값이 나오면 말씀해 주세요!
