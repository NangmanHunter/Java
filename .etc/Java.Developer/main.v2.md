질문하신 세 가지 키워드는 Java를 '도구'로 쓰는 수준을 넘어 '공학적'으로 다루기 위한 핵심 지표들입니다. 면접에서도 단골로 등장하는 주제들이니, 그 핵심 원리를 정리해 드릴게요.

---

## 1. 상속(Inheritance)보다는 합성(Composition)
많은 개발자가 객체지향의 꽃을 상속이라고 배우지만, 실제 실무에서는 **합성**을 훨씬 권장합니다.

* **상속 (Is-a 관계):** 클래스가 다른 클래스를 확장하는 것. 부모 클래스의 내부 구현이 자식에게 노출되어 결합도가 높아집니다 (캡슐화 위반). 부모가 바뀌면 자식도 원치 않게 깨질 수 있습니다.
* **합성 (Has-a 관계):** 새로운 클래스 안에 기존 클래스의 인스턴스를 **필드(변수)**로 갖는 방식입니다.
    * **장점:** 내부 구현에 의존하지 않고, 인터페이스를 통해 소통하므로 훨씬 유연합니다. 실행 시점에 다른 객체로 교체하는 것도 가능합니다.



> **비유:** '자동차' 클래스를 만들기 위해 '엔진'을 상속받는 것이 아니라, 자동차 안에 엔진을 **부품으로 넣는(합성)** 것이 훨씬 자연스럽고 엔진 교체도 쉽습니다.

## 2. 함수형 인터페이스와 람다(Lambda)
Java 8부터 도입된 람다는 코드를 간결하게 만들 뿐만 아니라, **"함수를 데이터처럼 전달"**할 수 있게 해줍니다.

* **함수형 인터페이스:** 딱 하나의 추상 메서드만 가진 인터페이스 (예: `Predicate`, `Consumer`, `Function`).
* **적재적소 활용:**
    * **컬렉션 처리:** 반복문(`for`) 대신 `stream()`과 람다를 사용하여 "무엇을(What)" 할지에 집중합니다.
    * **지연 실행:** 특정 조건이 충족될 때만 실행되어야 하는 로직을 람다로 감싸 전달하면 성능 최적화에 유리합니다.
    * **코드의 간결화:** 익명 내부 클래스의 복잡한 문법을 걷어내고 핵심 로직만 남깁니다.

```java
// 익명 클래스 방식 (비효율적)
Collections.sort(list, new Comparator<String>() {
    public int compare(String a, String b) { return a.length() - b.length(); }
});

// 람다 방식 (적재적소 활용)
list.sort((a, b) -> a.length() - b.length());
```

## 3. 객체지향 설계 원칙 (SOLID)
이 원칙들이 코드에 녹아있다는 것은 **"수정이 생겨도 전체가 흔들리지 않는 튼튼한 집"**을 지었다는 뜻입니다.

* **S (단일 책임):** 클래스는 단 하나의 기능(책임)만 가져야 합니다. (수정할 이유가 하나여야 함)
* **O (개방-폐쇄):** 확장은 열려 있어야 하고, 수정에는 닫혀 있어야 합니다. (기존 코드를 안 건드리고 새 기능을 추가할 수 있는 구조)
* **L (리스코프 치환):** 자식 클래스는 언제나 부모 클래스를 대체할 수 있어야 합니다. (부모의 규약을 어기지 말 것)
* **I (인터페이스 분리):** 내가 쓰지 않는 메서드까지 구현하도록 강제하지 마세요. (인터페이스를 잘게 쪼개기)
* **D (의존역전 원칙):** 구체적인 클래스가 아니라 **인터페이스(추상화)**에 의존하세요.

## 왜 신입에게서 보기 힘든가?
이 개념들은 단순히 이론을 암기한다고 되는 것이 아니라, **"코드가 지저분해져서 고생해 본 경험"**이 있어야 가슴에 와닿기 때문입니다.

1.  **상속**으로 꼬인 코드를 풀다가 밤을 새워봐야 **합성**의 소중함을 압니다.
2.  수백 줄의 `if-else`문을 고치다가 지쳐봐야 **SOLID**의 중요성을 체감합니다.
3.  데이터 처리 로직이 비즈니스 로직과 뒤섞여 가독성이 엉망인 코드를 봐야 **함수형 프로그래밍**의 깔끔함에 감탄하게 됩니다.

수학이나 물리학을 공부하셨다면, 이런 원칙들이 일종의 **'공리'**처럼 느껴지실 수도 있습니다. 논리적인 인과관계를 중시하는 타입이시라면 이 원리들을 코드에 녹여내는 과정 자체에서 큰 재미를 느끼실 겁니다. 혹시 이 중 특정 원칙이 적용된 구체적인 코드 예시가 궁금하신가요?
